Prerequisites for using SDK

To benefit most from the Active Roles SDK documentation and samples, the reader should be familiar with the use of scripting languages with ADSI. Knowing the Active Roles features and functionality is also necessary. Below is some prerequisite information to be considered before using  the Active Roles SDK.

  • Knowledge of Microsoft ADSI scripting is required to develop custom user interfaces and policy scripts that involve the use of ADSI-compliant interfaces. For more information about Microsoft ADSI scripting, please refer to the ADSI Scripting Tutorial in the MSDN Library.

  • Knowledge of the Active Roles security model is required to ensure that custom solutions comply with the existing permission settings. These settings specify what administrative operations a given user is allowed to perform, and what directory objects a given user is allowed to access and modify. For example, to be able to configure script policy, the user needs administrative access to the Administration Service computer.

    Additionally, since the Active Roles ADSI Provider connects to the Administration Service as the user in whose security context the client application or script is running, it is only allowed to retrieve and modify the object properties which that user is permitted to access and change with Active Roles. For more information about how Active Roles controls access to directory object data, refer to the Active Roles Administrator Guide.
