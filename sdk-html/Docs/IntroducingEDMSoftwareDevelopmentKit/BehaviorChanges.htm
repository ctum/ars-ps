Behavior changes

This topic describes the changes in behavior of some Active Roles features in the current version from their behavior in earlier versions of Active Roles.

 

ADSI Provider

When you bind to a directory object with the GetObject function, using the binding string that specifies a server name, for example, such as

Set objUser = GetObject("EDMS://myServer.mycompany.com/CN=John Smith,OU=Research,DC=Gamp,DC=com")

the provider will automatically select a service to which to connect in the following way:

Behaviour in Earlier Versions

Changed in Version

Behaviour in Current Version

The provider first searches the forest to which the domain Gamp.com belongs for any Administration service managing that domain.  If no Administration service is found, the provider will search the forest to which the domain mycompany.com belongs. If no Administration service is found, the provider will search the forest to which the current user's home domain belongs.

 

6.0

The provider first tries to connect to the Administration service running on the computer myServer.mycompany.com, if any. If this operation fails, the provider searches the forest to which the domain Gamp.com belongs for any Administration service managing that domain.  If no Administration service is found, the provider will search the forest to which the domain mycompany.com belongs. If no Administration service is found, the provider will search the forest to which the current user's home domain belongs.
