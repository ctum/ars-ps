The Menus Settings

This topic provides information about  the format of settings for menus associated with directory objects managed with the Web Interface. A menu represents a list of commands relating to objects of the appropriate type. The commands used in menus must be described in the attribute edsaWICommands of the WorkingCopy object (see The Commands Settings). Examples of menus: the User menu, the Group menu, etc.

Settings for menus are stored in the attribute edsaWIMenus of the WorkingCopy object. Modifying this attribute, you can add or remove commands from menus.

XML Elements

The attribute edsaWIMenus stores information in the Extensible Markup Language (XML) format. XML elements you can use are listed below.

Important:

After you modify the attribute edsaWIMenus, for changes to take effect, reload the Web Interface.

TaskLists

Describes the collection of menu.

Parent Elements: None

Child Elements: TaskList

Attributes:

Name

Description

CustomizationVersion

Required. Identifies the current customization version number. Must be the same for all attributes of the WorkingCopy object.

Important:

The value of this attribute depends on the application version. This value is stored into the attribute edsaWITemplateVersion of the WI configuration container objects (see Storing Customization Settings).

xmlns

Required. String. Identifies the XML-document namespace, such as "http://quest.com/ActiveRolesServer/WebInterface"

TaskList

Describes a menu from the menu collection such as the User menu, the Group menu, etc.

Parent Elements: TaskLists

Child Elements: Task

Attributes:

Name

Description

TargetClass

Required. Contains the ldapDisplayName property of the object class to which the menu is related.

DefaultTask

Optional. Identifies the default command for the menu. When this attribute is set, the element Task must contain the attribute ID set to that command name.

 Task

Describes a command of the menu identified with the parent element TaskList.

Parent Elements: TaskList

Child Elements: None

Attributes:

Name

Description

ID

Required. Contains the identifier of the menu command. The attribute edsaWICommands must contain the element Task with the same attribute ID.

Note:

All elements have the optional attribute PlainTextLabel that allows you to label elements. The use of your custom labels allows your applications to easily update or remove custom elements from the configuration settings.

Example

The following code snippet illustrates the use of the XML elements in the attribute edsaWIMenus.

<?xml version="1.0"?>
<TaskLists CustomizationVersion="33">
    <TaskList TargetClass="computer" DefaultTask="ComputerProperties">
        <Task ID="DisableAccount"/>
        <Task ID="EnableAccount"/>
        <Task ID="ResetAccount"/>
        <Task ID="Delete"/>
        <Task ID="Move"/>
        <Task ID="RestartComputer"/>
        <Task ID="ManageComputerResource"/>
        <Task ID="ViewRSOP"/>
        <Task ID="MemberOf"/>
        <Task ID="ComputerProperties"/>
   </TaskList>
</TaskLists>
