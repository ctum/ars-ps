Adding/Removing Nodes from the Tree View

This topic explains how you can add or remove a node from the Tree View. The Tree View allows you to navigate through hierarchical structures of data containers, making the location of the data easily discoverable.

To customize the Tree View, use the following procedure to make the appropriate changes to the attribute edsaWISettings of the object WorkingCopy residing in the Interface Setting container.

To add a node to the Tree View

  1. Save the value of the attribute edsaWISettings into a text file.

  2. Open this file with any XML editor, such as XML Spy, and find the element RootTreeNodes.

  3. For the element RootTreeNodes, insert the child element Node in the following way:
    <Node> Node_DN</Node>, where the Node_DN refers to the distinguished name of the node to be added. When finished, save the file.

  4. Save changes to the attribute edsaWISettings.

  5. Reload the Web Interface.

To remove nodes from the Tree View

  1. Perform the steps 1 to 2 from the procedure above.

  2. For the element RootTreeNodes, remove child elements Node that correspond to nodes to be removed.

  3. Save changes to the attribute edsaWISettings.

  4. Reload the Web Interface.

Example

The following code snippet illustrates how to have the Tree View include the following nodes: Active Directory, Managed Units, Sales.

<RootTreeNodes>
  <Node>CN=Active Directory</Node>
  <Node>CN=Managed Units,CN=Configuration</Node>
  <Node>OU=Sales,DC=mycompany,DC=com</Node>
</RootTreeNodes>
