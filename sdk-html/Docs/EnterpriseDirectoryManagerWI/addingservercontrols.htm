Using Server-Side Controls

This topic explains how you can create and use the custom Active Roles controls your client sends to Administration Service when performing custom commands. Administration Service can process that control providing an additional way for you to customize the menu commands. The custom Active Roles controls can be defined using the child element Setting for the element Task associated with your custom command. You can create any number of controls for a custom command.

To create Active Roles controls for a custom command

  1. In the attribute edsaWICommands, for the element Task associated with your custom command, create the child element Setting with the following values of its attributes:
    Name="Controls"
    Value = <A comma separated list of the pairs: <control name>, <control value>>
     

  2. For changes to take effect, reload the Web Interface.

Example

To clarify the above procedure, consider the following example illustrating how to add 2 custom controls to the custom command GroupManager identified with ResID="WIS_COMMAND_GROUPMANAGER"on the Group menu. The following code snippet described in The Commands Settings topic is used:

<?xml version="1.0"?>
<Tasks CustomizationVersion="33" xmlns="http://quest.com/ActiveRolesServer/WebInterface">
  <Task ID="GroupProperties" ResID="WIS_COMMAND_GENPROPERTIES" ToolTipResID="WIS_COMMAND_GENPROPERTIES_DES" HelperResID="WIS_COMMAND_GENPROPERTIES_HELP"
    TaskType="1" FormID="GroupProperties" DescriptionResID="WIS_COMMAND_GENPROPERTIES_GROUP_DESCRIPTION"/>
  <Task ID="GroupManager" ResID="WIS_COMMAND_GROUPMANAGER" ToolTipResID="WIS_COMMAND__GROUPMANAGER_TOOLTIP" HelperResID="WIS_COMMAND_GROUPMANAGER_HELP"
    TaskType="0" Action="my_groupManager.asp" DescriptionResID="WIS_COMMAND_GROUPMANAGER_DESCRIPTION"> 
    <Setting Name="Controls" Value="MYCONTROL1,controlvalue1,MYCONTROL2,controlvalue2"/>
    <IsCommandAllowedFilters><VBSFilter MethodName="IsAvailableAttribute"/></IsCommandAllowedFilters></Task>
  <Task ID="Separator" ResID="WIS_SEPARATOR_TITLE" PlainTextLabel="MySeparator" TaskType="5"/>
</Tasks>

Clicking the GroupManager command sends two controls MYCONTROL1 and MYCONTROL2 to the Administration Service. The control MYCONTROL1 is set to the value "controlvalue1" while the control MYCONTROL2 is set to the value "controlvalue2".

Important:

The XML schema describing the element Task requires its child elements to appear in the following order within the containing element Task:
<xsd:sequence>
<xsd:element ref="Setting" minOccurs="0" maxOccurs="unbounded" />
<xsd:element ref="IsCommandAllowedFilters" minOccurs="0" maxOccurs="unbounded" />
<xsd:element ref="RunTimeVariables" minOccurs="0" maxOccurs="unbounded" />
</xsd:sequence>

For example, the attribute edsaWICommands must contain the element Setting before the element IsCommandAllowedFilters; otherwise, the Web Interface will fail returning an error message.
For more information, refer to "XML Schema Elements" in the MSDN Library.
