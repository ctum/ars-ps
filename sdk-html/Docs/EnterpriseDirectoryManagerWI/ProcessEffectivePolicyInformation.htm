Generation of Custom Entry Value

This topic explains how you can use the RegisterAttributeControl method of the FormPage object to have the web client generate a value of a custom entry affected by the property generation administrative policy.

To clarify the use of the RegisterAttributeControl method, consider the example from the Create a Custom Entry topic that illustrates how to create the Employee ID text box and the Job description text area on the Organization tab of the User Properties form.

Let us apply the following property generation policy rule to the extensionAttribute2 property, associated with the Job description text area:  the job description for each user must be equal to him/her job title. To do this, prepare the appropriate Policy Object and then link it to an Organizational Unit where the users reside.

To prepare the Policy Object using the MMC console

  1. In the console tree, under the Active Roles snap-in item, expand the Configuration node.

  2. Under Configuration, expand the Policies node, right-click Administration, point to New, and then click Provisioning Policy Object to start the New Provisioning Policy Object wizard.

  3. On the Name and Description page, under Name and Description, specify the name and description of the Policy Object you want to create, and then click Next.

  4. On the Policy to Configure page, select Property Generation and Validation, and then click Next.

  5. On the Controlled Property page, click Select.
    The     Select Object Type and Property dialog box opens.

  6. In the Select Object Type and Property dialog box, do the following:

    • From the Object type list, select the User object.
    • Select the Show all possible properties check box.
    • From the Object property list, select the Custom Attribute 2 property (the extensionAttribute2 lDAPDisplayName), click OK, and then click Next. 

  1. On the Configure Policy Rule page, select the 'Custom Attribute 2' must be <value> (generates default value) check box, and then under Edit policy rule, click the <click to add value> link.
    The     Add Value dialog box opens.

  2. In the Add Value dialog box, click Configure.
    The     Configure Value dialog box opens.

  3. In the  Configure Value dialog box, click Add.
    The     Add Entry dialog box opens.

  4. In the Add Entry dialog box, under Entry type, select User Property, and then click the Select button next to Property.

  5. In the Select Object Property dialog, select the Show all possible properties check box, select the Job Title property (the title lDAPDisplayName), and then click OK.

  6. Click OK in the Configure Value and Add Value dialog boxes.

  7. Follow the instructions provided by the New Provisioning Policy Object wizard to link the Policy Object to the appropriate Organizational Unit and to complete the wizard.

To create custom entries

  1. Perform Steps 1 to 3 of the procedure in the Example section of the Create a Custom Entry topic.

  2. In Step 4, add the objFormPage.RegisterAttributeControl "cst_jobDescriptionIDControl", "extensionAttribute2" line to Sub Get_cst_JobDescription in the following way:

    Sub Get_cst_JobDescription(ByRef objFormContext, ByRef objFormPage)
    Dim strHTML, strJob
    strJob = CStr(objFormContext.GetValue("extensionAttribute2"))
    strHTML = "<textarea id='cst_jobDescriptionIDControl' name='cst_jobDescriptionIDControl' rows=3 cols=80>" & strJob & "</textarea>"
    Call objFormPage.Write(strHtml)
    objFormPage.RegisterAttributeControl "cst_jobDescriptionIDControl", "extensionAttribute2"
    End Sub

    Do not change other functions described in Step 4.

  3. For changes to take effect, reload the Web Interface (WI) site.

After you perform the procedures above, try to enter any information in the Job title text box. The web client generates the same value in the Job description text area (see the figure in Creating a Custom Entry).
