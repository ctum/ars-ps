FormContext Object

The FormContextobject provides an access to a directory object with which the current form is associated. The FormContextobject is available within the Get_XXX and Set_XXX functions used to create the custom entries. For more information about these functions, refer to Creating Custom Entries.

Properties and Methods

The FormContextobject supports the following properties and methods:

  • DN Property

  • IsCreateNewObjectForm Property

  • IsModifyObjectPropertiesForm Property

  • DirectoryObject Property

  • IsAttributeChangeAllowed Method

  • GetCurrentEntryXmlString Method

  • GetValue Method

  • SetValue Method

  • GetValues Method

  • SetValues Method

  • Object Property (deprecated)

DN Property

Read-only. Directory string. Returns the distinguished name of a directory object in whose context the current form is used.

Obj.DN
where Obj refers to a FormContext- type object.

IsCreateNewObjectForm Property

Read-only. Boolean. When TRUE, the custom entry is associated with a form used to create a new object.

Obj.IsCreateNewObjectForm
where Obj refers to a FormContext- type object.

IsModifyObjectPropertiesForm Property

Read-only. Boolean. When TRUE, the custom entry is associated with a form used to modify an existing object.

Obj.IsModifyObjectPropertiesForm
where Obj refers to a FormContext- type object.

DirectoryObject Property

Read-only. Returns the reference to the directory object in whose context the current form is used. The returned object implements the IADs interface and other interfaces that depend on the directory object class. The complete list of those interfaces as well as properties and methods the interfaces provide is in the section ADSI Provider Reference.

Obj.DirectoryObject
where Obj refers to a FormContext- type object.

IsAttributeChangeAllowed Method

The Function procedure that returns a Boolean value indicating whether you can change value of the specified attribute of the directory object in whose context the current form is used.

Obj.IsAttributeChangeAllowed(ByVal strAttr)
where Obj refers to a FormContext- type object.

Arguments

strAttr
[in] Required. String. Contains the ldapDisplayName of the object attribute.

Back to top

GetCurrentEntryXmlString Method

The Function procedure that returns an XML string containing the information on the current entry.

Obj.GetCurrentEntryXmlString()
where Obj refers to a FormContext- type object.

Arguments

This method requires no arguments.

GetValue Method

The Function procedure that returns the value of the specified attribute of the directory object in whose context the current form is used. This method is equivalent to using the Get method provided by the IADs interface.

Obj.GetValue(ByVal strAttr)

Arguments

strAttr
[in] Required. String. Contains the ldapDisplayName of the object attribute.

Back to top

SetValue Method

The Sub procedure that sets a new value to the specified attribute of the directory object in whose context the current form is used. This method is equivalent to using the Put method provided by the IADs interface.

Obj.SetValue(ByVal strAttr, ByVal value)

Arguments

strAttr
[in] Required. String. Contains the ldapDisplayName of the object attribute.
value
[in] Required. Contains a value to be assigned to the strAttr attribute.

Back to top

GetValues Method

The Function procedure that returns the value(s) of the specified attribute of the directory object in whose context the current form is used. The returned property values can be single-valued or multi-valued. Unlike the GetValue method, the property values are returned as a variant array of VARIANT, or a variant array of bytes for binary data. A single-valued property is then represented as an array of a single element.

This method is equivalent to using the GetEx method provided by the IADs interface.

Obj.GetValues(ByVal strAttr)

Arguments

strAttr
[in] Required. String. Contains the ldapDisplayName of the object attribute.

Back to top

SetValues Method

The Sub procedure that modifies the values of the specified attribute of the directory object in whose context the current form is used. For example, for attributes that allow multiple values, you can append additional values to an existing set of values, modify the values in the set, remove specified values from the set, or delete values from the set. This method is equivalent to using the PutEx method provided by the IADs interface.

Obj.SetValues(ByVal intCode, ByVal strAttr, ByVal arr)

Arguments

intCode
[in] Required. Long integer. Control code that indicates the mode of modification: Append, Replace, Remove, and Delete. For more information and a list of values, see ADS_PROPERTY_OPERATION_ENUM in the MSDN library.
strAttr
[in] Required. String. Contains the ldapDisplayName of the object attribute.
arr
[in] Required. Contains a VARIANT array that contains the new value or values of the property. A single-valued property is represented as an array with a single element. If intCode is set to ADS_PROPERTY_CLEAR, the value of the property specified by this argument is irrelevant.

Object Property (deprecated)

Read-only. Returns the reference to the directory object in whose context the current form is used. This property is equivalent to using the DirectoryObject property.

Obj.Object
where Obj refers to a FormContext- type object.  

Remarks

The Object property is considered obsolete. While this property is still supported in the current version of Active Roles, it may be removed in the future. When writing new scripts, you should avoid using this property. When modifying existing scripts, you are strongly encouraged to remove any dependency on this property. Use the DirectoryObject property instead.

Example Code (VBScript)

This code snippet illustrates how to get the description of the current entry as an XML string.

Function Get_EntryXml(formContext, FormPage)
  Dim oXml, entryId
  Set oXml = CreateObject("MSXML2.DomDocument")
  oXml.loadXml(formContext.GetCurrentEntryXmlString())
   ' Retrieve the entry ID
  entryId = oXml.documentElement.getAttribute("ID")
   '
   ' your code goes here
   '
   '
End Function
