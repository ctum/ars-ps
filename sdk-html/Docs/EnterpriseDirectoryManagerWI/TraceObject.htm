Trace Object

The Traceobject allows to record any information such as debug messages to the WI log file. By default, the log file has the following name:
<WI folder>/Log/<WI site name>/<Current Application name>/<Current Application name>.log
where
- <WI folder> refers to the folder where the Web Interface is installed;
- <WI site name> refers to the current WI site name;
- <Current Application name> refers to the application name.

The Traceobject is available within the functions Get_XXX and Set_XXX used to create the custom entries. For more information about these functions, refer to Creating Custom Entries.

Methods

The Traceobject supports the following methods:

  • Write Method

Write Method

The Sub procedure that adds an entry to the WI log file.

Trace.Write(ByVal strMsg)

Arguments

strMsg
[in] Required. String. Specifies a message to add to the WI log file.
 

Remarks

By default, the Write method is disabled and the use of this method has no effect. You can enable or disable this method using the following procedures.
 
To enable the Write method

  1. Open the web.nlog file with any text editor, such as Notepad.
    This file is stored into the <WI installation folder>\Public container.

  2. In this file, replace the globalThreshold="Fatal" string with globalThreshold="Debug".

  3. Save changes and then reload the Web Interface.

To disable the Write method

  1. Open the web.nlog file with any text editor, such as Notepad.
    This file is stored into the <WI installation folder>\Public container.

  2. In this file, replace the globalThreshold="Debug" string with globalThreshold="Fatal".

  3. Save changes and then reload the Web Interface.

Example Code (VBScript)

The following code snippet allows you to add the "Control point 2" line to the WI log file:

Trace.Write("Control point 2")
