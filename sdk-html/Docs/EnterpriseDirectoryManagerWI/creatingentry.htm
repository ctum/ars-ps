Creating an Auto Entry

This topic explains how you can create an auto entry. An entry is a group of form elements such as text boxes, check boxes, option buttons, and command buttons. Each entry is used to view and modify a value of a certain object property. An auto entry depends on the syntax of the directory object's property viewed or modified with that entry.

To create the auto entry, use the following procedure to make the appropriate changes to the attributes edsaWIEntries and edsaWIForms of the object WorkingCopy residing in the Customization Settings container.

To make changes to the attribute edsaWIForms

  1. Save the value of the edsaWIForms attribute into a text file.

  2. Open this file with any XML editor, such as XML Spy, and find the FormTab element  that defines a tab you want to contain the entry to be created.

  3. For the FormTab element, insert the child element FormEntry in the following way:

    <FormEntry ID=strID>
    Where:
    strID     - Unique identifier of the entry. The entry must be also defined in the attribute edsaWIEntries.

  4. Save changes to the attribute edsaWIForms.

To make changes to the attribute edsaWIEntries

  1. Save the value of the edsaWIEntries attribute into a text file.

  2. Open this file with any XML editor, such as XML Spy, and insert the FormEntry element in the following way:

    <FormEntry ID=strID ResID=strResID Properties=strLdap SingleValue=bSingle ReadOnly=bRead EntryType=intType >
    </FormEntry>
    Where:
    strID     - Unique identifier of the entry. The edsaWIForms attribute must contain the FormEntry element with the same ID.
    strResID     - Identifier of the string constant containing the caption of the entry when it is displayed on the form, such as Description on the User Properties form.
    strLdap     - A comma-separated list of the object property's ldapDisplayNames used by the entry.
    bSingle     - A boolean attribute that specifies whether an object's multi-valued property associated with the entry will be processed as a single-valued property.
    bRead     - A boolean attribute that specifies whether an object's multi-valued property associated with the entry will be processed as a single-valued property.
    intType     - The entry type.
    For more information about attributes of the element FormEntry, refer to The Entries Settings.

  3. Save changes to the attribute edsaWIEntries. 

Note:

 

For changes to take effect, reload the Web Interface.

Example

The following example illustrates how to create an auto entry and place it on the General tab in the Computer Properties dialog box.

This following snippet must be added to the edsaWIForms attribute. 

    <Form ID="ComputerProperties" FormType="1" ResID="WIS_FORM_COMPUTER_PROPERTIES" TargetClass="computer" DescriptionResID="">
        <FormTab ID="general" ResID="WIS_UP_GENERAL">
            <FormEntry ID="sAMAccountNameComp"/>
            <FormEntry ID="dNSHostName"/>
            <FormEntry ID="role"/>
            <FormEntry ID="description"/>
            <FormEntry ID="trustForDelegation"/>
           ' Add a custom entry
            <FormEntry ID="MyCustomEntry"/>
        </FormTab>
    </Form>

This following snippet must be added to the edsaWIEntries attribute. 

 <FormEntry ID="MyCustomEntry" ResID="CST_EXCHANGE_PRESENT" Properties="edsaIsMsExchangePresent" SingleValue="True" ReadOnly="True"
EntryType="1" DontShowCaption="False" >
</FormEntry>
  

Note:

The constants WIS_FORM_COMPUTER_PROPERTIES, WIS_UP_GENERAL, and CST_EXCHANGE_PRESENT must be defined in the language settings stored in edsaWIStrings.
