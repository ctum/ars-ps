LanguageUtils Object

The LanguageUtilsobject provides methods used to access the WI language settings. The LanguageUtilsobject is available within the Get_XXX and Set_XXX functions used to create the custom entries. For more information about these functions, refer to Creating Custom Entries.

Methods

The LanguageUtilsobject supports the following methods:

  • LoadString Method

LoadString Method

The Function procedure that returns the resource string identified by its resource ID.

LanguageUtils.LoadString(ByVal resID)

Arguments

resID
[in] Required. Specifies the ID of the resource string to load.
 

Remarks

The LoadString method searches the WI language settings for the requested resource string. The built-in language settings are stored in language files (.lng) located in the locale specific containers, while your custom language settings are stored into the edsaWIString attribute of the WorkingCopy object. For detailed information, see The Language Settings.

The LoadString method first searches your custom language settings and the language files specific to the WI language (e.g. US-English, French, etc.). If the requested resource string is not found, the LoadString method tries to search the language files for US-English.

If the LoadString method fails to find the requested resource string, it returns the empty string.

Example Code (VBScript)

This code snippet illustrates how to load the resource string with the Res ID="WIS_POLICY_RULE".

strResource = LanguageUtils.LoadString("WIS_POLICY_RULE")
