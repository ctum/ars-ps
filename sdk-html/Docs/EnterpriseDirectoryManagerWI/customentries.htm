Creating a Custom Entry

This topic explains how you can create your own custom entries.

If you want a form to include custom entries, you should develop the custom functions designed to create custom entries. This topic provides an example that illustrates how to develop custom functions that add a text box and a text area to a form.

To create a custom entry, you should develop a couple of the functions
[Get_<Entry ID>, Set_<Entry ID>] or [ Get_<FunctionAction's value>, Set_<FunctionAction's value>]
and put them into the file:
'<WI folder>\Public\CustomCode\Entries.vbs'.

The function's name depends on whether the attribute FunctionAction of the FormEntry element is set.
The <WI folder> refers to the folder where Web Interface is installed. This folder is determined at the Web Interface installation time.

Get_XXX Functions

These functions allow you to add a custom HTML code that creates custom entries.

The function Get_<Entry ID> or Get_<FunctionAction's value> (Sub procedure, as applied to VBScript) has the following syntax:

Sub Get_<Entry ID>(ByRef objFormContext, ByRef objFormPage)
Sub Get_<FunctionAction's value>(ByRef objFormContext, ByRef objFormPage)

Arguments

objFormContext

[in] Required. Refers to the FormContext object that represents the current context.
This object provides the DirectoryObject property that returns the reference to the directory object with which the current form is associated.

objFormPage

[in] Required. Refers to the FormPage object representing the current form that includes the custom entry.

Set_XXX Functions

These functions allow you to retrieve and process values passed to the server with custom entries.

The function Set_<Entry ID> or Set_<FunctionAction's value> (Sub procedure, as applied to VBScript) has the following syntax:

Sub Set_<Entry ID>(ByRef objFormContext)
Sub Set_<FunctionAction's value>(ByRef objFormContext)

Arguments

objFormContext
[in] Required. Refers to the FormContext object that represents the current context.
This object provides the DirectoryObject property that returns the reference to the directory object with which the current form is associated.

To process values passed to the server with custom entries, use the ASP built-in object Request. This object retrieves the values that the client browser passed to the server during an HTTP request. For information about the properties and methods of the Request object, refer to the MSDN library.

Important:

Don't confuse the ASP built-in object Request with the Active Roles intrinsic object Request.

Example

To clarify the use of the custom functions, consider the following example that illustrates how to create the Employee ID text box and the Job description text area on the Organization tab of the User Properties form. The custom entries allow you to view and modify the following properties of the selected user:

  • employeeID: Contains the user's employeeID

  • extensionAttribute2: Contains the description of the job that the user performs

Note:

In this example, the information about the user's job is stored into the extensionAttribute2 attribute. This attribute is added to the Active Directory schema only if you have installed Microsoft Exchange. Otherwise, use any other available attribute instead.

To use this example

  1. In the attribute edsaWIForms, define the following child elements FormEntry for the
    <FormTab ID="organization"> that is a child element of the <Form ID="UserProperties"...>:
     

    <FormEntry ID="cst_employeeID"/>
    <FormEntry ID="cst_JobDescription"/>

    For information about XML elements used in this file, see The Forms Settings.

  2. Append the following entries to entries defined in the first value of edsaWIEntries:
     

    <FormEntry ID="cst_employeeID" Properties="employeeID" EntryType="0" 
    FunctionAction="cst_EmployeeID"
    ReadOnly="False" SingleValue="True" ResID="CST_ENTRY_EMPLOYEE_ID"
    DescriptionResID="CST_ENTRY_EMPLOYEE_ID_DES"
    />
    <FormEntry ID="cst_JobDescription" Properties="extensionAttribute2" EntryType="0"
    FunctionAction="cst_JobDescription" ReadOnly="False" SingleValue="False"
    ResID="CST_ENTRY_JOBDESCRIPTION"
    DescriptionResID="CST_ENTRY_JOBDESCRIPTION_DES"
    />

  3. Append the following entries to entries defined in the first value of edsaWIStrings:
     

    <Res ID="CST_ENTRY_EMPLOYEE_ID" Value="Employee ID"/>
    <Res ID="CST_ENTRY_EMPLOYEE_ID_DES" Value="Allows you to view or modify the user employeeID"/>
    <Res ID="CST_ENTRY_JOBDESCRIPTION" Value="Job description"/>
    <Res ID="CST_ENTRY_JOBDESCRIPTION_DES" Value="Describes the job the user performs at your organization"/>

  4. Add the following Sub procedures to the file '<WI folder>\Public\CustomCode\Entries.vbs':
     

    5. ' Create the EmployeeID text box
    Sub Get_cst_EmployeeID(ByRef objFormContext, ByRef objFormPage)
    Dim strHTML, strEmployeeID
    strEmployeeID = CStr(objFormContext.GetValue("employeeID"))
    strHTML = "<input type='text' id='cst_employeeIDControl' name='cst_employeeIDControl' value='" & strEmployeeID & "'>"
    Call objFormPage.Write(strHtml)

    End Sub

    ' Save value of the employeeID attribute
    Sub Set_cst_EmployeeID(ByRef objFormContext)
    Dim strEmployeeID
    ' Retrieve the value passed with the EmployeeID text box
    strEmployeeID = Request("cst_employeeIDControl")
    objFormContext.SetValue "employeeID", strEmployeeID
    End Sub

    ' Create the Job description text area
    Sub Get_cst_JobDescription(ByRef objFormContext, ByRef objFormPage)
    Dim strHTML, strJob
    strJob = CStr(objFormContext.GetValue("extensionAttribute2"))
    strHTML = "<textarea id='cst_jobDescriptionIDControl' name='cst_jobDescriptionIDControl' rows=3 cols=80>" & strJob & "</textarea>"
    Call objFormPage.Write(strHtml)
    End Sub

    ' Save the user's job description into the extensionAttribute2 attribute
    Sub Set_cst_JobDescription(ByRef objFormContext)
    Dim strJob
    ' Retrieve the value passed with the Job description text area
    strJob = Request("cst_jobDescriptionIDControl")
    objFormContext.SetValue "extensionAttribute2", strJob
    End Sub

  5. For changes to take effect after customizing, reload the Web Interface (WI) site. To reload the WI site, use the special designed custom script.

For information about how to make changes to the attributes edsaWIEntries, edsaWIForms, and edsaWIStrings, see Specifying Customization Settings.

 

Remarks

We recommend that you don't call the GetInfoEx method of the IADs interface for the object properties listed in the Properties attribute of the FormEntry element.

We strongly recommend that you don't call the SetInfo method of the IADs interface. When you don't call this method, the server checks whether values of the directory object properties are in compliance with administrative policy.
