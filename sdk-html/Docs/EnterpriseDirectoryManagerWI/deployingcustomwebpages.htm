Developing custom ASP .NET page

This topic explains how you can deploy your custom .NET web pages to the Web Interface.

You can develop your custom .aspx page and deploy it to the Web Interface. The page can be developed using any from the following languages: C#, VB.NET, JScript.NET. Before deploying your custom page, it must be compiled using the ASP.NET compilation tool aspnet_compiler.exe.

To deploy a custom page

  1. Create your custom page and save it to any folder on the computer that runs the Web Interface. For example, create the page Hello.aspx (see Example) and save it to the folder C:\MyWebPages. Assume the Web Interface is deployed in the virtual directory ARSWI of the default web site.

  2. Click Start, point to Programs -> Microsoft .Net Framework SDK, and then click SDK Command Prompt.

  3. In the SDK Command Prompt window, type the following syntaxes at the command prompt:

    • cd C:\MyWebPages
    • aspnet_compiler -p . -v /ARSWI compiled_page

      • Note: The compiled page will be saved in the folder compiled_page residing within the folder C:\MyWebPages.

  4. Copy all files and folders from the folder C:\MyWebPages\compiled_page to the Public subfolder of the Web Interface installation folder.

To view a custom page

  • Using a web browser, open the page http://<WI computer>/<VirtualDir>/<CustomPage>
    where <WI computer> refers to the network name of the computer running the Web Interface; <VirtualDir> refers to the virtual directory of the default web site where the Web Interface is deployed; <CustomPage> refers to your custom page name compiled with the ASP.NET compilation tool aspnet_compiler.exe.

Example

The following is a sample custom page Hello.aspx.

<%@ Page Language="C#" %>
<html>
  <head>
    <title>It's compiled aspx page</title>
  </head>
  <body>
    <h4>Hello World!!!</h4>
  </body>
</html>

To view this custom page, perform the steps from the procedure "To deploy a custom page", and then open the page http://localhost/ARSWI/Hello.aspx with a web browser.

Note

To open your custom ASP .NET page with the Command Menu related to an object you are administering with the Web Interface, use the appropriate procedures available in Developing Custom ASP Web Page.
