Storing Customization Settings

The Active Roles Administration Service stores the Web Interface customization information in the Active Roles configuration database as objects of the class edsWICustomizationSettings. Those objects reside within the Web Interface configuration containers (the object class is edsWICustomizationSettingsContainer). The configuration containers DN: 

CN=Customization Settings,CN=<Container's ID>,CN=Web Interface,CN=Application Configuration,cn=Configuration
CN=Interface Settings,CN=<Container's ID>,CN=Web Interface,CN=Application Configuration,cn=Configuration

The <Container's ID> identifies the Web interface configuration. The number of the WI configuration containers is equal to the number of configurations created for the Web Interface.

Customization Settings Container

The Customization Settings container that resides into each WI configuration container host three objects of the edsWICustomizationSettings class described in the following table.

Object name

Object DN

Description

CurrentCopy

CN=CurrentCopy,CN=Customization Settings,CN=<Container's ID>,CN=Web Interface,CN=Application Configuration,cn=Configuration

Contains the customization settings currently used by the Web Interface.

Default

CN=Default,CN=Customization Settings,CN=<Container's ID>,CN=Web Interface,CN=Application Configuration,cn=Configuration

Contains the default customization settings specified at the Web Interface installation time using the configuration templates. Each configuration template provides an individual set of commands installed by default.

The object Default is copied to CurrentCopy after you click Restore Default on the Customization menu of the Web Interface.

WorkingCopy

CN=WorkingCopy,CN=Customization Settings,CN=<Container's ID>,CN=Web Interface,CN=Application Configuration,cn=Configuration

Contains the user-specified customization settings. The object WorkingCopy is copied  to CurrentCopy after you reload the Web Interface.

Note:

We don't recommend that you modify the CurrentCopy and Default objects that reside in the Customization Settings container. Save your customization settings to the WorkingCopy object (for more information, see Specifying Customization Settings), and then reload the Web Interface for the changes to take effect.

 The user customization settings are stored in the following attributes of the WorkingCopy object:

  • edsaWIMenus: describes menus associated with directory objects managed with the Web Interface.

  • edsaWICommands: describes commands used in the menus defined in the property edsaWIMenus.

  • edsaWIForms: describes forms associated with the Form-type commands defined in the property edsaWICommands.

  • edsaWIEntries: describes entries used in the forms defined in the property edsaWIForms.

  • edsaWIStrings: contains names and values of the string constants used in the properties above.

Note:

All those attributes are multivalued. They conform to the syntax CaseExactString. The attribute values are stored in the XML format.

Important:

The Web Interface only uses the first value of multivalued attributes. Other values are ignored and cannot influence the Web Interface configuration.

Interface Settings Container

Some additional settings for the user interface components are stored in the Interface Settings container:

Object name

Object DN

Description

CurrentCopy

CN=CurrentCopy,CN=Interface Settings,CN=<Container's ID>,CN=Web Interface,CN=Application Configuration,cn=Configuration

Contains settings for the user interface components currently used by the Web Interface.

Default

CN=Default,CN=Interface Settings,CN=<Container's ID>,CN=Web Interface,CN=Application Configuration,cn=Configuration

Contains the default customization settings specified at the Web Interface installation time using the configuration templates. 

The object Default is copied to CurrentCopy after you click Restore Default on the Customization menu of the Web Interface.

WorkingCopy

CN=WorkingCopy,CN=Interface Settings,CN=<Container's ID>,CN=Web Interface,CN=Application Configuration,cn=Configuration

Contains the user-specified customization settings. The object WorkingCopy is copied  to CurrentCopy after you reload the Web Interface.

Note:

We don't recommend that you modify the CurrentCopy and Default objects that reside in the Interface Settings container. Save your customization settings to the WorkingCopy object (for more information, see Specifying Customization Settings), and then reload the Web Interface for the changes to take effect.

Important:

The Web Interface only uses the first value of multivalued attributes. Other values are ignored and cannot influence the Web Interface configuration.

The attribute edsaWISettings of the WorkingCopy object contains settings that allow you to customize the user interface components. For more information, see "The Interface Settings" in Related Topics.
