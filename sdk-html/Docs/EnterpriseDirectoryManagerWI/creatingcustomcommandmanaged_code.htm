Creating a Custom Command Using Managed Code

This topic explains how you can create a custom command that uses a managed code. The following managed languages are supported: C#, JScript...

The custom command of this type displays a user interface that renders the state of a directory object to which the command applies. The user interface allows you to modify the directory object properties using the managed code associated with that command.

 

To create a command that uses a managed code

  1. Create a class that implements the interface ICustomCommand.

  2. Using Microsoft Visual Studio, create an assembly that uses that class.

  3. Save the assembly (such as a .dll file) to the folder <WI Installation Folder>/bin/>

  4. Define your command in the attribute edsaWICommands.

  5. Make the appropriate changes to the attributes edsaWIMenus and edsaWIStrings of the WorkingCopy object.

  6. For changes to take effect, restart IIS Admin Service.

Example

To clarify the procedure above, consider the following example.

Sample Managed Code

The file ...\Samples\WI\managedcode.cs contains an example of the managed C# code that implements the interface ICustomCommand. The methods provided by this interface allow you to define where to display the UI associated with the command. This code displays the command UI in the console right pane. Note that the interface ICustomCommand is defined in the library Web.Interfaces. The sample code also uses the libraries Interop.ActiveDS, System.Drawing, and System.Web.

To create an assembly using the sample managed code

  1. On the computer where the Web Interface is installed, run Microsoft Visual Studio.

  2. On the File menu, point to New, and then click Project.  This opens the New Project dialog box.

  3. In the New Project dialog box, do the following:

    • Under Project types, select Other Languages | Visual C# | Windows

    • Under Templates, select Class Library

    • In the Name text box, type in CustomDevProject

    • When finished, click OK.

  4. On the View menu, click Solution Explorer.

  5. In the Solution Explorer view, right-click your solution (CustomDevProject), on the shortcut menu, point to Add, and then click Existing Item.

  6. Add the delivered source managedcode.cs using the Add Existing Items - Solution Item dialog box.

  7. Build the project, and then save the created assembly (CustomDevProject.dll file) to the folder <WI installation folder>\bin, where <WI installation folder> refers to the folder where the Web Interface is installed.

Note

For more information about how to create Visual Studio projects, refer to the Microsoft Visual Studio documentation.

Changes to the WorkingCopy Object

The attribute edsaWICommands must contain the definition of your command. The appropriate changes must be also made to the attributes edsaWIMenus, edsaWIStrings. The following procedures help you make these changes.

 

To make changes to the edsaWICommands attribute

  1. Save the value of the edsaWICommands attribute  into a text file.

  2. Open this file with any XML editor, such as XML Spy, and insert the following element Task:

    <Task ID="SimpleManagedCustomCommand"
    ResID="WIS_SIMPLE_MANAGED_CUSTOM_COMMAND"
    DescriptionResID="WIS_SIMPLE_MANAGED_CUSTOM_COMMAND"
    ToolTipResID="WIS_SIMPLE_MANAGED_CUSTOM_COMMAND"
    HelperResID="WIS_SIMPLE_MANAGED_CUSTOM_COMMAND"
    TaskType="0"
    Action="CustomCommands.aspx"
    ClassNames="user"
    FullQualifiedTypeName="CustomDevelopment.SimpleCustomCommand,CustomDevProject">
    <Setting Name="Setting_1" Value="SettingValue_1"/>
    <Setting Name="Setting_2" Value="SettingValue_2"/>
    <Setting Name="Setting_3" Value="SettingValue_3"/>
    <Setting Name="Setting_4" Value="SettingValue_4"/>
    <RuntimeVariables>
    <Variable Name="LCID" QueryParamName="LCID" Scope="Application" />
    <Variable Name="TaskId" QueryParamName="TaskId" Scope="Application" />
    <Variable Name="IsDSAdmin" QueryParamName="IsDSAdmin" Scope="Application" />
    <Variable Name="BindInfo" QueryParamName="BindInfo" Scope="Application" />
    <Variable Name="OperationalDC" QueryParamName="OperationalDC" Scope="Application" />
    <Variable Name="userPrincipalName" QueryParamName="(SO)userPrincipalName" Scope="SecurableObject" />
    <Variable Name="distinguishedName" QueryParamName="(T)distinguishedName" Scope="Trustee" />
    </RuntimeVariables>
    </Task>

  3. Save changes to the edsaWICommands attribute.

Note that the attribute FullQualifiedTypeName of the element Task refers to the class that implements the custom command. The value of this attribute conforms the following syntax:
TopNamespace.SubNameSpace.ContainingClass+NestedClass, MyAssembly

where TopNamespace,  Subnamespace, etc. - refer to the namespaces defined in your project. For more information, refer to the MSDN Library.

To make changes to the attribute edsaWIMenus

  1. Save the value of the attribute edsaWIMenus into a text file.

  2. Open this file with any XML editor, such as XML Spy, and under the appropriate element TaskList insert the element Task that describes your command:

    <TaskList TargetClass="user" DefaultTask="UserProperties">
    <Task ID="DisableAccount" />
    …
    <Task ID="SimpleManagedCustomCommand" />
    </TaskList>

     

    This adds your command identified with ID="SimpleManagedCustomCommand" to the bottom of the User menu.

  3. Save changes to the attribute edsaWIMenus.

To make changes to the attribute edsaWIStrings

  1. Save the value of the attribute edsaWIStrings into a text file.

  2. Open this file with any XML editor, such as XML Spy, and insert the description of the resource WIS_SIMPLE_MANAGED_CUSTOM_COMMAND used in the attribute edsaWICommands:
    <Res ID="WIS_SIMPLE_MANAGED_CUSTOM_COMMAND" Value="Simple Managed Custom Command" />

  3. Save changes to the attribute edsaWIStrings.
