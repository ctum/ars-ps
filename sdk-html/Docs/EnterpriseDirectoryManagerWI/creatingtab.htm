Creating Tab

This topic explains how you can create a tab. A form includes one or several tabs that group related elements on the same page. Each set of related elements is presented within the tab labeled with the name of the set, such as General, Managed by, and Object on the Properties form. The form tabs are defined using the XML element FormTab.

To create a tab, use the following procedure to make the appropriate changes to the attribute edsaWIForms of the object WorkingCopy residing in the Customization Settings container.

To create a custom tab

  1. Save the value of the attribute edsaWIForms into a text file.

  2. Open this file with any XML editor, such as XML Spy, and find the element Form that defines the form you want to include the tab to be created.

  3. For the element Form, insert the child element Formtab in the following way:

    <FormTab ID=TabID ResID=strResID ToolTipResID=strToolTip>
    Where:
    TabID     - The tab identifier (any string);
    strResID     - Identifier of the string constant containing the tab name (optional attribute);
    strToolTip     - Identifier of the string constant containing the tab tooltip (optional attribute);
    For more information about attributes of the element FormTab, refer to The Form Settings.

  4. Optionally, add entries to the tab. When finished, insert the tag </FormTab> and save the file.

  5. Save changes to the attribute edsaWISettings.

  6. Reload the Web Interface.

Example

The following code snippet illustrates how to add the General tab to the Computer Properties form. The tab includes custom entries. 

<Forms CustomizationVersion="33">
    <Form ID="ComputerProperties" FormType="1" ResID="WIS_FORM_COMPUTER_PROPERTIES" TargetClass="computer" DescriptionResID="">
        <FormTab ID="general" ResID="WIS_UP_GENERAL">
            <FormEntry ID="sAMAccountNameComp"/>
            <FormEntry ID="dNSHostName"/>
            <FormEntry ID="role"/>
            <FormEntry ID="description"/>
            <FormEntry ID="trustForDelegation"/>
        </FormTab>
    </Form>
</Forms>

Note:

The constants WIS_FORM_COMPUTER_PROPERTIES and WIS_UP_GENERAL must be defined in the language settings stored in edsaWIStrings.
