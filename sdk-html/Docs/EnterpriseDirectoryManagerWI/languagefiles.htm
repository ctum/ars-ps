The Language Settings

This topic provides information about the language settings stored in the edsaWIStrings attribute. This attribute contains names and values of the string constants used in the Web Interface customization settings such as the menus settings, entries settings and forms settings.

Built-in Language Settings

The built-in language settings define language for displaying all menus, commands, and forms associated with the Web Interface, as well as tool tips and help. The built-in language settings are stored in language files (.lng) located in the locale specific containers, for example:

'<WI folder>\Public\LanguageFiles\1033\' - for US-English
'<WI folder>\Public\LanguageFiles\1031\' - for German
'<WI folder>\Public\LanguageFiles\1036\' - for French

The <WI folder> refers to the folder where the Web Interface is installed.

Important:

By default, the Web Interface setup only installs the US-English language files. To use other languages, install the Language Pack for Active Roles.

Note:

We don't recommend that you modify the built-in language files. Define your custom string constants in the edsaWIStrings property.

Custom Language Settings

After you add a new string constant to any customization setting, define that constant in the edsaWIStrings attribute.

Important:

After you modify edsaWIStrings, for changes to take effect, reload the Web Interface.

Note:

We strongly recommend that the names of your custom string constants, such as identifiers or function names, begin with the prefix cst_, for example cst_employeeID, cst_jobDescriptionIDControl, etc.

XML Elements

The edsaWIStrings attribute stores information in the Extensible Markup Language (XML) format.  XML elements you can use are listed below.

Resources

Describes the collection of string constants used in the Web Interface customization settings.

Parent Elements: None

Child Elements: Res

Attributes: None

 

Res

Describes a string constant.

Parent Elements: Resources

Child Elements: None

Attributes:

Name

Description

ID

Required. The string constant's identifier. The identifiers cannot begin with a number, and can only contain "A-Z", "a-z", "0-9", and the underscore character.

Value

Required. The string constant's value.

Example

The following snippet illustrates the use of the XML elements in the language settings.

<?xml version="1.0" encoding="utf-8"?>
<Resources>
    <Res ID="WIS_STAT_APPLY_FORM_SUC" Value="The operation was successfully completed."/>
    <Res ID="WIS_STAT_DELETE_SUC" Value="Object was successfully deleted."/>
</Resources>
