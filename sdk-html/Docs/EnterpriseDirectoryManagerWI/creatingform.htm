Creating Form

This topic explains how you can create a form associated with the Form-type commands defined in the attribute edsaWICommands. A form is a structured page with predefined areas for entering and changing information about directory objects.

A form consists of elements (entries) such as text boxes, check boxes, option buttons, etc. The form elements are defined in the attribute edsaWIEntries. A form can include one or several tabs that group related elements on the same page. Each set of related elements is presented within the tab labeled with the name of the set, such as General, Managed by, and Object on the Properties form. The form tabs are defined using the XML element FormTab.

To create a custom form, use the following procedure to make the appropriate changes to the attribute edsaWIForms of the object WorkingCopy residing in the Customization Settings container.

To create a custom form

  1. Save the value of the attribute edsaWIForms into a text file.

  2. Open this file with any XML editor, such as XML Spy, and find the element Forms.

  3. For the element Forms, insert the child element Form in the following way:

    <Form ID=FormID FormType=FType ResID=strConst TargetClass=TarClass DescriptionResID=strConst1>
    Where:
    FormID     - The form identifier (any string);
    FType     - The form type ("1", "2","3" or "4");
    strConst     - Identifier of the string constant containing the form name;
    TarClass     - The object class to which the form can be applied;
    strConst1     - Identifier of the string constant containing a description of the form.
    For more information about attributes of the element Form, refer to The Form Settings.

  4. Add tabs and entries to the form. When finished, save the file.

  5. Save changes to the attribute edsaWISettings.

  6.  Reload the Web Interface.

Example

The following code snippet illustrates how to create a custom form of the Object Properties type (FormType="1") for Computer objects. The form includes the General tab. 

<Forms CustomizationVersion="33">
    <Form ID="ComputerProperties" FormType="1" ResID="WIS_FORM_COMPUTER_PROPERTIES" TargetClass="computer" DescriptionResID="">
        <FormTab ID="general" ResID="WIS_UP_GENERAL">
            <FormEntry ID="sAMAccountNameComp"/>
            <FormEntry ID="dNSHostName"/>
            <FormEntry ID="role"/>
            <FormEntry ID="description"/>
            <FormEntry ID="trustForDelegation"/>
        </FormTab>
    </Form>
</Forms>

Note:

The constants WIS_FORM_COMPUTER_PROPERTIES and WIS_UP_GENERAL must be defined in the language settings stored in edsaWIStrings.
