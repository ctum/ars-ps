The Commands Settings

This section provides information about the format of settings for commands used in the menus defined in the attribute edsaWIMenus of the WorkingCopy object. This section also explains how you can create a page view command and a command using a managed code.

A command is an instruction that, when issued by a user, causes an action to be carried out. Commands are chosen from a menu. Examples of commands: New User on the Organizational Unit menu, Properties on the User menu, etc.

Settings for commands are stored in the attribute edsaWICommands of the WorkingCopy object. Modifying this attribute, you can set or modify parameters of commands, such as the command description, the command tooltip, etc. 

XML Elements

The attribute edsaWICommands stores information the Extensible Markup Language (XML) format.  XML elements you can use are listed below.

Important:

After you modify the attribute edsaWICommands, for changes to take effect, reload the Web Interface.  

Tasks

Describes the collection of commands.

Parent Elements: None

Child Elements: Task

Attributes:

Name

Description

CustomizationVersion

Required. Identifies the current customization version number. Must be the same for all attributes of the WorkingCopy object.

Important:

The value of this attribute depends on the application version. This value is stored into the attribute edsaWITemplateVersion of the WI configuration container objects (see Storing Customization Settings).

Task

Describes a command and its properties.

Parent Elements: Tasks

Child Elements: Setting, IsCommandAllowedFilters, RuntimeVariables

Attributes:

Name

Description

Action

Required, if TaskType=0 or TaskType=10. The URL of the .aspx or .asp file to be executed after you click the command. Save this file in the folder:
'<WI folder>\Public\CustomCode'
The <WI folder> refers to the folder where Web Interface is installed.

ClassNames

Optional. A comma-separated list of the ldapDisplayName properties of the object classes to which the command can be applied.

DescriptionResID

Optional. Identifier of the string constant containing a description of the command.

FormID

Required, if TaskType=1. Unique identifier of the form associated with the command. When this attribute is set, the attribute edsaWIForms must contain the element Form with the attribute ID equal to the FormID value.

FullQualifiedTypeName

Optional. Specifies the full qualified name of the customization type. For more information, and example of the use, refer to Creating a Custom Command Using a Managed Code.

HelperResID

Optional. Identifier of the string constant containing the command helper.

ID

Required. Unique identifier of the command. The identifier must be also defined in the property edsaWIMenus.

PlainTextLabel

Optional. Contains any string. This attribute allows you to label elements. The use of your custom labels allows your applications to easily update or remove custom elements from the configuration settings.

Properties

Optional. A comma-separated list of the object property names. The command can be applied only to objects that have those properties.

ResID

Required. Identifier of the string constant containing the name of the resource, such as tab or dialog box, where the command is used.

If TaskType=5, the string constant contains the title for the separator to be added to the menu. 

TaskType

Required. The type of the command. 

The possible values are:

  • 0  command Custom  -

  • 1  -  Form command

  • 5  -  Separator (adds a separator to the menu)

  • 9  -  Search command.

  • 10 - Page view command. For more information, see Creating Page View Command.

  • 11 - Attribute setter command. You can use this command type to assign a given value to a given attribute. The attribute to set and the value to assign are specified by the Setting element.

ToolTipResID

Optional. Identifier of the string constant containing the command tooltip.

UIPolicyID

Optional. Identifier of the UI policy connected with this command. When this attribute is set, the attribute edsaWISettings must contain the element UIPolicy with the attribute ID equal to the UIPolicyID value.

ConfirmRequired

Optional. Applies to the TaskType value of 11 (attribute setter command). When set to true, causes the command to display a confirmation message.

ConfirmMessageId

Optional. Applies to the TaskType value of 11 (attribute setter command). Specifies the ID of the Res element that holds the confirmation message to be displayed by the command.

Note:

Identifiers and values of the string constants are saved in the attributes edsaWIStrings. We strongly recommend that the names of your custom string constants, such as identifiers or function names, begin with the prefix cst_, for example cst_employeeID, cst_jobDescriptionIDControl, etc. 

RuntimeVariables

For a page view command (TaskType=10), describes a collection of values to be added to the page URL.

Parent Elements: Task

Child Elements: Variable

Attributes: None

Variable

Describes a command runtime parameter.

Parent Elements: RuntimeVariables

Child Elements: None

Attributes:

Name

Description

Name

Required. String. Contains the parameter name. The possible names depend on the value of the Scope attribute in the following way.

When the Scope attribute is set to Application, the Name attribute can take one of the following values:

  • LCID: String. Language identifier of the Web Interface user.

  • TaskID: String. Identifier of the current command to be executed in the Web Interface.

  • IsDSAdmin: Boolean. When True, the Web Interface user is an Adtive Roles  Admin.

  • BindInfo: String. The name (FQDN) of the computer running the Active Roles Administration Service to which the Web Interface is connected.

  • OperationalDC: String. FQDN of the domain controller the Active Roles Administration Service uses to access the managed domain.

  • PortalHomePage: String. The URL of the Home page of the Web Interface site you are customizing

When the Scope attribute is set to Trustee, the Name attribute is set to the name of any attribute of the user account under which the Web Interface is running.

When the Scope attribute is set to SecurableObject, the Name attribute is set to the name of any attribute of the object to which the command applies.

QueryParamName

Required. String. Contains the name of the query.

Scope

Required. String. This attribute must take one of the following values:

  • "Application"

  • "Trustee"

  • "SecurableObject"

IsCommandAllowedFilters

Describes a collection of the command filters. A filter enables/disables the command based on the certain criteria.

Parent Elements: Task

Child Elements: AVFilter, AAEFilter, VBSFilter

Attributes: None

AVFilter

Describes the command filter based on the values of the processed object properties.

Parent Elements: IsCommandAllowedFilters

Child Elements: None

Attributes: 

Name

Description

AttributeName

Required. Contains the ldapDisplay name of the processed object attribute.

AttributeValue

Required. A comma separated list of values. This filter enables the command when the value of the processed object attribute (AttributeName) is equal to any of values from the AttributeValue list.

AAEFilter

Describes the command filter based on the values of the allowedAttributesEffective property of the processed object.

Parent Elements: IsCommandAllowedFilters

Child Elements: None

Attributes: 

Name

Description

Attributes

Required. A comma separated list of attribute names. 

VBSFilter

Describes the command filter based on a VBScript function that checks whether the command is available for the Web Interface's user. 

Parent Elements: IsCommandAllowedFilters

Child Elements: None

Attributes: 

Name

Description

MethodName

Required. The name of a function that checks whether the command is available for the Web Interface's user. Write this function into the file:
'<WI folder>\Web\Public\CustomCode\IsCommandAvailable.vbs'
The <WI folder> refers to the folder where Web Interface is installed.

Examples

The following code snippet illustrates the use of the XML elements in the attribute edsaWICommands.

<?xml version="1.0"?>
<Tasks CustomizationVersion="33" xmlns="http://quest.com/ActiveRolesServer/WebInterface">
  <Task ID="GroupProperties" ResID="WIS_COMMAND_GENPROPERTIES" ToolTipResID="WIS_COMMAND_GENPROPERTIES_DES" HelperResID="WIS_COMMAND_GENPROPERTIES_HELP"
    TaskType="1" FormID="GroupProperties" DescriptionResID="WIS_COMMAND_GENPROPERTIES_GROUP_DESCRIPTION"/>
  <Task ID="GroupManager" ResID="WIS_COMMAND_GROUPMANAGER" ToolTipResID="WIS_COMMAND__GROUPMANAGER_TOOLTIP" HelperResID="WIS_COMMAND_GROUPMANAGER_HELP"
    TaskType="0" Action="my_groupManager.asp" DescriptionResID="WIS_COMMAND_GROUPMANAGER_DESCRIPTION"> 
    <IsCommandAllowedFilters><VBSFilter MethodName="IsAvailableAttribute"/></IsCommandAllowedFilters></Task>
  <Task ID="Separator" ResID="WIS_SEPARATOR_TITLE" PlainTextLabel="MySeparator" TaskType="5"/>
</Tasks>

The following function (VBScript) illustrates how to check whether the Web Interface user can modify the "description" property of a directory object managed with the Web Interface.

  To use this function

  1. Set the attribute MethodName of the VBSFilter element to the function name (IsAvailableAttribute in this example).

  2. Append the following code snippet to the file '<WI folder>\Public\CustomCode\IsCommandAvailable.vbs'.
    The <WI folder> refers to the folder where Web Interface is installed.

Function IsAvailableAttribute(ByRef objCommandContext)
  Dim allowedAttributesEffective, Item, attributeName
   ' Set the attribute name
  attributeName = LCase("Description")
  objCommandContext.DirectoryObject.GetInfoEx Array("allowedAttributesEffective"),0
  allowedAttributesEffective = objCommandContext.DirectoryObject.GetEx("allowedAttributesEffective")
  IsAvailableAttribute = False   
 
  If IsArray(allowedAttributesEffective) Then
       For each Item in allowedAttributesEffective
           If LCase(Item) = attributeName Then
               IsAvailableAttribute = True
               Exit Function
           End If
       Next
  End If
End Function
