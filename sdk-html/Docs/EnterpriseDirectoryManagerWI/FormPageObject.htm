FormPage Object

The FormPageobject represents the current form that includes the custom entry. The FormPageobject is available within the Get_XXX function used to create the custom entries. For more information about this function, refer to Creating Custom Entries.

Methods

The FormPageobject supports the following methods:

  • AddCustomHTML Method (deprecated)

  • AddDropDownList Method

  • RegisterAttributeControl Method

  • Write Method

AddCustomHTML Method (deprecated)

The Sub procedure that adds your custom HTML code used to create the custom entry.

objFormPage.AddCustomHTML(ByVal ID,ByVal strHTML)

Arguments

ID
[in] Required. String. Contains the entry identifier
strHTML
[in] Required. String. Specifies the HTML code you want to add.

Remarks

The AddCustomHTML method is considered obsolete. While this method is still supported in the current version of Active Roles, it may be removed in the future. When writing new scripts, you should avoid using this method. When modifying existing scripts, you are strongly encouraged to remove any dependency on this method. Use the Write method instead.

AddDropDownList Method

The Function procedure that adds a drop-down list to the current form and returns a reference the DropDownList object. The DropDownList object provides the properties and methods that allow you to easily configure the newly created drop-down list control.

objFormPage.AddDropDownList(ByVal ID)

Arguments

objFormPage
[in] Required. Refers to a FormPage type object.
ID
[in] Required. String. Specifies the ID for the drop down list to add.

RegisterAttributeControl Method

The Sub procedure that registers the specified custom entry on the Administration Service.

ObjFormPage.RegisterAttributeControl(ByVal strEntryID, ByVal strAttrName)

Arguments

objFormPage

Required. Refers to the FormPage object.

strEntryID

[in] Required. String. Contains the entry identifier.

strAttrName

[in] Required. String. Contains the ldapDisplayName of the object attribute associated with the entry.

 

The RegisterAttributeControl method can be called only within a Get_XXX function used to create a custom entry. Once thismethod is called, the web client will behave in the following way: when you enter a value of an entry, the web client will automatically generate the same value for the custom entry if the object property associated with this custom entry is affected by the specific property generation policy described in the following example.

Write Method

The Sub procedure that adds your custom string to the current form. The HTML format is supported.

objFormPage.Write(ByVal str)

Arguments

objFormPage
[in] Required. Refers to the FormPage object.
str
[in] Required. String. Specifies the string you want to add.
