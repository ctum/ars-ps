The Interface Settings

This topic provides information about  the format of settings for the user interface components. For a brief description of the user interface components, refer to "User Interface Overview" in the Active Roles Web Interface Help.

XML Elements

The edsaWISettings attribute of the WorkingCopy object within the Interface Settings container stores information in the Extensible Markup Language (XML). XML elements that control the Web Interface view are listed below.

Important:

After you modify the edsaWISettings attribute, for changes to take effect, reload the Web Interface.

 

Settings

Describes a collection of the interface settings.

Parent Elements: None

Child Elements: Banner, CustomObjectIcons, CustomizationPageText, DefaultMenu, GlobalSettings, HomePageText, RootTreeNodes, TaskBoxes, UIPolicies, SummaryPanes, ErrorPage

Attributes:

Name

Description

CustomizationVersion

Required. Identifies the current customization version number. Must be the same for all attributes of the WorkingCopy object.

Important:

The value of this attribute depends on the application version. This value is stored in the attribute edsaWITemplateVersion of the WI configuration container objects (see Storing Customization Settings).

 

Banner

Describes banners and links displayed on the Web Interface.

Parent Elements: Settings

Child Elements: None

Attributes:

Name

Description

LeftImage

Required. String. Contains URL of the image displayed in the left part of the WI window, above the Navigation Bar.

RightImage

Required. String. Contains URL of the image displayed in the right part of the WI window, above the Navigation Bar.

LeftImageUrl

Required. String. Contains URL of the page displayed after you click the image in the left part of the WI window

RightImageUrl

Required. String. Contains URL of the page displayed after you click the image in the right part of the WI window

 

ConditionFilterSets

Describes a collection of the custom filters. A filter enables/disables the corresponding menu command or item on a Web Interface page basing on the certain criteria.

Parent Elements: Settings

Child Elements: ConditionFilterSet

Attributes: None

 

ConditionFilterSet

Describes a custom filter for menu command or item on a Web Interface page.

Parent Elements: ConditionFilterSets

Child Elements: AAEFilter, AVFilter, and VBSFilter

Attributes:

Name

Description

ID

Required. Contains the identifier of the filter set.

 

AAEFilter

Describes the command filter based on the values of the allowedAttributesEffective property of the processed object.

Parent Elements: ConditionFilterSet

Child Elements: None

Attributes:

Name

Description

Attributes

Required. A comma separated list of attribute names. 

 

AVFilter

Describes the command filter based on the values of the processed object properties.

Parent Elements: ConditionFilterSet

Child Elements: None

Attributes: 

Name

Description

AttributeName

Required. Contains the ldapDisplay name of the processed object attribute.

AttributeValue

Required. A comma separated list of values. This filter enables the command when the value of the processed object attribute (AttributeName) is equal to any of values from the AttributeValue list.

 

VBSFilter

Describes the command filter based on a VBScript function that checks whether the command is available for the Web Interface's user. 

Parent Elements: ConditionFilterSet

Child Elements: None

Attributes:

Name

Description

MethodName

Required. The name of a function that checks whether the command is available for the Web Interface's user. Write this function into the file:
'<WI folder>\Public\CustomCode\IsCommandAvailable.vbs'
The <WI folder> refers to the folder where Web Interface is installed.

Scope

Required. String. Specifies the scope of the filter.

This attribute must take one of the following values:

  • DN - the filter context is the object whose DN is specified in the TargetDN attribute.  

  • Trustee - the filter context is set to the user account under which the Web Interface is running.

  • SecurableObject - the filter context is set to the object to which the command applies

TargetDN

Optional, String. Specifies the DN of an object.

 

CustomObjectIcons

Describes a collection of custom icons for directory objects the Web Interface displays.

Parent Elements: Settings

Child Elements: Icon

Attributes: None

 

Icon

Describes custom icons for an object class.

Parent Elements: CustomObjectIcons

Child Elements: None

Attributes:

Name

Description

ObjectClass

Required. Contains the name of the object class to which the custom icon will apply. 

BigIconFileName

Required. Sets the file name for the big custom icon. The recommended size for big icons is 32 x 32.

SmallIconFileName

Required. Sets the file name for the small custom icon. The recommended size for small icons is 16 x 16.

The image files set by the BigIconFileName and SmallIconFileName attributes must be saved into the folder <WI folder>\Public\images, where <WI folder> refers to the folder where the Web Interface is installed.

Parent Elements: Settings

Attributes:

DefaultMenu

Describes a collection of menus on the navigation bar (see "User Interface Overview" in the Web Interface Help).

Parent Elements: Settings

Child Elements: Item

Attributes: None

 

Item

Describes an item of the navigation bar menu.

Parent Elements: DefaultMenu

Child Elements: None

Attributes:

Name

Description

DescriptionResID

Required. Contains the identifier of the string constant containing a description of the command.

ID

Required. Contains the identifier of the menu item.

NavigateCommand

Required. Contains a JScript code snippet to be run when you click the item.

NavigateUrl

Required. Contains a shortcut to a Web page to be opened after clicking the item.

ResID

Required. Identifier of a string constant that contains the resource ID.

ToolTipResID

Required. Contains the identifier of the string constant containing the command tooltip.

Visible

Optional. Boolean. When True (default value) the menu item is visible.

FilterSetID

Optional. String. Identifies the ConditionFilterSet element that controls visibility of the item. If all filters in that ConditionFilterSet element evaluate to True, the item is visible; otherwise, the item is hidden. The value of the FilterSetID attribute should be the same as the ID attribute value of the ConditionFilterSet element.

The FilterSetID attribute has no effect if the Visible attribute is set to False.

 

GlobalSettings

Describes a collection of the Web Interface global settings.

Parent Elements: Settings

Child Elements: AutoLogout

Attributes:

Name

Description

SearchPageSize

Required. Integer. Specifies the maximum number of search results per page. This setting applies to any search page available in the Web Interface.

FilterOn

Optional. Boolean. The default value is true. Setting this attribute to false causes the filter bar to be hidden by default on all pages and dialog boxes that display lists of objects (for example, the filter bar will be hidden on the Select Object dialog box.)

ObjectPickerShowFindIn

Optional. Boolean. The default value is true. When true, the Find in text box and the Browse button are displayed on the Select Object dialog box. The Find in text box displays the name of the container where you want the search for objects to start.

ObjectPickerShowObjectName

Optional. Boolean. The default value is true. When true, the Name text box and the Search button are displayed on the Select Object dialog box. This text box provides a space for you to type the name (or multiple names separated by semicolons (;)) of directory objects to find.

Note

You can override the global settings specified with the ObjectPickerShowFindIn and ObjectPickerShowObjectName attributes for individual custom entries. For more information and a sample, refer to The Entries Settings.

SearchOperator

Optional. String. The default value is "StartsWith." This attribute defines the search operator used by the Web Interface search function when you search for objects using the Select Object dialog box,Quick Search or Filter Bar.

The SearchOperator attribute can take any of the following values:

  • "StartsWith": Causes the search function to find objects whose naming property begins with the string of characters you typed, for example, in the Select Object dialog box or in the Quick Search text box.

  • "Contains": Causes the search function to find objects for which the string of characters you typed occurs anywhere in their naming property.

  • "Equals": Causes the search function to find only the objects whose naming property exactly matches the string of characters you typed.

SearchPageCount

Required. Integer. Specifies the number of links to available search results pages displayed at the bottom of any search results page.

LanguageId

Required. Integer. Specifies the default language LCID.

MaxRowsWhenSaveCSV

Required. Integer. Specifies the maximum number of search results to save into a CSV file.

UIPolicyId

Required. String. Specifies the default UI policy ID.

ColorScheme

Required. String. Specifies the default color scheme ID.

LeafClasses

Required. Specifies a list of the leaf object classes. A leaf object is an object that has no child objects.

QuickSearchInAdLds

Required. Boolean. When true, the Web Interface will search the managed Active Directory domains and AD LDS partitions.

ShowRecycledObject

Required. Boolean. When true, cause the Web Interface to display recycled objects when viewing the Deleted Objects container.

Examples

The following snippet illustrates how to specify custom icons for the user objects.

<CustomObjectIcons>
  <Icon ObjectClass="user" BigIconFileName="customUser_big.gif" SmallIconFileName="customUser_small.gif" />
</CustomObjectIcons>

The following snippet illustrates how to display your custom item on the Web Interface home page. The constant CST_HOME_MYITEM_HEADER contains the item header to be displayed on the home page, such as "My Custom Commands."

<Item ID="customization" Location="default" Header="CST_HOME_MYITEM_HEADER"
Description="CST_HOME_MYITEM_DESCRIPTION" NavigateUrl="~/Customization/MY.aspx" NavigateCommand="" />

Important:

The XML schema describing the element Settings requires its child elements to appear in the following sequence within the containing element Settings:
<xsd:sequence>
<xsd:element ref="Banner" minOccurs="1" maxOccurs="1" />
<xsd:element ref="HomePageText" minOccurs="1" maxOccurs="1" />
<xsd:element ref="CustomizationPageText" minOccurs="1" maxOccurs="1" />
<xsd:element ref="TaskBoxes" minOccurs="1" maxOccurs="1" />
<xsd:element ref="DefaultMenu" minOccurs="1" maxOccurs="1" />
<xsd:element ref="RootTreeNodes" minOccurs="0" maxOccurs="1" />
<xsd:element ref="CustomObjectIcons" minOccurs="0" maxOccurs="1" />
<xsd:element ref="UIPolicies" minOccurs="0" maxOccurs="1" />
<xsd:element ref="ConditionFilterSets" minOccurs="0" maxOccurs="1" />
<xsd:element ref="GlobalSettings" minOccurs="1" maxOccurs="1" />

</xsd:sequence>

For example, the edsaWISettings attribute must contain the CustomObjectIcons element before the UIPolicies element, otherwise the Web Interface will fail returning an error
For more information, refer to "XML Schema Elements" in the MSDN Library.

Show/Hide Menu Item Depending on User Type

This example demonstrates how to show the Approval menu item only for users whose Description attribute is set to Administrator.

  1. Add the following code into the <wi_folder>\CustomCode\IsCommandAvailable.vbs file:

Function IsApprovalAvailable(ByRef objCommandContext)
    objCommandContext.GetInfoEx Array("description"),0
    Dim description : description = objCommandContext.Get("description")
    Write "description = " & description
    If description = "Administrator" Then
        IsApprovalAvailable = True
    Else   
        IsApprovalAvailable = False
    End If
    
End Function

  1. Add the filter set into the edsaWISettings attribute of the WorkingCopy objest:

<ConditionFilterSets>
   <ConditionFilterSet ID="filterSet1">
     <VBSFilter MethodName="IsApprovalAvailable" Scope="Trustee"/>
   </ConditionFilterSet>
</ConditionFilterSets>

  1. Link the filter set you created to the Approval menu item by opening for edit the XML code included the edsaWISettings attribute and setting to filterSet1 the Item element in which the ID attribute is set to Approval and which is included into the TaskBoxes element. For example:

<Item ID="approval"
          Location="default"
          Header="WIS_TB_HOME_APPROVAL_HEADER"
          Description="WIS_TB_HOME_APPROVAL_DESCRIPTION"
          NavigateUrl="~/Approval/SearchResult.aspx?TaskID=TASKS_TO_COMPLETE"
          NavigateCommand=""
          IconPath="~/images/taskApproval.png"
          IconHoverPath="~/images/taskApproval_hover.png"
          Visible="true"
          FilterSetID="filterSet1"
           />

  1. Select Reload in the Customization menu of the Web Interface.
