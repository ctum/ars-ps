Creating a Menu Command

This topic explains how you can create a command for custom menus associated with directory objects managed with the Web Interface. A menu represents a list of commands relating to objects of the appropriate type. Examples of menus: the User menu, the Group menu, etc.

The commands used in menus are described in the attribute edsaWICommands while the menu structure is described in the attribute edsaWIMenus of the WorkingCopy object .

To create a custom menu, use the following procedure to make the appropriate changes to the attributes edsaWICommands and edsaWIMenus of the object WorkingCopy residing in the Customization Settings container.

To make changes to the attribute edsaWIMenus

  1. Save the value of the attribute edsaWIMenus into a text file.

  2. Open this file with any XML editor, such as XML Spy, and find the element TaskList that defines a menu to which you want to add a custom command.

  3. For the element TaskList, insert the child element Task in the following way:

    <Task ID=strID>
    Where:
    strID     - Contains the identifier of the menu command. Note, that the attribute edsaWICommands must contain the element Task with the same attribute ID.

  4. Save changes to the attribute edsaWIMenus.

To make changes to the attribute edsaWICommands

  1. Save the value of the attribute edsaWICommands into a text file.

  2. Open this file with any XML editor, such as XML Spy, and insert the element Task in the following way:

    <Task Action=strfName ID=strID ResID=strResID ToolTipResID=strToolResID HelperResID=strHelperResID
        TaskType=intType FormID=strFormID DescriptionResID=strDescrResID/>
    Where:
    strfName     - Required, if TaskType=0. The URL of the .asp file to be executed after you click the command. 
    strID     - Unique identifier of the command. The identifier must be also defined in the property edsaWIMenus.
    strResID     - Identifier of the string constant containing the name of the resource, such as tab or dialog box, where the command is used.
    strToolResID     - Optional. Identifier of the string constant containing the command tooltip.
    strHelperResID     - Optional. Identifier of the string constant containing the command helper.
    intType - Required. The type of the command. The possible values are 0, 1 or 5.
    strFormID     - Required, if TaskType=1. Unique identifier of the form associated with the command.
    strDescrResID     - Optional. Identifier of the string constant containing a description of the command.

    For detailed information about attributes of the element Task, refer to The Commands Settings.

  3. Save changes to the attribute edsaWICommands.

Note:

 

For changes to take effect,  reload the Web Interface.
