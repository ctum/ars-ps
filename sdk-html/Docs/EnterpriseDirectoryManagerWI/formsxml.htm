The Forms Settings

This topic provides information about  the format of settings for forms associated with the Form-type commands defined in the attribute edsaWICommands.

A form is a structured page with predefined areas for entering and changing information about directory objects.

A form consists of elements (entries) such as text boxes, check boxes, option buttons, etc. The form elements are defined in the attribute edsaWIEntries.

A form includes one or several tabs that group related elements on the same page. Each set of related elements is presented within the tab labeled with the name of the set, such as General, Managed by, and Object on the Properties form. The form tabs are defined using the XML element FormTab.

Settings for forms are stored in the attribute edsaWIForms. Modifying this attribute, you can create or modify forms associated with the menu commands.

XML Elements

The attribute edsaWIForms stores information in the Extensible Markup Language (XML).  XML elements you can use are listed below.

Important:

After you modify the attribute edsaWIForms, for changes to take effect, reload the Web Interface.  

Forms

Describes the collection of forms.

Parent Elements: None

Child Elements: Form

Attributes:

Name

Description

CustomizationVersion

Required. Identifies the current customization version number. Must be the same for all attributes of the WorkingCopy object.

Important:

The value of this attribute depends on the application version. This value is stored into the attribute edsaWITemplateVersion of the WI configuration container objects (see Storing Customization Settings).

Form

Describes a form and its properties.

Parent Elements: Forms

Child Elements: FormTab

Attributes:

Name

Description

ChildClass

Required, if FormType=2. Identifies the ldapDisplayName property of the object class to be created using the form.

DescriptionResID

Optional. Identifier of the string constant containing a description of the form.

FormType

Required. The type of the form.

The possible values are:

1 - Object Properties form
2 - Create New Object form
3 - Rename form
4 - Copy form
9 - ModifyLooksLikeCreate form:

ID

Required. Unique identifier of the form. The form identifier cannot begin with a number, and can only contain "A-Z", "a-z", and "0-9" characters.

ResID

Optional. Identifier of the string constant containing the form name.

ShowEffectivePolicyIcons

Optional. Boolean. The default value is TRUE. When FALSE, the icon of the information window Show Effective Policy is hidden on the form.

TargetClass

Required, if FormType<>2. Contains the ldapDisplayName of the object class to which the form can be applied. For example, for the User Properties form, this attribute must be set to "user".

VerificateType

Optional. The type of the form validation. Verifies whether the command associated with the form is available for the Web Interface's user.

The possible values are:

0 - the command is always available (default value)
1 - the command is available if the form allows the user to modify at least one attribute of the directory object
2 - the command is available only if the form allows the user to modify all attributes of the directory object available on the form tabs

 FormTab

Describes the form tab and its properties.

Parent Elements: Form

Child Elements: FormEntry

Attributes:

Name

Description

ID

Required. Unique identifier of the tab.

ResID

Optional. Identifier of the string constant containing the tab name.

ToolTipResID

Optional. Identifier of the string constant containing the tab tooltip.

Note:

Identifiers and values of the string constants are saved in the attribute edsaWIStrings. We strongly recommend that the names of your custom string constants, such as identifiers or function names, begin with the prefix cst_, for example cst_employeeID, cst_jobDescriptionIDControl, etc.

FormEntry

Describes the form entries.

Parent Elements: FormTab

Child Elements: None

Attributes:

Name

Description

ID

Required. Unique identifier of the entry. The entry must be also defined in the attribute edsaWIEntries.

 Example

The following snippet illustrates the use of the XML elements in the attribute edsaWIForms.

<?xml version="1.0"?>
<Forms CustomizationVersion="33">
    <Form ID="ComputerProperties" FormType="1" ResID="WIS_FORM_COMPUTER_PROPERTIES" TargetClass="computer" DescriptionResID="">
        <FormTab ID="general" ResID="WIS_UP_GENERAL">
            <FormEntry ID="sAMAccountNameComp"/>
            <FormEntry ID="dNSHostName"/>
            <FormEntry ID="role"/>
            <FormEntry ID="description"/>
            <FormEntry ID="trustForDelegation"/>
        </FormTab>
    </Form>
</Forms>
