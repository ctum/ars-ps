Developing Custom ASP Web Page

This topic explains how you can create your custom ASP page and display it with a page view command related with an object administered with the Web Interface.

Note

To use .asp pages, you should enable the Web service extension "asp" on the computer that runs the Web Interface. Otherwise, clicking your page view commands will result in a 404 error.
The procedure used to enable the Web service extensions depends on the installed version of Internet Information Services (IIS). For IIS 6.0, the appropriate procedure can be found in Microsoft TechNet using the link http://technet2.microsoft.com/windowsserver/en/library/1f59ef09-b534-450d-8e54-5c87c8af777b1033.mspx?mfr=true
For procedures applied to other versions of IIS, refer to Internet Information Services Help.

 

To create a page view command, use the following procedures to make the appropriate changes to the edsaWICommands, edsaWIStrings, edsaWIMenus attributes of the object WorkingCopy residing in the Customization Settings container and to the edsaWISettings attribute of the object WorkingCopy residing in the Interface Settings container.
 

To make changes to the attribute edsaWICommands

  1. Save the value of the edsaWICommands attribute  into a text file.

  2. Open this file with any XML editor, such as XML Spy, and insert the element Task in the following way:

    <Task Action=strPageUrl ID=strID ResID=strResID ToolTipResID=strToolResID HelperResID=strHelperResID
        TaskType=10 FormID="" DescriptionResID=strDescrResID FullQualifiedTypeName="" UIPolicyID=strUIPolicyID>
    <RuntimeVariables>
    <Variable Name=strldapName QueryParamName=strParam Scope=strScope />
    <!--
    code goes here
    -->
    </RuntimeVariables>
    </Task>

    Where:
    strPageUrl         - The URL of the .asp file to be opened after you click the command. 
    strID     - Unique identifier of the command. The identifier must be also defined in the property edsaWIMenus.
    strResID     - Identifier of the string constant containing the name of the resource, such as tab or dialog box, where the command is used.
    strToolResID     - Optional. Identifier of the string constant containing the command tooltip.
    strHelperResID     - Optional. Identifier of the string constant containing the command helper.
    strDescrResID     - Optional. Identifier of the string constant containing a description of the command.
    strUIPolicyID     - Optional. Identifier of the UI policy connected with this command. The identifier must be also defined in the attribute edsaWISettings.

    The child elements Variable of the element RuntimeVariables describe the URL parameters to be passed to the Web Interface.
    To pass a value of an attribute of the object to which the page view command applies, the strldapName must be set to the attribute's ldapDisplayname, while the strScope must be set to "SecurableObject."
    For more information about attributes of the element Variable, see The Commands Settings.

  3. Save changes to the attribute edsaWICommands.

To make changes to the attribute edsaWIMenus

  1. Save the value of the attribute edsaWIMenus into a text file.

  2. Open this file with any XML editor, such as XML Spy, and under the appropriate element TaskList insert the element Task that describes the page view command:
    <Task ID=strID />
    Where strID - Unique identifier of the command. It must be equal to strID defined in Step 2 of the procedure above.

  3. Save changes to the attribute edsaWIMenus.

Note

To add your page view command to the Command Menu related to an object you are administering, the element Task must be inserted under the parent element TaskList that describes the object command menu (see examples in Sample Custom ASP Pages below).

To make changes to the attribute edsaWIStrings

  1. Save the value of the attribute edsaWIStrings into a text file.

  2. Open this file with any XML editor, such as XML Spy, and insert the description of resources used in the attribute edsaWICommands.

  3. Save changes to the attribute edsaWIStrings.

To make changes to the attribute edsaWISettings

  1. Save the value of the attribute edsaWISettings into a text file.

  2. Open this file with any XML editor, such as XML Spy, and insert the description of User Interface (UI) policy for your page view command. For example:
    <UIPolicies>
    <UIPolicy ID=strUIPolicyID
    ShowQuickSearch="false"
    ShowPathToAdObject="false"
    ShowLeftPane="false"
    ShowMenuTab="false"
    ShowTreeTab="false"
    ShowTopPanel="false" />
    </UIPolicies>

    Where strUIPolicyID - identifier of the UI policy defined in the above procedure for changing the attribute edsaWICommands.

Note

This procedure applies only if you have defined the attribute UIPolicyID of the element Task in esdaWICommands. Otherwise, leave the attribute edsaWISettings unchanged.

Sample Custom ASP Pages

To clarify the procedures above, consider the following examples illustrating how to add a page view command to the User menu and to the Group menu. Clicking these commands displays the custom ASP pages named  test.asp and groupmembers.asp, respectively.

Test.asp Page

The Test.asp page displays values of URL parameters passed to this page.

In the folder '<WI installation folder>\Public\', create the following file test.asp:

<%@ Language="VBS" %>
<html>



  <style type="text/css">
    H4, TD
    {
      font-family: Verdana;    }
    TD
    {
      font-size: 10px;
    }
  </style>
  <body bgcolor="">
    <h4>Page Viewer</h4>
    <table cellpadding="3">
      <% For Each param In Request.QueryString %>
        <tr>
          <td style="text-align: right;"><%= param %></td><td>=</td><td><%= Request.QueryString(param) %></td>
        </tr>
      <% Next %>
    </table>
  </body>
</html>

To deploy the Test.asp page

  1. Add the following code snippet to the attribute edsaWICommands:

 <Task ID="PageViewer"
        ResID="WIS_PAGE_VIEWER"
        DescriptionResID=" WIS_PAGE_VIEWER"
        ToolTipResID="WIS_PAGE_VIEWER"
        HelperResID="WIS_PAGE_VIEWER "
        TaskType="10"
        FormID=""
        Action="test.asp?param1=value1&amp;param2=value2"
        ClassNames="user"
        Properties=""
        FullQualifiedTypeName=""
        UIPolicyID="TestUIPolicy">
        <RuntimeVariables>
      <Variable  Name="LCID" QueryParamName="LCID" Scope="Application"  />
      <Variable  Name="TaskId" QueryParamName="TaskId"  Scope="Application" />
      <Variable  Name="IsDSAdmin" QueryParamName="IsDSAdmin" Scope="Application"  />
      <Variable  Name="BindInfo" QueryParamName="BindInfo" Scope="Application"  />
      <Variable  Name="OperationalDC" QueryParamName="OperationalDC"  Scope="Application" />
      <Variable  Name="userPrincipalName" QueryParamName="(SO)userPrincipalName" Scope="SecurableObject" />
      <Variable  Name="distinguishedName" QueryParamName="(T)distinguishedName"  Scope="Trustee" />
    </RuntimeVariables>
  </Task>

  1. Add the following code snippet to the attribute edsaWIMenus:

<Task ID="PageViewer" />

Note

To add your custom page view command to the User menu, add the child element Task under the element <TaskList TargetClass="user" DefaultTask ="UserProperties"...>.

  1. Add the following code snippet to the attribute edsaWIStrings:

<Resources>
    <Res ID="WIS_PAGE_VIEWER" Value="Page View Command" />


</Resources>

  1. Add the following code snippet to the attribute edsaWISettings:

<UIPolicies>
    <UIPolicy ID="TestUIPolicy"
              ShowQuickSearch="false"
              ShowPathToAdObject="false"
              ShowLeftPane="false"
              ShowMenuTab="false"
              ShowTreeTab="false"
              ShowTopPanel="false" />
  </UIPolicies>

  1. For changes to take effect,  reload the web interface.

GroupMembers.asp Page

The GroupMembers.asp page displays the number of members for a selected group.

In the folder '<WI installation folder>\Public\', create the following file groupmembers.asp:

<%@ Language="VBS" %>
<html>
  <style type="text/css">
    H4, TD
    {
      font-family: Verdana;    }
    TD
    {
      font-size: 10px;
    }
  </style>
  <body bgcolor="">
    <h4>Total number of members in the group <%= Request.QueryString("groupDN") %> </h4>
    <%
       Set Group=GetObject("EDMS://" + Request.QueryString("groupDN"))
       Result = ""
       Ndirect = 0
       NNested = 0
	   on error resume next
	   Group.GetInfoEx Array("edsaMember"),0
	   arrMembers=Group.GetEx("edsaMember")
	   If err.Number = 0 then
	       on error goto 0
	       Result = Result + "Number of direct members: "
	       For Each Member in arrMembers
	   	      Ndirect = Ndirect +1
	       Next
	   End If
	   Result = Result + CStr(Ndirect) + "<BR>"
	   on error resume next
	   Group.GetInfoEx Array("edsaMemberIndirect"),0
	   arrMembers=Group.GetEx("edsaMemberIndirect")
	   If Err.Number = 0 Then
	       on error goto 0
	       Result = Result + "Number of members due to group nesting: "
	       For Each Member in arrMembers
	   	      NNested = NNested +1
	       Next
	       Result = Result + CStr(NNested) + "<BR>"
	   End If
	   Ntotal = NDirect + NNested
	   on error goto 0
	   If Result="" then
	       Response.Write "The selected group has no members"
	   Else
	       Result = Result + "Total number: " + CStr(Ntotal)
	       Response.Write Result
	   End If
    %>
  </body>
</html>

To deploy the GroupMembers.asp page

  1. Add the following code snippet to the attribute edsaWICommands:

<Task ID="GroupViewer"
        ResID="WIS_GROUP_VIEWER"
        DescriptionResID="WIS_GROUP_VIEWER"
        ToolTipResID="WIS_GROUP_VIEWER"
        HelperResID="WIS_GROUP_VIEWER "
        TaskType="10"
        FormID=""
        Action="GroupMembers.asp"
        ClassNames="group"
        Properties=""
        FullQualifiedTypeName=""
        UIPolicyID="GroupUIPolicy">
        <RuntimeVariables> 
          <Variable  Name="distinguishedName" QueryParamName="groupDN" Scope="SecurableObject" />
        </RuntimeVariables>
  </Task>

  1. Add the following code snippet to the attribute edsaWIMenus:

<Task ID="GroupViewer" />

Note

To add your custom page view command to the Group menu, add the child element Task under the element <TaskList TargetClass="group" DefaultTask="GroupProperties">.

  1. Add the following code snippet to the attribute edsaWIStrings:

<Resources>
      <Res ID="WIS_GROUP_VIEWER" Value="Total number of members" />
</Resources>

  1. Add the following code snippet to the attribute edsaWISettings:

<UIPolicies>
    <UIPolicy ID="GroupUIPolicy"
              ShowQuickSearch="false"
              ShowPathToAdObject="false"
              ShowLeftPane="true"
              ShowMenuTab="true"
              ShowTreeTab="true"
              ShowTopPanel="true" />
  </UIPolicies>

  1. For changes to take effect, reload the web interface.

After you click the Total number of members command on the Group menu, you will be presented with a page that displays the number of the members in the group.
