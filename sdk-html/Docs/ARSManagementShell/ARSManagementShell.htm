About Active Roles Management Shell

Active Roles Management Shell, built on Windows PowerShell technology, provides a powerful command-line interface for Active Directory and Active Roles that enables automation of administrative tasks. With Active Roles Management Shell, you can administer directory objects, such as users and groups. For example, you can create new users and groups, modify user properties, and add or remove members from groups.

The management operations are performed either via the Active Roles proxy service or by directly accessing directory data on Active Directory domain controllers. In both cases, Active Roles Management Shell provides a flexible scripting platform that can reduce the complexity of scripts. By accessing directory data through the Active Roles proxy service, Active Roles Management Shell takes full advantage of the security, workflow integration and reporting benefits of Active Roles.

The command-line tools (cmdlets) provided by Active Roles Management Shell can be used in policy scripts and workflows.

For more information about Active Roles Management Shell, see the Active Roles Management Shell Reference Manual. You can open the Reference Manual by typing QuickRef in the Active Roles Management Shell console window.
