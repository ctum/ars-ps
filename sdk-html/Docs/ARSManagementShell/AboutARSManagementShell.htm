Develop Policy Scripts Using Active Roles Management Shell

This example illustrates how you can develop policy scripts using cmdlets provided by Active Roles Management Shell.

The script allows you to enforce the following administrative policy: each newly created user will be automatically added to the group specified in the "User group" parameter (for details see "Using Parameters in Policy Scripts" and "Declaring Parameters in Policy Scripts" in Related Topics). In addition, the "manager" property of the newly created user will be automatically set to the "managedBy" property of his/her parent container.

function onInit($Context)
{
#---- Declare the policy parameter "User group" ----
$par = $context.AddParameter("User group")
$par.DefaultValue = ""
$par.Description = "Specifies the group to which the user will be added."
$par.Syntax = "DN"  
}
 

function onPreCreate($Request)
{
#---- Verify the in-process object class ----
if ($request.Class -ne "user") {return}

  trap {continue}

  #---- Retrieve the user parent OU's manager ----
$manager = (Get-QADObject $Request.Parent.SubString(7) -IncludedProperties "managedBy").managedBy
if ($manager)
{
#---- Specify the user's manager ----
$Request.Put("manager",$manager)
}
}
 

function onPostCreate($Request)
{
#---- Verify the in-process object class ----
if ($request.Class -ne "user") { return }

  trap { continue }

  #---- Read the group DN from the "User group" parameter ----
$groupDN = [string]$PolicyEntry.Parameter("User group")
if ($groupDN)
{
#---- Add newly created user to the specified group ----
Add-QADGroupMember $groupDN -Member $Request.GUID
}
}

Note

This sample policy script uses cmdlets from the Windows PowerShell modules loaded by default. If your custom policy script requires additional PowerShell modules, load them using the Import-Module cmdlet.

  To use this sample policy script

  1. Register this script as a policy script module (script language: PowerShell) on the Administration Service.

  2. Create a Provisioning Policy Object (PO), and add the created script module to the PO.

  3. Use the Active Roles console to set the "User group" custom parameter to the distinguished name of a group to which you want to add newly created users.

  4. Link the PO to a directory folder, such as an Organizational Unit.

  5. Try to create a new user in the directory folder to which the PO is linked.
