Troubleshooting Policy Scripts

When you create a policy script, you may encounter a situation where a policy based on that script does not behave as expected. It may be difficult to determine the cause of the problem. The Script Editor in the Active Roles console can help you locate general syntax errors by using color to highlight and distinguish syntax elements such as command names, statement identifiers, quoted strings, and variables. Additionally, the console is capable of reporting script syntax errors at script policy run time. But when the policy behavior is unexpected whereas the syntax of the script is correct, it may be much more difficult to diagnose the problem. For this reason the console includes simple debugging functionality to troubleshoot Windows PowerShell based policy scripts by examining each step that the script makes as it executes. This functionality is called tracing.

To enable tracing for a given script, use the Active Roles console as follows:

  1. Open the Properties dialog box for the script module containing the script in question, and go to the Debugging tab in that dialog box.

  2. On the Debugging tab, select the Enable debugging check box, select the appropriate option from the Debug trace level list, and then click OK.

If you only want to examine each command step in the script, select the Trace script lines as they are executed option from the Debug trace level list. To examine each step and each variable assignment as they are made, choose the option Trace script lines, variable assignments, and function calls.

Given that tracing is enabled for a particular script, when Active Roles executes a policy based on that script, it creates a log containing the trace output. You can check the trace output in the Active Roles console, by clicking View Log on the Debugging tab in the Properties dialog box for the corresponding script module.
