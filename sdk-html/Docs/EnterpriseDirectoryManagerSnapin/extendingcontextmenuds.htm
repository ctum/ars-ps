Extending Context Menu

Active Roles allows you to add an item to the context menu displayed for directory objects. A context menu item can be added by implementing a COM in-proc server known as a context menu extension. A context menu item can also be added that invokes any file started with the ShellExecute API, such as an application or Web page URL. This is known as a static context menu item.

Extending the Context Menu With a Context Menu Extension

A context menu extension is a COM in-proc server that implements certain interfaces and is registered with Active Directory.

To create and install a context menu extension

  1. Create the context menu extension DLL. A context menu extension is a COM in-proc server that, at a minimum, implements the IShellExtInit and IContextMenu interfaces. In addition, you can implement the IEDMExtInit interface that defines a method allowing you to get the name of Administration Service to which the Active Roles snap-in is currently connected. For more information, see the MSDN library and Related Topics.

  2. Install and register the context menu sheet extension on computers where the context menu extension is used. For information about how to register the context menu extension, refer to Registering COM Objects.

Extending the Context Menu With a Static Context Menu Item

A static context menu item can be used to invoke any file started with the ShellExecute API, such as an application or Web page URL. To accomplished this, the static context menu item for a particular object class must be registered in the appropriate display specifier so that the static context menu item is added to the context menu of objects of that class.

Registering a Static Context Menu Item in a Display Specifier

Active Roles provides two display specifier attributes edsaEDMSContextMenu and edsaLDAPContextMenu that a static context menu extension item can be registered under. These attributes identify the static context menu items to display in the Active Roles snap-in. The attributes also identify programs of files executed by the Active Roles snap-in when you click the appropriate menu items.

All of these attributes are multi-valued and they require the following format:

<order number>,<item name>,<command>

The <order number> is an unsigned number that represents the item's position in the context menu. When a context menu is displayed, the values are sorted using a comparison of each value's <order number>. If more than one value has the same <order number>, those context menu extensions are loaded in the order they are read from Active Directory. If possible, use a non-existing <order number>, that is, one that has not been used by other values in the property. There is no prescribed starting position, and gaps are allowed in the <order number> sequence.

The <item name> is the string displayed in the context menu. The <item name> can include one "&" character that precedes the keyboard shortcut character for the menu item. This will cause the preceded character to be underlined. For example, if the <item name> is "My&Item", the menu text will be displayed as "MyItem", the "I" will be underlined and "I" will be the keyboard shortcut for the menu item.

The <command> is the program or file executed by the Active Roles snap-in. Either the full path must be specified or the file must exist in the computer's path environment variable. The file is invoked using the ShellExecute function. The <command> cannot contain additional parameters, for example, Notepad.exe Myfile.txt. Because ShellExecute is used, any file or address that can be passed to ShellExecute can be used for <command>. For example, if <command> contains "c:\Myfile.txt", c:\Myfile.txt will be opened with the application associated with the .txt extension. Likewise, if <command> contains "http://www.mycompany.com", the default Web browser is opened and will display the specified web page. Paths and application names with spaces are allowed.

Important:

If <command> is an application, such as a VBScript file (.vbs), the selected object's ADsPath and class are passed as command line arguments, separated by a space. When you employ the attribute edsaEDMSContextMenu, the ADSPath argument is passed as an Active Roles binding string (such as EDMS://<DC name>/<object's DN>). When you employ the attribute edsaLDAPContextMenu, the ADSPath argument is passed as an LDAP binding string (such as LDAP://<DC name>/<object's DN>).

For more information, and an example of a static context menu item, see Adding a Static Context Menu Item.
