Display Specifiers

In Active Directory, an object class defines a type of object that can be created in Active Directory. The definition of each object class is stored as a classSchema object in the Active Directory schema. In addition to the classSchema definition, each object class can have one or more display specifiers that specify the user interface data for objects of that class.

A display specifier is an Active Directory object of the displaySpecifier class. The attributes of a display specifier specify localized user interface data that describes the various UI elements for a particular object class. Display specifiers store data for property sheets, context menus, icons, creation wizards, and localized class and attribute names. For property pages and context menus, the Windows shell and Active Directory administrative snap-ins use this data to form different user interfaces for administrators and end users—one set of property pages and/or context menus can be associated with administrative applications while a different set of elements can be associated with end user applications.

Using Display Specifiers in Active Roles

In Active Roles, the displaySpecifier objects are stored in locale-specific containers within the Consolidated Display Specifiers container. To get the container's distinguished name, use the property edsvaEDMDisplaySpecifiersDN of the object rootDSE. The default DN of this container is as follows: 

CN=Consolidated Display Specifiers,CN=Application Configuration,cn=Configuration

The Consolidated Display Specifiers container has subcontainers that correspond to the various locales supported by the enterprise installation. These subcontainers are named using language identifiers. Active Roles supports several locales. For example, the name of the locale container for US-English is 409 while the German locale's container is named 407. Thus, an object class can have multiple display specifiers: one in each locale subcontainer.

Note:

You cannot change display specifiers stored in the container Consolidated Display Specifiers. Make your custom changes to the display specifiers stored in the locale containers within the container DisplaySpecifiers by Active Roles (Custom). For changes to take effect, restart the Active Roles Administration Service. 

Display Specifiers for Object Classes

The name of a displaySpecifier object is formed by appending the string "-Display" to the lDAPDisplayName of the object class. For example, the name of a displaySpecifier object for the user class is "user-Display".

If the display specifier for the object class of your interest is absent within the locale container, create it with the Active Roles snap-in. You can also create the locale containers.

To create a new locale container

  1. Chose the Raw mode for Active Roles snap-in display:

    • On the View menu, click Mode, and then in the View Mode dialog box, select the Raw Mode option and click OK.

  2. Start the New Directory Object Wizard:

    • In the console tree, expand the Configuration node, expand the Application Configuration node, and then right-click Active Roles Display Specifiers (Custom). In the context menu, point to All Tasks and then click Advanced Create.

  3. In the Create In window, do the following:

    • In the Name text box, specify the name for the locale container to be created (409 - for US-English, 407 - for German, etc.)

    • In the Object class list, click EDS-Display-Specifier-Container.

    When finished, click Next, and then click Finish to complete the wizard.

To create a new display specifier

  1. Chose the Raw mode for Active Roles snap-in display:

    • On the View menu, click Mode, and then in the View Mode dialog box, select the Raw Mode option and click OK.

  2. Start the New Directory Object Wizard:

    • In the console tree, expand the Configuration node, expand the Application Configuration node, expand Active Roles Display Specifiers (Custom), and then right-click the appropriate locale-specific container (the locale container for US-English is 409). In the context menu, point to All Tasks and then click Advanced Create.

  3. In the Create In window, do the following:

    • In the Name text box, specify the name for the display specifier to be created. Follow the DS name generation rule above.

    • In the Object class list, click Display-Specifier.

    When finished, click Next, and then click Finish to complete the wizard.

In addition to the standard ADSI displaySpecifier class attributes, Active Roles defines the following multi-valued attributes, which are used to extend the property sheets and context menus:

  • edsaEDMSContextMenu

  • edsaLDAPContextMenu

  • edsaEDMSPropertyPages

  • edsaLDAPPropertyPages

For information about how to use these attributes, see Registering Property Page and Context Menu Extensions.

Note:

Active Roles does not use the DS object's attributes adminPropertyPages and adminContextMenu the AD MMC snap-ins use for extensions of user interface. Setting these attributes has no effect.
