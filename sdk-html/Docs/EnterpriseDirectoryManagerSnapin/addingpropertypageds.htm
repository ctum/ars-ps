Adding Property Page

This sample C++ project demonstrates how to add an extension snap-in's property page to a property sheet displayed by the primary Active Roles snap-in.

All files related to this project are saved in the folder '...\SDK\Samples\ARServer Snap-in\Ds\Property Page Extension'.

Building the Sample

To build the sample project

  1. In Microsoft Visual Studio 2008, open the solution file DsPropTest.sln.

  2. On the Standard toolbar, choose Release from the Solution Configurations list box.

  3. On the Build menu, click Rebuild Solution.

Registering the Property Sheet Extension DLL in a Display Specifier

After you build the project, register the property sheet extension DLL under any of the following attributes of the appropriate display specifier: edsaEDMSPropertyPages, edsaLDAPPropertyPages. The format of values for those attributes are described in Registering Property Page and Context Menu Extensions.

To clarify the registration procedure, consider the steps you should perform to extend the property sheet for Organization Units.

To register the property sheet extension DLL in the user-Display DS

  1. Choose the Raw mode for Active Roles snap-in display:

    • On the View menu, click Mode, and then in the View Mode dialog box, select the Raw Mode option and click OK.

  2. In the console tree, expand the Configuration node, expand the Application Configuration node, expand Active Roles Display Specifiers (Custom), and then click the appropriate locale-specific container (the locale container for US-English is 409). Note that if the locale-specific container does not exist, you should create that container using this procedure.

  3. In the details pane, right-click the display specifier organizationalUnit-Display, point to All Tasks and then click Advanced Properties. Note that if the DS organizationalUnit-Display is absent within the locale-specific container, you should create that DS using this procedure.

  4. Using the Advanced Properties dialog box, set the attribute edsaLDAPPropertyPages to the following value: 1,{486B8AF4-DDEA-4C5B-B959-D9022922B4BA}

  5. Close the Advanced Properties dialog box and restart the Active Roles Administration Service.

Note:

 

{486B8AF4-DDEA-4C5B-B959-D9022922B4BA} is the string representation of the CLSID for the property page extension created with this sample project. This value is set in the file \DsPropTestATL\DSPropTest.idl (see the line: uuid(486B8AF4-DDEA-4C5B-B959-D9022922B4BA)).

How the Sample Works

After you build this project and register the created property page DLL in the display specifier organizationalUnit-Display, the Active Roles console will display a new tab Page From DSPropTestAtl.dll in the Properties dialog box for the OU objects.
