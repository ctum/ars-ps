Context Menu and Property Page Extensions

You can add context menu items to the existing context menus for a particular object class within the Active Roles snap-in. You can also add property pages to the property sheet of a particular object class. For example, you can create a context menu extension that adds menu items to the context menu of the group class. When a user views a group object, your context menu items are displayed with other context menu items in the group's context menu.

This section provides generic instructions for writing context menu and property page extensions to the Active Roles snap-in.

A snap-in is a COM in-process server DLL that exposes certain interfaces to MMC. A snap-in can house one or more COM objects that are actually responsible for exposing the interfaces of the snap-in to MMC.

To extend context menus with context menu items from an extension snap-in

  1. Create a context menu extension snap-in. The snap-in must implement and expose the IExtendContextMenu interface (for details, see the MSDN library). Note that you can use any existing a context menu extension snap-in for Active Directory Users and Computers instead.

  2. On the computers where you want the extension context menu items to be used, install and register the context menu extension snap-in for a particular object class so that its context menu items are added to the context menus of objects of that class. Note that you can register the same context menu extension object for more than one class.  For more information, refer to Registering Node Type.

  3. When the AddMenuItems method of your snap-in is called, the piDataObject value passed in the call is a pointer to the data object of the currently selected Active Directory object. Your snap-in can call the GetData method of the data object using the CFSTR_DSOBJECTNAMES clipboard format to extract context information about the selected directory object. For more information, see the MSDN library. 

To extend property sheets with property pages from an extension snap-in

  1. Create a property sheet extension snap-in. The snap-in must implement and expose the IExtendPropertySheet (or IExtendPropertySheet2) interface (for details, see the MSDN library). Note that you can use any existing a property sheet extension snap-in for Active Directory Users and Computers instead.

  2. On the computers where you want the extension property pages to be used, install and register the property sheet extension snap-in for a particular object class so that its property pages are added to the property sheets of objects of that class. Note that you can register the same property sheet extension object for more than one class.  For more information, refer to Registering Node Type.

  3. When the CreatePropertyPages method of your snap-in is called, the lpIDataObject value passed in the call is a pointer to the data object of the currently selected Active Directory object. Your snap-in can call the GetData method of the data object using the CFSTR_DSOBJECTNAMES clipboard format to extract context information about the selected directory object. For more information, see the MSDN library.
