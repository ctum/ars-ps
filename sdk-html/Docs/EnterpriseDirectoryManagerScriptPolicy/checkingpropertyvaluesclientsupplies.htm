Checking Property Values

To check values of the directory object properties Active Roles client supplies to Administration Service, use the onCheckPropertyValues event handler. This event handler is activated when you click the Next button in the New Object wizards or the Apply/OK button in the Properties dialog box.

When the supplied values don't meet an administrative policy, the onCheckPropertyValues event handler should call the SetPolicyComplianceInfo method provided by the IEDSPolicyComplianceRequest interface. This method displays an error message that informs an administrator about the policy violation (see the Figure beneath this page).

To provide complete information about administrative policies defined for the processing object, we recommend that you use the SetEffectivePolicyInfo method of the IEDSEffectivePolicyRequest interface. This method can be called within the onGetEffectivePolicy event handler.

Example Code

The following code snippets illustrate how to check a value of the user's department property that you supply when creating or modifying a user account. The script exhibits the following characteristics and features:

  • Retrieves an array (list) of valid department names using the GetDepartmentNames function. You can manually define that list or obtain it, for example, from the HR database (see Integrating an HR Database with Active Roles).

  • Checks whether the supplied department property is in that list.

  • Displays the special designed window (see the Figure below) when the department property value does not meet the administrative policy.

[VBScript]
' Return a list of valid department names for your organization.
Function GetDepartmentNames(Request)
   ' Modify this function before using the script
   '
   ' Add your code here
   ' Example: connect to your HR database to retrieve a list of the organization departments
End function
' Returns TRUE if the department property value meets the administrative policy
' or FALSE otherwise
Function ValidateDepartment(Request)
   ValidateDepartment = True
   ' Check the class of the object to be created or modified
    If Request.Class <> "user" Then
        Exit Function
    End If
    Dim arrDepNames
    ' Retrieve the list of the valid department names
    arrDepNames = GetDepartmentNames(Request)
    Dim strDep, strUserDep
    ' Retrieve the supplied value of the department property
    strUserDep=CStr(Request.Get("department"))
    If strUserDep = vbEmpty Then
        Exit Function
    End If
    ' Check the supplied value
    For Each strDep In arrDepNames
        If strDep = strUserDep Then
             Exit Function
        End If
    Next
    ValidateDepartment = False
End Function
' onGetEffectivePolicy event handler
Sub onGetEffectivePolicy(Request)
    If Request.Class <> "user" Then Exit Sub
    strAttrname="department"
    arr=GetDepartmentNames(Request)
    ' Call the SetEffectivePolicyInfo method
    Request.SetEffectivePolicyInfo strAttrname,EDS_EPI_UI_DISPLAY_NOTE,_
    "The value is offered by Administrative Policy"
    ' This method displays the Values tab that contains a list of possible department names
    Request.SetEffectivePolicyInfo strAttrname,EDS_EPI_UI_POSSIBLE_VALUES,arr
End Sub
' onCheckPropertyValues event handler
Sub onCheckPropertyValues(Request)
    If ValidateDepartment(Request) <> True Then
        Request.SetPolicyComplianceInfo "department",_
        EDS_POLICY_COMPLIANCE_ERROR,_
            "The value of the department property does not meet the corporate policy."
    End If
End Sub

# Returns TRUE if the department property value meets the administrative policy
# or FALSE otherwise
function ValidateDepartment($Request)
{
    $result = $true
	# Check the class of the object to be created or modified
	if($Request.Class -eq "user")
	{
            $result =$false
            # Retrieve the list of the valid department names
            $DepNames = GetDepartmentNames $Request
            # Retrieve the supplied value of the department property
            $Department = $Request.Get("department")
            if($Department -ne $null)
            {
                # Check the supplied value
                foreach($UserDep in $DepNames)
                {
                    if($Department -eq $UserDep)
                    {
                        $result = $true
                        break;
                    }
    	          }
	      }
	}
    $result 
}
# onGetEffectivePolicy event handler
function onGetEffectivePolicy($Request)
{
    if($Request.Class -ne "user"){return}
        $Attrname = "department"
    $arr = GetDepartmentNames $Request
	# Call the SetEffectivePolicyInfo method
    $Request.SetEffectivePolicyInfo($Attrname, $Constants.EDS_EPI_UI_DISPLAY_NOTE, "The value is offered by Administrative Policy")
	# This method displays the Values tab that contains a list of possible department names
    $Request.SetEffectivePolicyInfo($Attrname, $Constants.EDS_EPI_UI_POSSIBLE_VALUES, $arr)
}
# onCheckPropertyValues event handler
function onCheckPropertyValues($Request)
{
$result =ValidateDepartment $Request
$eventlog.ReportEvent(2, "enter test 1", $result)
    if($result  -eq  $false)
    {
	$Request.SetPolicyComplianceInfo("department", $Constants.EDS_POLICY_COMPLIANCE_ERROR, "The value of the department property does not meet the corporate policy.")
    }
}
