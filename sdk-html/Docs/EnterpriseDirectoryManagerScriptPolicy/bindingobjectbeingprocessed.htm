Binding to the Object Being Processed

Since the object Request acts as a property bag for the in-process object data, it can only be used to access the properties that are involved in the request being handled by Administration Service. If you need to retrieve or modify other properties of the object being processed, you can use the system ADSI provider to bind to that object and access the directory object data.

After you have bound to the processed object, you can retrieve and modify the object properties using the system ADSI provider. For information about how to use the system ADSI provider, please refer to the MSDN Library.

To bind to the requested directory object, use the object DirObj or the GetObject function, as shown in the following examples.

Note:

Whenever possible, we recommend that you use DirObj object.  For more information, refer to DirObj object.

Examples

The following code snippets make the Department property of the moved user object be identical with the name of the destination container. This sample script employs the server-defined variable TargetContainer that identifies the distinguished name of the destination container folder.

[VBScript]
Sub onPreMove (Request)
    If Request.Class <> "user" Then Exit Sub
     ' Bind to the destination container
    Set ou = GetObject("EDMS://" + Request.Parameter("TargetContainer"))
     ' Set the user department
    DirObj.Put "Department", ou.Get("name")
     ' Commit changes
    DirObj.SetInfo
End Sub




[PowerShell]
function onPreMove($Request)
{
	If($Request.Class -ne "user"){return}
	
	# Bind to the destination container
	$ou = Get-QADObject $Request.Parameter("TargetContainer") 
		
	# Set the user department
    $DirObj.Put("Department", $ou.Name)
	
	# Commit changes
    $DirObj.SetInfo()
}

The following code snippet enforces the following policy: the user's manager must belong to the user's department.

[VBScript]
Sub onPreModify (Request)
   Const strError = "Manager must belong to the user's department."
   Const EDS_EVENTLOG_WARNING_TYPE = 2
   If StrComp(Request.Class, "user", 1) = 0 And _
       Request.Get("manager") <> "" Then
        ' User's Manager property modification requested




        ' Bind to the processed user object




        Dirobj.GetInfo
         ' Bind to the user's manager
        Set objManager = GetObject ("EDMS://" + Request.Get("manager"))
        objManager.GetInfo
        If Dirobj.Department <> objManager.Department Then
            ' The user's department does not match the manager's department
           Err.Raise 1, "Administrative Policy", strError
        End If
   End If
End Sub




[PowerShell]
function onPreModify($Request)
{
   $Error = "Manager must belong to the user's department."
   if(($Request.Class -eq "user") -and ($Request.Get("manager") -ne $null))
   {
      # User's Manager property modification requested
      # Bind to the processed user object
      $Dirobj.GetInfoEx(@("Department"), 0)
      
      # Bind to the user's manager
      $Manager = Get-QADObject $Request.Get("manager")
      if($Dirobj.get("Department") -ne $Manager["Department"])
      {
                # The user's department does not match the manager's department
                throw "Administrative Policy: $Error"
      }
   }
}
