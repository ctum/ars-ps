EDS_EPI_UI_POLICY_RULE

In the methods of the IEDSEffectivePolicyRequest interface, the EDS_EPI_UI_POLICY_RULE type indicates that the Arr argument is a policy rule based on which an Active Roles client application can generate the value of the AttrName property. A policy rule can be a literal value or it can be an instruction to compose a value by combining values of other properties. Such an instruction may use one or more tokens of the following syntax, representing the value of a particular property: %xy<name> where name is the LDAP display name of the property.

Request.SetEffectivePolicyInfo AttrName, EDS_EPI_UI_POLICY_RULE, Arr

Arr Syntax

Type

Description

Syntax

Literal value

The value of the AttrName property is a literal string of character.

Plain text

Dynamic value

Other properties are used to build the value of the AttrName property.

Literal characters combined with tokens of the following syntax:

%xy<name>

  • name stands for the LDAP display name of a certain property. The token indicates that the value of the name property is to be used for building the AttrName property value.

  • y is an optional integer parameter. When y is supplied, the token indicates that not more than y leading characters of the name property value are to be used. If y is omitted, the entire value of the name property is to be used for building the AttrName property value.

  • x is an optional character that complements the y parameter. When x is supplied, the token indicates that exactly y leading characters of the name property value are to be used. The appropriate number of x characters is to be added at the end of the value if the current value of the name property is less than y characters, to ensure that the resulting value is exactly y characters long.

For instance, %*5<displayName> represents an instruction to use exactly 5 leading characters of the display name, appending asterisk characters if needed.

Example

The following policy rule instructs an Active Roles client application to compose the display name of the first name, initial and last name, such as John F. Smith (notice blank spaces between tokens and a period character following the %<initials> token):

Request.SetEffectivePolicyInfo "displayName", EDS_EPI_UI_POLICY_RULE, "%<givenName> %<initials>. %<sn>"
