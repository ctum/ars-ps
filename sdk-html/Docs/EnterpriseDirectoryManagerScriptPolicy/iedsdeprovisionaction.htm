IEDSDeprovisionAction

The IEDSDeprovisionAction interface enables your policy script to define information to be written to the Deprovisioning Results report or the Results of Undo Deprovisioning report.

Note:

All methods and properties of IEDSDeprovisionAction can be used only within the following event handlers: onPreDeprovision, onDeprovision, onPreUnDeprovision, onUnDeprovision.

Properties

The IEDSDeprovisionAction interface has the following properties.

Property

Description

Description

Sets the description of the deprovision/un-deprovision operation.

IsSuccess

Specifies whether the deprovision/un-deprovision operation is successful.

Name

Sets the name for the deprovision/un-deprovision operation.

Methods

The IEDSDeprovisionAction interface defines the following methods.

Method

Description

AddListAttribute

Adds detailed records to the Deprovisioning Results report or the Results of Undo Deprovisioning report.

AddListItem

Adds short records to the Deprovisioning Results report or the Results of Undo Deprovisioning report.

Code Example 1

The following sample policy script illustrates the use of the IEDSDeprovisionAction interface. Upon a request to deprovision a user object, this policy script disables the user account and sets the employeeID property of the user to "000". The script also adds appropriate records to the Deprovisioning Results report for that user.

[VBScript]
Sub onPreDeprovision(Request)
' Deprovisioning the processed user
  DirObj.Put "edsaAccountIsDisabled", True
  DirObj.Put "employeeID", "000"
  DirObj.SetInfo
' Add these records to the Deprovisioning Results report
  Set action = Request.AddDeprovisioningReportRecord 
  action.Name = "The following properties of the object " & DirObj.Name & " have been changed:"
  action.IsSuccess = True
  old_employeeID = CStr(Request.OriginalUserAccount.Get("employeeID"))
  action.AddListAttribute "employeeID", CStr(DirObj.Get("employeeID")), old_employeeID
End Sub




[PowerShell]
function onPreDeprovision($Request)
{
    # Deprovisioning the processed user
    $DirObj.Put("edsaAccountIsDisabled", $true)
    $DirObj.Put("employeeID", "000")
    $DirObj.SetInfo()
    # Add these records to the Deprovisioning Results report
    $action = $Request.AddDeprovisioningReportRecord()
    $action.Name = "The following properties of the object " + $DirObj.Name + " have been changed:"
    $action.IsSuccess = $true
    $old_employeeID = [string]$Request.OriginalUserAccount.Get("employeeID")
    $action.AddListAttribute("employeeID", [string]$DirObj.Get("employeeID"), $old_employeeID)
}

To use this policy script

  1. Prepare the corresponding script module and create a deprovisioning Policy Object that uses this script module. For related procedures, refer to Applying Policy Scripts.

  2. Link the created deprovisioning Policy Object to a container, such as an Organizational Unit.

  3. In the console tree, click that container, and then in the details pane, select any user that resides in the container.

  4. On the Action menu, click Deprovision.

To view the Deprovisioning Results report for a user

  • In the Active Roles snap-in, right-click the user, and then click Deprovisioning Results on the shortcut menu.

The records your policy scripts add to the Deprovisioning Results report or the Results of Undo Deprovisioning report are displayed under the node Script Execution.

Code Example 2

The following sample policy script illustrates the use of the IEDSDeprovisionAction interface. Upon a request to undo deprovisioning of a user object, this policy script enables the user account. The script also adds appropriate records to the Results Undo Deprovisioning report for that user.

[VBScript]
Sub onUnDeprovision(Request)
' UnDeprovisioning the processed user
  DirObj.Put "edsaAccountIsDisabled", False
  DirObj.SetInfo
' Add these records to the Results of Undo Deprovisioning report
  Set action = Request.AddUnDeprovisioningReportRecord 
  action.Name = "User account " & DirObj.Name & " is enabled"
  action.IsSuccess = True
End Sub




[PowerShell]
function onUnDeprovision($Request)
{
    # UnDeprovisioning the processed user
    $DirObj.Put("edsaAccountIsDisabled", $false)
    $DirObj.SetInfo()
    # Add these records to the Results of Undo Deprovisioning report
    $action = $Request.AddUnDeprovisioningReportRecord()
    $action.Name = "User account " + $DirObj.Name + " is enabled"
    $action.IsSuccess = $true
}

To use this policy script

  1. Prepare the corresponding script module and create a undeprovisioning Policy Object that uses this script module. For related procedures, refer to Applying Policy Scripts.

  2. Link the created deprovisioning Policy Object to a container, such as an Organizational Unit.

  3. In the console tree, click that container, and then in the details pane, select any user that resides in the container.

  4. On the Action menu, click Undo Deprovisioning.
