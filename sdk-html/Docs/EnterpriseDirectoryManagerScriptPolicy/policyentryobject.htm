PolicyEntry Object

The object PolicyEntry provides methods that allow you to read the custom (user-defined) parameters of the policy script.

Important:
The PolicyEntry object is deprecated.

Methods

The object PolicyEntry supports the following methods:

  • Parameter Method

  • ParameterEx Method

Parameter Method

The Function procedure that returns the value of the specified single-value parameter of the policy script. 

Object.Parameter(ByVal paramID)

Arguments

Object
Required. Always the PolicyEntry object.
paramID
[in] Required. String. Identifies the custom parameter of the policy script.

ParameterEx Method

The Function procedure that returns the value of the specified multi-value parameter of the policy script. 

Object.ParameterEx(ByVal paramID)

Arguments

Object
Required. Always the PolicyEntry object.
paramID
[in] Required. String. Identifies the custom parameter of the policy script.

Example

The following code snippet illustrates how you can use the object PolicyEntry to access custom parameters of the policy script.

[VBScript]
Sub onPreCreate(Request)
   Dim strDescription, strClass
   strClass = Request.Class
   IF (strClass = "user") Then
      strDescription = PolicyEntry.Parameter("userDescription")
      Request.Put "description", strDescription
   End IF
End Sub




[PowerShell]
function onPreCreate($Request)
{
   $strClass = $Request.Class
   if ($strClass -eq "user")
   {
      $strDescription = [string]$PolicyEntry.Parameter("userDescription")
      $Request.Put("description", $strDescription)
   }
}

This policy script enforces the administrative policy that sets the "description" property of all newly created users to the value stored in the custom parameter userDescription of the policy script.
