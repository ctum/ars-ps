IEDsRequestParameters::GetOutControl

The GetOutControl method returns the value of the specified OutControl.

Function  GetOutControl(ControlID)

Arguments

ControlID

[in] Required. String. Contains the control ID.

Settings

The ControlID argument must identify your custom OutControl (any unique string).

Remarks

If the control in question is not placed in a request, the call of this method will result in an error. For information about how to handle the error, see Remarks in GetInControl method.

Sample Code

This sample policy script illustrates how you can use the GetOutControl method to add the information type entry to the EDM Server event log. The entry contains an ID of the operation to modify a directory object. This operation ID is returned by the OperationID built-in Active Roles control.

[PowerShell]
function onPostModify($Request)
{
$EventLog.ReportEvent($Constants.EDS_EVENTLOG_INFORMATION_TYPE,`
"The " + $DirObj.Name + " object was modified. Operation ID: " + $Request.GetOutControl("OperationID"))
}

This script adds a new entry to the EDM Server event log. The entry is similar to the following text:

Script policy has reported an event.
Operation GUID: 6530df50-4518-4133-8b2e-e34251807a8b
Object name: JNihei
Parent container: dom.lab.local/Research/Employees/Tokyo
Object class: user
Object GUID: 7289403a-7fa9-4b87-a2d0-0afbb9a05dfe
Script Module: GetOutControl
The CN=JNihei object was modified. Operation ID:1-216
