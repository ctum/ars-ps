Developing Filtering Condition Functions

A filter can be used to define any additional conditions on objects involved in a workflow. The workflow starts only if the operation satisfies those conditions. Filtering conditions can be based on properties of objects involved in the operation request, or a custom script can be used to implement complex filtering conditions.

This topic explains how to create a custom script function that implements complex filtering conditions. The function must be saved in a library script module. The syntax of the function is as follows:

Function <FunctionName>($Request)

Arguments

$Request
[in] Required. The intrinsic object Request.

Remarks

Within this function, you can use any interfaces the object Request implements.

Sample Filtering Function

The following sample function retrieves the current day of week, and returns $false if the current day is Sunday or Saturday; otherwise, the function returns $true.

[PowerShell]
function IsWeekend($Request)
{
# --- Retrieve the day of the week - number (1 - Monday, etc.) ---
$DayNumber = Get-Date -uformat "%u"
$IsWeekend = ($DayNumber -ne 6) -and ($DayNumber -ne 7)
$IsWeekend
}

How to Add This Function to Workflow Start Conditions

Use the following steps to add this function to the workflow start conditions.

To add a script filtering function to the workflow start conditions

  1. Create a library script module, and add the script function to that module. For instructions, see Understanding Library Scripts.

  2. In the console tree, expand Configuration| Policies | Workflow, and select the workflow you want to configure.
    This opens the Workflow Designer window in the details pane, representing the workflow definition as a process diagram.

  3. In the Workflow Designer window, click to expand the Workflow options and start conditions area, and then click Configure.

  4. On the Conditions tab in the Workflow Options and Start Conditions dialog box, under Filtering Conditions, click AND group (all of these conditions are true), and then click Insert condition.

  5. Click Configure condition to evaluate, and then click Value generated by rule expression.

  6. In the Configure Rule Expression dialog box, click Add entry, and then click Value generated by script.

  7. In the Configure Entry dialog box, click Select to choose the script module you created in Step 1, and then, under Script function, select the name of the script function you saved in that script module.

  8. Click OK to close the Configure Entry dialog box.

  9. Click OK to close the Configure Rule Expression dialog box.

  10. Under Filtering Conditions, click Define value to compare to, and then click Text string.

  11. In the Configure Entry dialog box, type TRUE in the Text string box, and then click OK.

  12. Click OK to close the Workflow Options and Start Conditions dialog box.

  13. Click Save Changes in the Workflow Designer.
