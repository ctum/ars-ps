Understanding Access to In-Process Request

To provide policy scripts with access to Administration Service in-process data, Active Roles offers an ADSI-compliant scriptable COM object called Request. The object Request contains all the data Administration Service needs to perform the requested operation. The data in the object Request falls into two categories:

  • In-process object data - the attribute values of the directory objects involved in the operation, such as object name; the in-process object data only includes the attribute values specified by the client or policy script

  • Request object data - the request parameters, such as request type (creating, updating, deleting, or moving the object), target container for the "move" operation, new object name for the "rename" operation, and user-defined (custom) parameters

Administration Service creates the object Request prior to the execution of a policy script. The script does not need to call the function CreateObject or GetObject to create or get the object Request.

The object Request implements the following interfaces:

  • IADs- provides policy scripts with access to in-process object data

  • IADsPropertyList - provides policy scripts with access to in-process object data

  • IEDSRequestParameters - provides policy scripts with access to request object data

  • IEDSEffectivePolicyRequest - enables policy scripts to retrieve and modify information about Active Roles Policy constraints and definitions that are effectively applied to the object whose processing has been requested

  • IEDSPolicyComplianceRequest - enables policy scripts to generate the information and error messages to be sent to the Active Roles client. For example, the script can inform the Active Roles client on whether information it supplies is in compliance with Active Roles Policy requirements.

  • IEDSRequestedAttributes - enables policy scripts to control a list of the processed object's properties requested to be retrieved.
