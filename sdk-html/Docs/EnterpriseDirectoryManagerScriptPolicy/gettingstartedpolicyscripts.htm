Getting Started with Policy Scripts

Policy scripts are used to implement procedures that Administration Service activates upon client requests to perform administrative actions such as the creation, modification, or deletion of objects. You can develop policy scripts using any scripting language supported by the Microsoft Windows Script Host, such as Windows PowerShell, VBScript and JScript. 

To write the most efficient policy scripts, you should understand the following key area:

  • Fundamentals of a scripting language (Windows PowerShell, VBScript or JScript) you want to use for writing policy scripts (see, for example, the MSDN Library).

  • Event Handlers - Depending on the functionality of your policy script, it can include one or more event handlers. Each event handler is a function (Sub procedure, as applied to VBScript) that has one from specific names listed in Understanding Event Handlers. 

  • The objectRequest - To provide policy scripts with access to Administration Service in-process data, Active Roles offers an ADSI-compliant scriptable COM object called Request (for more information about the object Request, see Understanding Access to In-Process Request, Request object).

  • Getting and setting in-process property values - Interfaces the object Request implements allow your scripts to access and modify the Administration Service in-process data. For more information, refer to Interfaces to Access In-Process Request.

  • Binding to the Object Being Processed - To access or modify the properties of the object being processed that are not contained in the request to perform an administrative operation, you should bind to that object, and then use the Get/GetEx or Put/PutEx methods of the IADs interface.

  • Generating Run-Time Errors - In policy scripts, to generate run-time errors, use the Raise method of the Err object or the SetPolicyComplianceInfo method provided by the IEDSPolicyComplianceRequest interface. For more information, please refer to Returning Custom Error Messages.
