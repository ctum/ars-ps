Restricting the Scope of Groups

File name: RestrictGroupScope.ps1

Event Handlers: onCheckPropertyValues

This sample script enforces the following policy: Creation of universal groups is not allowed, that is, the script does not allow an administrator to create a security or distribution universal group as well as to make an existing non-universal group universal.

Note

The onCheckPropertyValues event handler is activated upon the following events:

  • Clicking the Next button in the New Object Wizard

  • Clicking the Apply/OK button in the object Properties dialog box

  • Checking an object for policy compliance (for details, refer to "Checking for Policy Compliance" in Active Roles Help.

Thus, this script policy can be used only if you create or modify groups using the Active Roles snap-in or Web Interface. When creating or modifying groups using your custom ADSI scripts, this script policy has no effect.

Demonstrated Techniques

  • How to determine the class of the in-processed object.

  • How to retrieve the object properties specified in the Request object.

  • How to send an error message to an Active Roles client.

How It Works

The script works when you create, modify or check a policy compliance for a group  residing within a directory folder to which the administrative policy is applied.

The script performs the following steps:

  1. Retrieves the in-processed object class specified in the Request object. If this object class is not "group", terminates without raising an error.

  2. Retrieves the value of the "groupType" property for the processed group using the Get method implemented by the Request object.

  3. If the group scope is not Universal, terminates without raising an error. Otherwise, the script displays the "You cannot create Universal groups in this folder" error message.
    To continue, an administrator must specify other group scope, such as "Domain local" or "Global."
