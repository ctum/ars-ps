Altering the parent container of the operation target object

When you configure a workflow, you may need a method that would change the Distinguished Name of the container that holds the target object of the request that is being processed by the workflow (parent container of the request target object). This requirement arises from the need to apply the policies and workflows of an alternative container instead of those of the container originally specified in the request. Another option is to change the container where to create a new object while ensuring that the policies and workflows are applied from the container where the object will actually be created rather than from the container that was originally specified in the object creation request.

To address this requirement, Active Roles offers the ChangeParentDN method which modifies the operation request to set the Distinguished Name of the alternative container for the operation target object. As a result, the request is being processed as if the operation target object were located in the alternative container rather than the container originally specified in the request. When used in a script-based workflow activity, this method ensures that the workflows and policies of the alternative container are applied to the operation target object. For an object creation operation, this method causes the object to be created in the alternative container.

ChangeParentDN method

Use the ChangeParentDN method to set the Distinguished Name of an alternative parent container for the operation target object. This method causes Active Roles to processes the operation request as if the operation target object were located in the alternative container rather than the container originally specified in the request. When used in a script-based workflow activity, ensures that the workflows and policies of the alternative container are applied to the operation target object. For an object creation operation, causes the object to be created in the alternative container.

Syntax

$Request.ChangeParentDN (DNString)

Arguments

DNString

[in] Required. String. Specifies the Distinguished Name of the parent container to set.

Examples

The following script function causes Active Roles to create the object in the container whose Distinguished Name is specified by the workflow parameter named Container. This script function is intended to be used within a script activity of a workflow that would be triggered by the object creation request.

Function ChangeCreationContainer($Request)

{

    $request.ChangeParentDN($workflow.Parameter("Container"))

}

 

The following example illustrates the use of the ChangeParentDN method in the onGetEffectivePolicy event handler to retrieve the policy settings that are in effect on the request target object as if it were in the container with the Distinguished Name specified by the workflow parameter Container. If you do not use the ChangeParentDN method, the event handler shows the policy settings that the operation target object receives from its parent container originally specified in the request. This example applies to script-based workflow activities as the ChangeParentDN method has no effect on the policy settings when used within a script-based policy.

Function onGetEffectivePolicy($Request)

{

    $request.ChangeParentDN($workflow.Parameter("Container"))

}
