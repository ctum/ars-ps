IADsPropertyList

The IADsPropertyList interface provides all standard methods for enumerating the attribute entries in the Request object. Since the Request objectis ADSI-compliant, its IADsPropertyList interface conforms to the specification of properties and methods outlined in IADsPropertyList ( Active Roles ADSI Provider).

Examples of Policy Scripts (VBScript, PowerShell)

This section provides two sample policy scripts that illustrate how to use properties and methods of the IADsPropertyList and IADsPropertyEntry interfaces.

Write Changes to the Log File

The following policy script writes all the in-process property values into the specified log file.

[VBScript]
Sub onPreModify(Request)
 str = vbCrLf
 str = "Property values modified in the directory object" + vbCrLf
 str = str + "Object DN: " + Request.Name + vbCrLfEnd
 str = str + "Object type: " + Request.Class + vbCrLf + vbCrLf
  ' Retrieve properties from in-process data
 For i=0 To Request.PropertyCount-1
    Set item = Request.Item(i)
    str = str + "Property name: " + item.Name
    str = str + ", Property value(s): "
 ' Retrieve Property values
  For Each value In item.Values
    Select Case value.ADsType
        Case ADSTYPE_DN_STRING
            str = str + value.DNString + "// "
        Case ADSTYPE_CASE_EXACT_STRING
            str = str + value.CaseExactString + "// "
        Case ADSTYPE_CASE_IGNORE_STRING
            str = str + value.CaseIgnoreString + "// "
        Case ADSTYPE_PRINTABLE_STRING
            str = str + value.PrintableString + "// "
        Case ADSTYPE_NUMERIC_STRING
            str = str + value.NumericString + "// "
        Case ADSTYPE_BOOLEAN
            str = str + CStr (value.Boolean) + "// "
        Case ADSTYPE_INTEGER
            str = str + CStr (value.Integer) + "// "
    End Select
  Next
    str = str + vbCrLf
 Next
' Write output into log file
 Set fso = CreateObject("Scripting.FileSystemObject")
 Set MyFile = fso.OpenTextFile("c:\EDMLog.txt", 8, True)
 MyFile.WriteLine(str)
 MyFile.Close
End Sub




[PowerShell]
function onPreModify($Request)
{
  $str += [System.Environment]::NewLine + "Property values modified in the directory object" + [System.Environment]::NewLine
  $str += "Object DN: " + $Request.Name + [System.Environment]::NewLine
  $str += "Object type: " + $Request.Class + [System.Environment]::NewLine + [System.Environment]::NewLine
  # ---- Retrieve properties from in-process data -----
  for($i = 0; $i -lt $Request.PropertyCount; $i++)
   {
    $item = $Request.Item($i)
    $str += "Property name: " + $item.Name
    $str += ", Property value(s): "
    # ----- Retrieve Property values -----
    foreach($value in $item.Values)
      {
        switch($value.Type)
         {
           $Constants.DSTYPE_DN_STRING {$str += [string]$value + "// "}
           $Constants.ADSTYPE_CASE_EXACT_STRING {$str += [string]$value + "// "}
           $Constants.ADSTYPE_CASE_IGNORE_STRING {$str += [string]$value + "// "}
           $Constants.ADSTYPE_PRINTABLE_STRING {$str += [string]$value + "// "}
           $Constants.ADSTYPE_NUMERIC_STRING {$str += [string]$value + "// "}
           $Constants.ADSTYPE_BOOLEAN {$str += [string]$value + "// "}
           $Constants.ADSTYPE_INTEGER {$str += [string]$value + "// "}          }
      }
     $str += [System.Environment]::NewLine
   }
  # ----- Write output into log file ----
  [System.IO.File]::AppendAllText("c:\EDMLog.txt", $str)
}

After you link a Policy Object that uses this script to a directory folder (for related procedures, refer to Applying Policy Scripts), the file c:\EDMLog.txt will contain values and names of the object's properties you have changed. The policy will affect any object that resides in the directory folder to which the Policy Object has been linked.

Prohibit Removing the Group Members

The following script allows you to enforce the administrative policy that prevents an administrator from removing members from groups affected by that policy.

[VBScript]
Sub onPreModify(Request)
    If Request.Class <> "group" Then Exit Sub
        For i=0 To Request.PropertyCount-1
           Set item = Request.Item(i)
           If item.Name = "member" Then
              ItemCC=CLng(item.ControlCode)
              If ItemCC=ADS_PROPERTY_CLEAR Or ItemCC=ADS_PROPERTY_DELETE Then
                  Err.Raise -1, "Administrative Policy",_
                   "Removing members from this group is denied by policy."
           End If
    End If
        Next
End Sub




[PowerShell]
function onPreModify($Request)
{
  if($Request.Class -ne "group"){return}
  for($i = 0; $i -lt $Request.PropertyCount; $i++)
   {
     $item = $Request.Item($i)
     if($item.Name -eq "member")
      {
        $Operation = $item.ControlCode
        if(($Operation -eq $Constants.ADS_PROPERTY_CLEAR) -or ($Operation -eq $Constants.ADS_PROPERTY_DELETE))
         {
           throw ("Administrative Policy:" + [System.Environment]::NewLine + "Removing members from this group is denied by policy.")
         }
      }
   }
}

After you link a Policy Object that uses this script to a directory folder, an administrator trying to remove members from any group residing in that folder will be presented with the following error message:
"Administrative Policy: Removing members from this group is denied by policy."
Note that this policy allows an administrator to add new members to groups.
