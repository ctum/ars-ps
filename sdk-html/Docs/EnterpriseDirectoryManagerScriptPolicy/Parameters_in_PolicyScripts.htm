Declaring Parameters in Policy Scripts

You can declare parameters directly in the policy script using the onInit function (Sub procedure as applied to VBScript) or using any function whose name you specify in the edsaApeListXML attribute of a edsPolicyObject object. For each parameter, the declaration defines the name and syntax of the parameter and it may define a number of other characteristics, such as a description, a list of acceptable values, the default value, and whether a value is required. You can modify the parameters values with both custom ADSI scripts (see Modifying Parameters) and the Active Roles MMC snap-in.

The OnInit Function

To declare parameters in your policy script, define the onInit function. The onInit function has the following syntax:

[VBScript]
Sub onInit(byRef  Context)




[PowerShell]
function onInit($Context) 
Arguments
Context

[in] Required. A reference to an object that stores definitions for all parameters the policy script includes. For details, refer to The Context Object.
