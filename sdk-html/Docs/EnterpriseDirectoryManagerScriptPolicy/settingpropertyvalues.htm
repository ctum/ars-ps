Setting In-Process Property Values

The object Request supports the IADs interface that allows you to get and modify the in-process property values of the directory object being processed (in-process object data). Here the object Request acts as a property bag used to store the property values that are specified by the client or a policy script. Unless a property value is set in the object Request, the value cannot be accessed.

The object Request is created by Administration Service based on the data the service retrieves from its client's request. For example, the client may request to modify certain properties of a directory object. When a client sends a request, it specifies the property values to be set. The object Request contains all those values, allowing your script to access and modify them.

Note:

The object Request can only be used to access the in-process attribute values. If you need to get or modify other attribute values of the object being processed, you can use the system ADSI provider to access the object instance stored in Active Directory (stored object data).

You can modify in-process object data using the Put or PutEx method of the object Request. Each of these methods allows you to set the value of a specific property by name, modifying data stored by the object Request. Administration Service commits the changes at the final stage of processing of the client's request. You do not need to use the SetInfo method for the changes made to in-process object data to take effect.

Since the object Request is ADSI-compliant, the syntax of its Put and PutEx methods conforms to the ADSI specification.

  • Single-value properties can be modified using the Put method of the object Request; for information about how to use the Put method of any ADSI-compliant object, please refer to the MSDN Library

  • Multi-value properties can be modified using the PutEx method of the object Request; for information about how to use the PutEx method of any ADSI-compliant object, please refer to the MSDN Library

Unlike the PutEx method defined in the IADs interface supported by the ADSI LDAP provider, the Request::PutEx method does not allow you to add or remove values from the multi-valued in-process properties. This method always overwrite existing in-process values with those you pass to the method (third argument of PutEx). To clarify this, consider the following examples. 

[VBScript]
' ----------------------------------------------------------
' Assume the in-process otherTelephone has the values
' 129-0505, 129-0508
' ----------------------------------------------------------
Request.PutEx ADS_PROPERTY_APPEND,"otherTelephone",Array("129-0000","129-0001")
 ' Now the in-process otherTelephone has the values 129-0000, 129-0001


' ----------------------------------------------------------
' Assume the in-process otherTelephone has the values
' 129-0505, 129-0508
' ----------------------------------------------------------
Request.PutEx ADS_PROPERTY_DELETE,"otherTelephone",Array("129-0505")
 ' Now the in-process otherTelephone has the value 129-0505




 
[PowerShell]
# ----------------------------------------------------------
# Assume the in-process otherTelephone has the values
# 129-0505, 129-0508
# ----------------------------------------------------------
$Request.PutEx($Constants.ADS_PROPERTY_APPEND, "otherTelephone", @("129-0000","129-0001"))
# Now the in-process otherTelephone has the values 129-0000, 129-0001
# ----------------------------------------------------------
# Assume the in-process otherTelephone has the values
# 129-0505, 129-0508
# ----------------------------------------------------------
$Request.PutEx($Constants.ADS_PROPERTY_DELETE, "otherTelephone", @("129-0505")
# Now the in-process otherTelephone has the value 129-0505

Example Code

The following code snippetы show how to modify values of the in-process multi-valued property otherTelephone for a user object to be created. This policy script adds the phone number 129-0000 to the list of the user's other phone numbers specified in the request to create a user object.

The script performs the following steps:

  1. Retrieves the list of the user's other phone numbers specified in the request to create a user, if any.

  2. If the list contains no data, sets the in-process property otherTelephone to 129-0000 and terminates without raising an error.

  3. Calculates the number of the phone numbers the list includes, declares the array Phones that contains that list, and then increases the array's size by 1.

  4. Adds 129-0000 to the array Phones, and then makes the array Phones new value of  otherTelephone.

[VBScript]
Sub onPreCreate(Request)
 ' This policy script adds 129-0000 to the user's other phone
 ' numbers list specified in the request to create a user
 If Request.Class <> "user" Then Exit Sub
 
On Error Resume Next
    ' Get user's other phone numbers from the request
   Phones = Request.GetEx("otherTelephone")
   If VarType(Phones) = vbEmpty Then
       Request.PutEx ADS_PROPERTY_UPDATE, "otherTelephone", Array("129-0000")
       Exit Sub
   End If           
   N = 0
    ' Calculate the number of the in-process phone numbers 
   For Each Phone In Phones
        N = N +1
   Next
   ReDim Preserve Phones(N)
    ' Add the number 129-0000 to all in-process phone numbers
   Phones(N)="129-0000"
    ' Modify the in-process phone numbers
  Request.PutEx ADS_PROPERTY_UPDATE, "otherTelephone", Phones
  
End Sub




[PowerShell]
function onPreCreate($Request)
{
   # This policy script adds 129-0000 to the user's other phone
   # numbers list specified in the request to create a user
   if($Request.Class -ne "user"){return}
 
   # Get user's other phone numbers from the request
   $Phones = $Request.GetEx("otherTelephone")
   if($Phones -eq $null)
   {
      $Request.PutEx($Constants.ADS_PROPERTY_UPDATE, "otherTelephone", @("129-0000"))
      return
   }
   $list = New-Object System.Collections.Generic.List[string]
   foreach ($phone in $phones)
   {
   $list.Add($phone);
   }
    # Add the number 129-0000 to all in-process phone numbers
   $list.Add("129-0000")
   $Request.Put("otherTelephone", $list.ToArray())
}
