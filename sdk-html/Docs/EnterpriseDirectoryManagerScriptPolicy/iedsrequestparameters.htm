IEDSRequestParameters

The IEDSRequestParameters interface enables policy scripts to access data that is specific to the object Request.

Note:

All methods and properties described in this topic, except the LdapFilter, FilterSearchResults and SearchPref properties, can be used in any event handler. These properties are only accessible from the onPreSearch event handler.

Properties

The IEDSRequestParameters interface has the following properties.

Property

Description

FilterSearchResults

Allows you to filter the search results for certain Active Roles clients.

LdapFilter

Sets and returns the Ldap search filter that the ADSI Provider uses to search for directory objects. The filter specifies the objects to be found according to the object classes, names or properties.

Parameter

Sets and returns values of the server-defined or user-defined variables in the in-process (request) object. 

SearchPref

Sets and returns the search preferences.

Methods

The IEDSRequestParameters interface defines the following methods.

Method

Description

OverridePolicy

Allows you to disable the specified event handler within the specified script policy.

ReportEvent

Allows policy scripts to add events to the EDM Server Log.

WhoAmI

Allows you to get the distinguished name and pre-Windows 2000 name of the security principal in whose security context the Active Roles client that sends the request is running.

Methods Used to Process Active Roles Controls

Active Roles implements special parameters called  Active Roles controls that allow you to customize the request processing. 

The Active Roles client can send a control to Administration Service with a request to perform an administrative operation. Administration Service can process that control.  On the other hand, Administration Service can return its own control to the request source (client) and the client can process that control. The controls a client sends to Administration Service are referred to as InControls while the controls Administration Service returns to the client are referred to as OutControls. For more information, refer to Active Roles controls.

The following methods of IEDsRequestParameters are designed to process the Active Roles controls.

Method

Description

GetInControl

Returns the value of the specified InControl.

GetOutControl

Returns the value of the specified OutControl.

PutInControl

Sets the value of the specified InControl.

PutOutControl

Sets the value of the specified OutControl.
