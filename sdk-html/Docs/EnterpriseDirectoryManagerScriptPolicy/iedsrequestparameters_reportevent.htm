IEDsRequestParameters::ReportEvent

The ReportEvent method allows you to record events to the EDM Server Log from a policy script. The method writes an entry at the end of the EDM Server Log. 

Sub  ReportEvent(ByVal eventType, ByVal Msg1[,Msg2,Msg3])

Arguments

eventType
[in] Required. Identifies the type of the event to be logged.
Msg1
[in] Required. Contains the user-defined message to be recorded into the EDM Server Log.
Msg2
[in] Optional.  Contains the user-defined message to be recorded into the EDM Server Log.
Msg3
[in] Optional. Contains the user-defined message to be recorded into the EDM Server Log.

Settings

The eventType argument can take any from the following values.

Constant

Value

Event Type

EDS_EVENTLOG_SUCCESS  

0

Success event

EDS_EVENTLOG_ERROR_TYPE

1

Error event

EDS_EVENTLOG_WARNING_TYPE

2

Warning event 

EDS_EVENTLOG_INFORMATION_TYPE

4

Information event

EDS_EVENTLOG_AUDIT_SUCCESS

8

Success Audit event

EDS_EVENTLOG_AUDIT_FAILURE

16

Failure Audit event

Example Code

The following code snippet illustrates the use of the ReportEvent method. In this example, the onPostModify event handler allows you to write in the EDM Server Log the report on a modification of the "department" property of the processed user object.  

[VBScript]
Sub onPostModify(Request)
Dim Dept
If Request.Class <> "user" Then Exit Sub
Dept=Request.Get("department")
If (VarType(Dept) <> vbEmpty) Then
   Request.ReportEvent EDS_EVENTLOG_INFORMATION_TYPE, "The department property of the user " + Request.Name, "has been changed"
End If
End Sub  



[PowerShell]
function onPostModify($Request)
 {
 if($Request.Class -ne "user"){return}
 $Dept = $Request.Get("department")
 if ($Dept -ne $null)
 {
   $Request.ReportEvent($Constants.EDS_EVENTLOG_INFORMATION_TYPE, ("The department property of the user " + $Request.Name), "has been changed")
 }
}
