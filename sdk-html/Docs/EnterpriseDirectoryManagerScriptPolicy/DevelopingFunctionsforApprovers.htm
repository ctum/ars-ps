Developing Functions for Designating Approvers

The approval activities let you define who in your organization will be entitled to approve or reject administrative operations that require approval according to the Approval Activities. You can implement a script function designed to dynamically designate approvers for the approval activity. The function must be saved in a library script module. The syntax of the function is as follows:

Function <FunctionName>($Request)

Arguments

$Request
[in] Required. The intrinsic object Request.

Remarks

Within this function, you can use any interfaces the object Request implements. This function returns a string or an array of strings that identifies approvers in one of the following forms:

  • SID string

  • Distinguished name of a user or a group

  • NT4-style user or group name in the form <NTDomain_Name>/<AccountName>

Sample Script Function

The following sample script function retrieves the distinguished name of the in-process user's manager.

[PowerShell]

Function Manager($Request)
{
if ($DirObj.class -eq "user")
{
$DirObj.GetInfoEx(@("manager"),0)
$manager= $DirObj.Get("manager")
$manager
}
}

How to Add This Function to an Approval Activity

Use the following steps to add this function to an approval activity defined in an existing workflow.

To add a custom function designating approvers to an approval activity

  1. Create a library script module, and then save the created function into this module. For related procedures, refer to Understanding Library Scripts.

  2. In the Active Roles console tree, expand Configuration| Policies | Workflow, and select the workflow containing the Approval activity you want to configure.
    This opens the Workflow Designer window in the details pane, representing the workflow definition as a process diagram.

  3. In the process diagram, right-click the name of the Approval activity and click Properties.

  4. In the Properties dialog box, click the Approvers tab, and then click Designate approvers.

  5. In the Approvers Selection dialog box, select the Users or groups specified by script check box, and then click Select.

  6. In the Select Script dialog box, click Select to choose the script module you created in Step 1, and then, under in the Script Function box, type the name of the script function you saved in that script module.

  7. Click OK to close the Select Script dialog box.

  8. Click OK to close the Approvers Selection dialog box.

  9. Click OK to close the Properties dialog box.

  10. Click Save Changes in the Workflow Designer.

Implemented Scenario

In the given example, for each in-process user, the script will designate the user manager as an approver in the approval activity.
