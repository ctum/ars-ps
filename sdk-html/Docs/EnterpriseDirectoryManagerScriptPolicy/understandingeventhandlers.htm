Understanding Event Handlers

A policy script includes one or more event handlers. Each event handler is a function that has one of the following names:

onPreCreate       onPostCreate
onPreDelete       onPostDelete
onPreUnDelete     onPostUnDelete
onPreDeprovision  onPostDeprovision
onPreUnDeprovision onPostUnDeprovision
onPreModify       onPostModify
onPreMove         onPostMove
onPreRename       onPostRename
onPreGet          onPostGet
onPreSearch
onFilter
onCheckPropertyValues
onDeprovision
onUnDeprovision
onGetEffectivePolicy
onInit

All event handlers, except the onFilter and onInit handlers, have the following syntax:

[VBScript]
Sub <event handler name>(Request)
   ' Here goes the event handler code
End Sub




[PowerShell]
function <event handler name>($Request)
{
   # Here goes the event handler code
}

The onFilter event handler has the following syntax:

[VBScript]
Function onFilter(Request)
   ' Returns TRUE or FALSE 
   ' Here goes the event handler code
End Sub




[PowerShell]
function onFilter($Request)
{
    # Returns TRUE or FALSE
    # Here goes the event handler code
}

Arguments

Request

[in] Required. A reference to the intrinsic object Request.

 

The onInit event handler has the following syntax:

[VBScript]
Sub onInit(Context)
   
   ' Here goes the event handler code
End Sub




[PowerShell]
function onInit($Context)
{
    # Here goes the event handler code
}

Arguments

Context

[in] Required. A reference to the intrinsic object Context.

 

Administration Service activates event handlers upon the following occurrences:

  • onPreCreate/onPostCreate: Object creation or copy requested/committed

  • onPreDelete/onPostDelete: Object deletion requested/committed

  • onPreUnDelete/onPostUndelete: Object undeletion (restore) requested/committed

  • onPreDeprovision/onPostDeprovision: User account deprovision operation requested/committed

  • onPreUnDeprovision/onPostUnDeprovision: User account undo deprovisioning operation requested/committed

  • onPreModify/onPostModify: Object updating requested/committed

  • onPreMove/onPostMove: Object moving requested/committed

  • onPreRename/onPostRename: Object renaming requested/committed

  • onPreGet/onPostGet: Retrieving specified properties of Object requested/committed

  • onPreSearch: Searching and retrieving specified properties of Object requested

  • onFilter: For details and sample script, refer to IEDSRequestParameters Properties.

  • onCheckPropertyValues: Validation of property values requested (When you click the Next button in the New Object Wizards or the Apply/OK button in the Properties dialog box)

  • onGetEffectivePolicy: Effective policy settings requested by a client application

  • onDeprovision: Receives control in the course of processing a request to execute the Deprovision operation.

  • onUnDeprovision: Receives control in the course of processing a request to execute the Undo Deprovisioning operation.

  • onInit: This event handler is used for declaring parameters in policy scripts.

Note:

In a script policy applied to a container, the onPostMove event handler is activated after an object is moved (arrives) into that container, and is not activated after an object is moved from (leaves) that container.

When processing a client's request, Administration Service checks if there exists a policy script with a pre-event handler corresponding to the requested action. If so, Administration Service directs the control to that event handler and resumes the request processing after the handler function finishes off.

Pre-event handlers may raise exceptions. In that case, the request processing terminates and an error is returned to the client.

When changes are committed to Active Directory, Administration Service checks if there exists a policy script with a post-event handler corresponding to the committed transaction. If so, Administration Service directs the control to that event handler.

Post-event handlers may raise exceptions. However, this does not terminate the request processing. An error message is recorded into the Application event log.

Sample Code (VBScript and PowerShell)

In the following code snippet, the pre-delete event handler prevents the deletion of User objects and issues an error when the Administration Service receives a request to delete a User. 

[VBScript]
Sub onPreDelete(Request)
    If (Request.Class = "user") Then Err.Raise 1, "Administrative policy", "Deletion is not allowed."
End Sub




[PowerShell]
function onPreDelete ($Request){
   # Here goes the event handler code
   if ($Request.Class -eq "user")  {
        throw "Deletion of users is not allowed";
                                   }
                                }
