Using Parameters in Policy Scripts

The use of parameters lets you create scripts based on information not available to you at design time, such as information obtained from your user or calculated at run time.

To read the custom parameters of policy scripts, use the Parameter method of the Active Roles intrinsic object PolicyEntryas shown in the following sample policy script.

This policy script enforces the administrative policy that sets the "description" property of all newly created users and groups to values stored in the parameters userDescription and groupDescription, respectively.

[VBScript]
Sub onPreCreate(Request)
   Dim strDescription, strClass
   strClass = Request.Class
   Select Case strClass
       Case "user" strDescription = PolicyEntry.Parameter("userDescription")
       Case "group" strDescription = PolicyEntry.Parameter("groupDescription")
       Case Else Exit Sub
   End Select
       Request.Put "description", strDescription   
End Sub




[PowerShell]
function onPreCreate($Request)
{
	$Class = $Request.Class
	switch($Class)
	{
		"user"
		{
			$Description = $PolicyEntry.Parameter("userDescription")
			break
		}
		"group"
		{
			$Description = $PolicyEntry.Parameter("groupDescription")
			break
		}
		default{return}
	}
	$Request.Put("description", $Description)
}




 

  To use this policy script

  1. Register this script as a policy script module on the Administration Service.

  2. Define the custom parameters named userDescription and groupDescription. For information about how to define custom parameters, refer to Specifying Parameters for Policy Scripts.

  3. Create a Policy Object (PO), and add the script policy to the PO.

  4. Link the PO to a directory folder or Managed Unit.
