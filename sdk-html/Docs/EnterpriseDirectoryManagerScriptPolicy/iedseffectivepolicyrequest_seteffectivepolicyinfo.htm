IEDSEffectivePolicyRequest::SetEffectivePolicyInfo

The SetEffectivePolicyInfo method allows you to set and modify the policy constraints defined in the object Request for the specified attribute of the processed object.

Sub SetEffectivePolicyInfo(byVal AttrName, byVal Type, byVal Arr)

Arguments

AttrName
[in] Required. Ldap display name of the attribute to which you want to apply the policy definitions and constraints. 
Type
[in] Required. Identifies the type of the policy definitions you want to apply.
Arr
[in] Required. String or an array that depends on the policy type. For details, see the Description column in the EDS_EFFECTIVE_POLICY_ENUM table.

Settings

The possible values for the Type argument are listed in the EDS_EFFECTIVE_POLICY_ENUM enumeration.

Example Code

The following script enforces an administrative policy that automatically sets the "department" value in the Advanced Properties dialog box to "Sales" when creating a new user account.

[VBScript] 
Sub onGetEffectivePolicy(Request)
  If Request.Class <> "user" Then Exit Sub
  Dim arr, strAttrname
 ' Set LDAP display name of the desired attribute
  strAttrname="department"
  arr="Sales"
 ' Call the SetEffectivePolicyInfo method
  Request.SetEffectivePolicyInfo strAttrname,EDS_EPI_UI_DISPLAY_NOTE,_
   "The value is generated by administrative policy"
  Request.SetEffectivePolicyInfo strAttrname,EDS_EPI_UI_GENERATED_VALUE,arr
End Sub




[PowerShell] 
function onGetEffectivePolicy($Request)
{
  if($Request.Class -ne "user"){return}
 # Set LDAP display name of the desired attribute
  $strAttrname = "department"
  $arr = "Sales"
 # Call the SetEffectivePolicyInfo method
  $Request.SetEffectivePolicyInfo($strAttrname, $Constants.EDS_EPI_UI_DISPLAY_NOTE, "The value is generated by administrative policy")
  $Request.SetEffectivePolicyInfo($strAttrname, $Constants.EDS_EPI_UI_GENERATED_VALUE, $arr)
}

The following script enforces an administrative policy that offers the specified values for the user's department. An administrator can select those values from a list  when creating a new user account with the Active Roles Web Interface. 

[VBScript]
Sub onGetEffectivePolicy(Request)
  If Request.Class <> "user" Then Exit Sub
  strAttrname="department"
  arr=Array("Sales","Software","HR")
 ' Call the SetEffectivePolicyInfo method
  Request.SetEffectivePolicyInfo strAttrname,EDS_EPI_UI_DISPLAY_NOTE,_
    "The value is offered by administrative policy"
  Request.SetEffectivePolicyInfo strAttrname,EDS_EPI_UI_POSSIBLE_VALUES,arr
End Sub




[PowerShell]
function onGetEffectivePolicy($Request)
{
  if($Request.Class -ne "user"){return}
  $strAttrname = "department"
  $arr = @("Sales","Software","HR")
 # Call the SetEffectivePolicyInfo method
  $Request.SetEffectivePolicyInfo($strAttrname, $Constants.EDS_EPI_UI_DISPLAY_NOTE, "The value is offered by administrative policy")
  $Request.SetEffectivePolicyInfo($strAttrname, $Constants.EDS_EPI_UI_POSSIBLE_VALUES, $arr)
}

The following script enforces an administrative policy that defines a set of forbidden characters for the user's home phone number.

[VBScript]
Sub onGetEffectivePolicy(Request)
If Request.Class <> "user" Then Exit Sub
strAttrname="homePhone"
' Call the SetEffectivePolicyInfo method
arr="-()"
Request.SetEffectivePolicyInfo strAttrname,EDS_EPI_UI_PROHIBITED_SYMBOLS,arr
End Sub

[PowerShell]
function onGetEffectivePolicy($Request)

{
if($Request.Class -ne "user"){return}
$strAttrname = "homePhone"
# Call the SetEffectivePolicyInfo method
arr = "-()"
$Request.SetEffectivePolicyInfo($strAttrname, $Constants.EDS_EPI_UI_PROHIBITED_SYMBOLS, $arr)
}
