IEDsRequestParameters::PutInControl

The PutInControl method allows you to set the value of specified InControl.

Sub  PutInControl(byVal ControlID,byVal ControlSynt,byVal ControlValue)

Arguments

ControlID
[in] Required. String. Contains the control ID.
ControlSynt
[in] Required. Integer. Identifies the data type of the control value. This argument can take any value from the ADSTYPEENUM enumeration (see the MSDN Library). For all Active Roles built-in controls, set this argument to ADSTYPE_CASE_IGNORE_STRING.
ControlValue
[in] Required. Contains the control value you want to set. It can be both a single value and an array of values.

Settings

For built-in controls, the ControlID argument can take any value from the Constant or Control ID column of this table. For your custom InControl, set this argument to the control name (any unique string).

Sample Code

The following policy script illustrates how you can set the value of the EDS_CONTROL_TREE_DELETE built-in control.

[VBScript]
Sub onPreDelete(Request)
Request.PutInControl EDS_CONTROL_TREE_DELETE,ADSTYPE_CASE_IGNORE_STRING, Array("any")
End Sub

If you apply the Policy Object that includes this script policy to a container object, such as an Organizational Unit, you will be able to delete that container with the Active Roles snap-in, even if the container has child objects. Note that you may set the control EDS_CONTROL_TREE_DELETE to any string value ("any" in this example).

The following policy script illustrates how you can set the value of the InheritInitiatorInfo built-in control.

[PowerShell]
function onPreDelete($Request)
{
 $Request.PutInControl("InheritInitiatorInfo",$Constants.ADSTYPE_CASE_IGNORE_STRING,"1")
 #
 # ---- your script code goes here ---
 #
}

If you apply the Policy Object that includes this script policy to an object, the Administration Service treats the request to delete the object as if it were initiated by the client that requested the operation that caused the script to execute. As a result, the requested changes are properly recorded in the management history trail (change history), and are submitted for approval if this is required by the approval rules that are in effect.
