Understanding Scheduled Tasks

Active Roles provides scheduled tasks. The term scheduled task refers to a script the Active Roles Administration Service executes at scheduled time. When executed by the Administration Service, the scheduled task script can use all interfaces supported by the Active Roles ADSI provider as well as the Task, EventLog, and Security objects exposed by the Administration Service. You can develop scheduled task scripts using any scripting language supported by the Microsoft Windows Script Host, such as Windows PowerShell, VBScript and JScript.

When to Use Scheduled Tasks

Use scheduled tasks if you need to regularly start your custom scripts at specific time.

Scheduled tasks are especially useful if you want to regularly monitor the state of the AD objects or state of the replication group (see Notifying Administrator When Failing the Replication). You can specify the schedule type for your task, such as Daily, Weekly, When the AR Service Starts, etc.

Storing Scheduled Tasks

Active Roles stores scheduled tasks in the Active Roles configuration database as objects of the class edsScheduledTask. Those objects reside within the Scheduled Tasks configuration container. The container's DN: 

CN=Scheduled Tasks,CN=Server Configuration,CN=Configuration

Each object of the edsScheduledTask class stores the following information related to the scheduled task:

  • The task schedule

  • The GUID of the scheduled task script associated with this task

  • Optionally, a set of parameters that can control the execution of the script. For more information, see Parameterizing Script Policies and Scheduled Tasks. 

Active Roles stores scheduled task scripts in configuration database as script modules (objects of the edsScriptModule class). The script modules reside within the Script Modules configuration container. The container's DN: 

CN=Script Modules,cn=Configuration

For scheduled task script modules, the attribute edsaScriptType is set to 1.

Since the scheduled task objects and scripts are saved in the configuration database, they will be available for all Administration Services participating in the Active Roles replication.

Creating Scheduled Tasks

To create a scheduled task, perform the following steps:

  1. Create a script you want to associate with the scheduled task, and then register that script with the Administration Service as a scheduled task script module.

  2. Create a scheduled task using the New Scheduled Task wizard provided by the Active Roles snap-in. The wizard allows you to specify the schedule type and the script module you want the Administration Service to run.

Example

This sample script (PowerShell) can be used to create a scheduled task designed to monitor the health of Exchange services related to Exchange Servers installed in managed domains.

The scheduled task implements the following scenario: at the scheduled time, the script monitors the health (status) of all found Exchange services, and adds to the EDM Server Log an error event containing a list of Exchange services that are not running.

Important

Before using this sample script, ensure that you have installed Exchange Management Tools on the computer running Active Roles Administration service.

 

[PowerShell]

$snapin= 'Microsoft.Exchange.Management.PowerShell.admin' #- Exchange 2007 snap-in

#$snapin= 'Microsoft.Exchange.Management.PowerShell.E2010' #- Exchange 2010 snap-in

# --- Load Microsoft Exchange 2007/2010 snap-in  

Add-PSSnapin  $snapin  -ErrorAction SilentlyContinue

$eventlog.ReportEvent($Constants.EDS_EVENTLOG_INFORMATION_TYPE, "The Exchange services health checking started.")

# --- Retrieve the list of Exchange Servers installed in the managed domains. ---

$ExchangeServers = Get-QADObject -type msExchExchangeServer | %{$_.Name}

# --- Add the list of the found Exchange Servers to the EDM Server Log. ---

$strReport = [string]@($ExchangeServers).count + "  Exchange Servers found in the managed domains: `n" +`

       [string]::Join(", ", $ExchangeServers)

$eventlog.ReportEvent($Constants.EDS_EVENTLOG_INFORMATION_TYPE, $strReport)

# --- Enumerate Exchange services related to each found Exchange Server ---

foreach ($server in $ExchangeServers)

{

# --- For each Exchange Server, get a list of services that are stopped. ---

$services_not_running = [string]::Join(", ", (Test-ServiceHealth $server | %{$_.ServicesNotRunning}))

$eventlog.ReportEvent($Constants.EDS_EVENTLOG_ERROR_TYPE,`

    "ATTENTION! The following services required for Exchange tasks are not running on the Exchange Server "`

    + [string]$server + ":`n" + $services_not_running)

}

 

For a sample of a VBScript script you can use for your sample scheduled task, see Using Parameters in Scheduled Task Scripts.
