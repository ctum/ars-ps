Example: Creating bound list-boxes

This topic demonstrates how to implement a scenario where a list of possible values for a given attribute entry in the Web Interface changes depending on the value selected for another attribute. We are going to create and deploy a script-based policy that

  • Defines a list of possible values for the Department (department) attribute along with an individual list of possible values for the Job Title (title) attribute specific to each possible value of the Department attribute, and

  • Causes the Web Interface to display the appropriate "Job title" list when you select a value from the "Department" list on the "Organization" page for a user account.

To establish the dependency between the two attributes we will use the new policy setting introduced in Active Roles version 6.9 - EDS_EPI_UI_RELOAD_EPI_BY_RULE. This policy setting directs the Active Roles Web Interface to reload policy information for the target attribute every time the rule attribute changes on the Web Interface page, thereby enabling the use of different policy rules depending on the value of the rule attribute. With a policy script that defines a separate list of the target attribute's possible values for each possible value of the rule attribute, this policy setting makes it possible for the Web Interface to apply the appropriate list of possible values to the target attribute entry when a particular value is selected for the rule attribute. As applied to the scenario covered by this topic, the target attribute is Job Title (title) and the rule attribute is Department (department).

The example discussed in this topic assumes the following lists of possible values:

Department

Possible Job Title

Customer Service

  • CS Representative

  • CSR Manager

Development

  • SD Engineer

  • SD Manager

Quality Control

  • QC Engineer

  • QC Manager

Sales

  • Sales Manager

Our goal is to configure Active Roles so that the Web Interface lets you select a department from the list and then displays the list of job titles for the department you have selected, allowing you to select the appropriate job title.

The configuration steps are as follows.

  1. Create a PowerShell script module containing the following policy script:

$listDepartments = @('Development', 'Quality Control', 'Sales', 'Customer Service')

$listTitlesDevelopment = @('SD Engineer', 'SD Manager')

$listTitlesQC = @('QC Engineer', 'QC Manager')

$listTitlesSales = @('Salesperson', 'Sales Manager')

$listTitlesCustomerService = @('CS Representative', 'CSR Manager')

$listAllTitles = $listTitlesDevelopment

$listAllTitles += $listTitlesQC | Where-Object {$listAllTitles -notcontains $_}

$listAllTitles += $listTitlesSales | Where-Object {$listAllTitles -notcontains $_}

$listAllTitles += $listTitlesCustomerService | Where-Object {$listAllTitles -notcontains $_}

function onGetEffectivePolicy($Request)

{

    if($Request.Class -eq "user")

    {

        $Request.SetEffectivePolicyInfo('title', $Constants.EDS_EPI_UI_RELOAD_EPI_BY_RULE, 'department')

        $Request.SetEffectivePolicyInfo('department', $Constants.EDS_EPI_UI_POSSIBLE_VALUES, [string[]]$listDepartments)

        $department = GetCurrentValue -Request $Request -AttributeName 'department'

        switch($department)

        {

            'Development'         {$possibleValues = $listTitlesDevelopment        }

            'Quality Control'     {$possibleValues = $listTitlesQC                 }

            'Sales'               {$possibleValues = $listTitlesSales              }

            'Customer Service'    {$possibleValues = $listTitlesCustomerService    }

            default               {$possibleValues = $listAllTitles                }

        }        

        $Request.SetEffectivePolicyInfo('title', $Constants.EDS_EPI_UI_POSSIBLE_VALUES, [string[]]$possibleValues)                 

    }

}

function GetCurrentValue($Request, [string]$AttributeName)

{   

    trap {continue}

    $value = $Request.Get($AttributeName)

    if($value -eq $null)

        {

        $DirObj.GetInfoEx(@($AttributeName),0) | Out-Null

        $value = $DirObj.Get($AttributeName)

        }

    $value

}

  1. Create a Provisioning Policy Object containing a script execution policy based on the script module you created in Step 1.

  2. Apply that Policy Object to the scope (organizational unit, domain or managed unit) that holds the user accounts whose Department and Job Title you want to manage by using this policy.

After you have completed these configuration steps, you can verify that the Web Interface performs as expected. Select a user account from the scope to which you applied the Policy Object, go to the "General Properties/Organization" page, and select a department from the "Department" list. Confirm that the "Job title" list includes only those job titles that apply to the department you selected.
