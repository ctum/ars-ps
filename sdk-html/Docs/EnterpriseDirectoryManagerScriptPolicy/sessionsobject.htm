Sessions Object

The object Sessions allows you to get information about all active Active Roles client's sessions. Active Roles uses the intrinsic object Session to store information about each client's session.  The object Session can be accessed using the appropriate properties of the object Sessions.

Before using the Sessions object in your scripts, obtain the reference to this object using the Sessions property of the Security object as shown the following code snippet:

Set objSessions = Security.Sessions

Properties and Methods

The object Sessions supports the following methods:

  • _NewEnum Property

  • Item Property

  • Count Property

  • Current Property

  • Refresh Method

_NewEnum Property

Read-only. Returns an interface on an enumerator object.

Note:

In Visual Basic and VBScript, the For Each ... In ... Next statement invokes this property implicitly.

Applying the For Each ... In ... Next statement to a Sessions object allows you to enumerate all members in the collection of active sessions the object Sessions stores.

Remarks

The following code snippet illustrates how to enumerate all active sessions.

[VBScript]
Set objSessions = Security.Sessions
For Each objSession in objSessions
    '
    ' Do something with the objSession object
    '
Next




 




[PowerShell]




function onPreModify($Request)
{
    $objSessions = $Security.Sessions
    foreach($objSession in $objSessions)
    {
        #
        # do something with the objSession object
        #
    }
}

Item Property

Read-only. Returns a reference to the Sessionobject by its number in the collection of active sessions. 

Object.Item(ByVal Ind)

Arguments

Object
Required. Always the Sessions object.
Ind
[in] Required. Long integer. Specifies the Session object's number in the collection of sessions.

Count Property

Read-only. Returns the number of the client's sessions in the collection of sessions the object Sessions stores.

Object.Count

Arguments

Object
Required. Always the Sessions object.

Back to top

Current Property

Read-only. Returns the reference on the object Session representing the current session. The current session is the session in which context the policy script is running.

Object.Current

Arguments

Object
Required. Always the Sessions object.

Note:

The Current property returns NOTHING when using in the scheduled task scripts.
