IEDsRequestParameters::OverridePolicy

The OverridePolicy method allows you to disable the specified event handler within the specified script policy.

Sub  OverridePolicy(byVal PolicyID)

Arguments

PolicyID

[in] Required. Identifies the policy within which you want to override an event handler.

Remarks

When you call this method within an event handler, only the event handler of the same type in the script policy identified with the PolicyID identifier will be disabled. Event handlers of other types will be not affected.

Example Code

The following code snippets (Policy Script 1 and Policy Script 2) illustrate how you can disable the onPreModify event handler in Policy Script 2. In this example, the Policy Script 1 is thought to be processed earlier than the Policy Script 2. Otherwise, the use of the OverridePolicy method takes no effect.

Policy Script 1 

[VBScript]
Sub onPreModify(Request)
' Here goes your code
'
' Disable the onPreModify event handler in the Policy Script 2 
If Request.Class = "user" Then 
    Request.OverridePolicy "Script2"
End If
End Sub  



[PowerShell]
function onPreModify($Request)
{
 # Here goes your code
 #
 # Disable the onPreModify event handler in the Policy Script 2
if($Request.Class -eq "user")
 {
  $Request.OverridePolicy("Script2")
 }
}

Policy Script 2 

[VBScript]
Sub onPreModify(Request)
' Here goes your code
'
End Sub
' Identify the Policy
Function onGetPolicyMarker()
  onGetPolicyMarker="Script2" 
End Function  



[PowerShell]
function onPreModify($Request)
  {
   # Here goes your code
   #
  }
   # Identify the Policy
function onGetPolicyMarker()
 {
  return "Script2"
 }

To label a policy script, in VBScript, use the onGetPolicyMarker Function procedure with no arguments, as it is shown in the example above. That function must return a policy identifier (any string). The policy ID should be unique for each policy script. For more information, please refer to Using Policy Identifiers.

Note:

The OverridePolicy method disables only the specified event handler within the specified policy. For information about how to completely override a policy (for example, one from the Active Roles built-in policies listed in this table), refer to Overriding Active Roles Built-In Policies, Using Policy Identifiers.
