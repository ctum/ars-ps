Policy Object

The Active Roles intrinsic object Policy allows your policy script to access Policy Object that executes that script. The use of the Policy object is especially useful when a policy script is used by several Policy Objects.

Note:

Don't confuse the Active Roles intrinsic object Policy with the Active Roles-specific Policy Object (the object class edsPolicyObject).

Properties and Methods

The Policy object refers to the appropriate Policy Object and implements the interface IADs and all interfaces available for Policy Objects, such as IEDM, IEDMPolicyObject, etc. The complete list of those interfaces as well as properties and methods the interfaces provide is in the sections Basic Interfaces, Policy Object Interfaces.

Example Code

The following code snippet illustrates how you can use the object Policy to access properties of the Policy Object that executes this script. 

[VBScript]
Sub onPreCreate(Request)
 ' Retrieve the number of policy entries
  NumberofPolicies = Policy.PolicyEntriesCount
  If NumberofPolicies > 1 Then Exit Sub
     ' your code goes here
     '
End Sub




[PowerShell]
function onPreCreate($Request)
{
     # Retrieve the number of policy entries
  $NumberofPolicies = $Policy.DirectoryEntry.PolicyEntriesCount
  if ($NumberofPolicies -gt 1) { return }
     # your code goes here
     #
}

This policy script retrieves the number of policy entries the Policy Object that executes the script includes. If the Policy Object includes other policies (i.e. NumberofPolicies > 1) the script terminates without raising an error. Otherwise, it enforces the Administrative policy you want to define with this script.
