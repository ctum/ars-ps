Context Object

Active Roles provides an internal object "Context" that implements the AddParameter method used to add a new parameter to your policy script. The Context object is accessible in the onInit function and in policy types.

Methods

The Context object supports the following methods:

  • AddParameter Method

  • Parameter Method

  • ParameterEx Method

AddParameter Method

The AddParameter method is applicable only inside functions which declare script parameters, for example, onInit(). The AddParameter method uses the following syntax:

[VBScript]
Context.AddParameter(byVal Name)

[PowerShell]
$Context.AddParameter($Name) 

Arguments
Name

[in] Required. String. Specifies the name for a newly created parameter.

The following table describes properties implemented by the object returned by the AddParameter method. This object allows you to define characteristics of your custom parameter.

 

Property

Description

DefaultValue

Access: Read/Write

Data type: Depends on the parameter syntax

Optional. Sets and returns the default value for the custom parameter, if any.

Description

Access: Read/Write

Data type: String

Optional. Sets and returns the parameter description.

MultiValued

Access: Read/Write

Data type: Boolean

Optional. The default value is FALSE. When TRUE, the parameter is multivalued.

PossibleValues

Access: Read/Write

Data type: a single value or an array of values.

Optional. Sets and returns acceptable value(s) for the parameter.

Note: The PossibleValues property is used only if the CustomPicker property is not set. Otherwise, setting this property takes no effect.

Required

Access: Read/Write

Data type: Boolean

Optional. The default value is FALSE. When TRUE, specifies that this parameter is mandatory.

Syntax

Access: Read/Write

Data type: String

Optional. The default value is String. You can use this property to control how Active Roles treats the parameter value and to have the console provide appropriate tools for setting up the parameter value. This property can take one of the following values:

  • DN  The parameter stores the Distinguished Name of a certain object. The console provides an object picker for setting the parameter value of this syntax.

  • GeneralizedTime  The parameter stores a date and time value. The console provides a date/time picker for setting the parameter value of this syntax.

  • SID  The parameter stores the Security Identifier of a certain object. The console provides an object picker for setting the parameter value of this syntax.

  • ObjectGUID  The parameter stores the Globally Unique Identifier of a certain object. The console provides an object picker for setting the parameter value of this syntax.

  • Boolean  Indicates that the parameter stores a TRUE or FALSE value. The console provides a value picker for setting the parameter value of this syntax.

  • SecureString  The console displays a string of asterisks when you set the parameter value of this syntax, to mask the parameter value.

  • Attribute,<objectClass>,<restrict>,<single>  Indicates that the parameter stores the LDAP Display Name of a certain attribute. The console provides an attribute picker for setting the parameter value of this syntax. The syntax options such as <objectClass>, <restrict> and <single> control the behavior of the attribute picker.

<objectClass>  The LDAP Display Name of the object class whose attributes are listed in the attribute picker by default. Depending on the <restrict> option, the attribute picker may allow a different object class to be selected.

<restrict>  The true or false literal indicating whether or not the attribute picker allows you to change selection of the object class. If true, you can select attributes of any object class. If false, you can select only attributes of the object class identified by the <objectClass> option.

<single>  The true or false literal indicating whether or not the attribute picker allows multiple attributes to be selected. If true, you can select a single attribute. If false, you can select more than one attribute at a time (this requires the MultiValued property of the parameter to be set to TRUE).

For instance, $param1.Syntax = "Attribute,user,true,true" configures the attribute picker so that you can select a single attribute of the user object class and cannot change the object class.

  • GenerationRule,<objectClass>,<attribute>  Indicates that the parameter stores a policy rule representing an instruction on how to generate the value of a certain attribute based on values of other attributes. The console provides a rule builder for setting the parameter value of this syntax. The syntax options such as <objectClass> and <attribute> control the behavior of the rule builder.

<objectClass>  The LDAP Display Name of the object class to which the policy rule applies. The rule builder configures a rule to generate the attribute value based on attributes of this object class.

<attribute>  The LDAP Display Name of the attribute to which the policy rule applies. The rule builder configures a rule to generate the value of this attribute.

For instance, $param1.Syntax = "GenerationRule,user,sAMAccountName" indicates that the parameter stores a rule for generating the value of the sAMAccountName attribute based on other attributes of the user object class.

For information about the structure of a generation policy rule, see EDS_EPI_UI_POLICY_RULE.

 

Note: If a given parameter has the CustomPicker property set, then the Syntax property has no effect for that parameter.

CustomPicker

Access: Read/Write

Date type: String

Optional. Used for parameters of a custom type. Specifies a DLL assembly file in the following format: "Namespace.Classname:Libraryname.dll, for example, CustomPickerDialog.CustomPickerObj:CustomPickerDialog.dll.

The .dll file must be stored in the \MMC\Extensions sub-folder of the Active Roles installation folder.

CustomPickerInitInfo

Access: Read/Write

Date type: String

Optional. Specifies a parameter to be passed to the DLL assembly specified in CustomPicker.

Note: This property is used only if the CustomPicker property is specified. Otherwise, setting this property takes no effect.

displayName

Access: Read/Write

Date type: String

Optional. Specifies the name of the parameter to be displayed on UI.
 

Parameter Method

The Parameter method is used in script policy event handlers, such as onPreCreate, onPostModify, etc, to retrieve the value of a single-valued parameter.

Object.Parameter(ByVal paramID)

Arguments

Object
Required. Always the Context object.
paramID
[in] Required. String. Identifies the custom parameter of the policy script.

ParameterEx Method

The ParameterEx method is used in script policy event handlers, such as onPreCreate, onPostModify, etc, to retrieve the value of a multi-valued parameter.

Object.ParameterEx(ByVal paramID)

Arguments

Object
Required. Always the Context object.
paramID
[in] Required. String. Identifies the custom parameter of the policy script.

Sample Script

The following code snippets illustrate how to declare and use the UserDept parameter in your policy script.

[VBScript]
Sub OnInit(Context)
   Set param1 = Context.AddParameter("UserDept")
   param1.MultiValued = False
   param1.Description = "Employee's department"
   param1.Defaultvalue = "Sales"
   param1.PossibleValues = Array("Sales", "HR", "Software Development")
End Sub
Sub onPreCreate(Request)
   strClass = Request.Class
   If (strClass <> "user") Then Exit Sub   
   strDept = CStr(PolicyEntry.Parameter("UserDept"))
   Request.Put "department", strDept   
End Sub




[PowerShell]
function onInit($Context)
 {
   $par1 = $Context.AddParameter("UserDept")
   $par1.MultiValued = $False
   $par1.Description = "User department"
   $par1.Defaultvalue = "Sales"
   $par1.PossibleValues = "Sales", "HR", "Software development"
 }




 function onPreCreate($Request)
 {
   $strClass = $Request.Class
   if ($strClass -eq "user")  {
      $strDept = [string]$Context.Parameter("UserDept")
      $Request.Put("department", $strDept)
                             }
 }
