Developing Script Condition Functions

To implement a script-based condition for an If-Else Branch activity, you need to create and apply a script module containing a function that analyzes the requested operation to determine whether to run the branch. The branch runs if the function returns TRUE. The function must be saved in a library script module. The syntax of the function is as follows:

Function <FunctionName>($Request)

Arguments

$Request
[in] Required. The intrinsic object Request.

Remarks

Within this function, you can use any interfaces the object Request implements.

Sample Script Condition Function

The following sample function retrieves the value of the "department" property of the in-process user and returns TRUE if the user belongs to the Sales department.

[PowerShell]

Function Criterion($Request)
{
if($DirObj.Class -ne "user"){return $false}
$DirObj.GetInfoEx("department", 0)
$Dept = $DirObj.Get("department")
if($Dept -eq "Sales"){return $true}
return $false
}

How to Add This Function to an If-Else Branch Activity

Use the following steps to add this function to an If-Else Branch activity defined for an existing workflow.

To add a script condition function to an If-Else Branch activity

  1. Create a library script module, and add the script function to that module. For instructions, see Understanding Library Scripts.

  2. In the console tree, expand Configuration| Policies | Workflow, and select the workflow containing the If-Else activity you want to configure.
    This opens the Workflow Designer window in the details pane, representing the workflow definition as a process diagram.

  3. In the process diagram, right-click the name of the If-Else Branch activity and click Properties.

  4. In the Properties dialog box, under Conditions, click AND group (all of these conditions are true), and then click Insert condition.

  5. Click Configure condition to evaluate, and then click Value generated by rule expression.

  6. In the Configure Rule Expression dialog box, click Add entry, and then click Value generated by script.

  7. In the Configure Entry dialog box, click Select to choose the script module you created in Step 1, and then, under Script function, select the name of the script function you saved in that script module.

  8. Click OK to close the Configure Entry dialog box.

  9. Click OK to close the Configure Rule Expression dialog box.

  10. In the Properties dialog box, click Define value to compare to, and then click Text string.

  11. In the Configure Entry dialog box, type TRUE in the Text string box, and then click OK.

  12. Click OK to close the Properties dialog box.

  13. Click Save Changes in the Workflow Designer.

In this example, the If-Else activity executes the branch if the script function returns TRUE, i.e. if the in-process user belongs to the Sales department.
