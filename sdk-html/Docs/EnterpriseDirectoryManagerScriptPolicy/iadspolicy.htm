IADs

The IADs interface provides all standard methods for access to the in-process object data contained in the Requestobject.

Using the IADs interface property methods, you can read (get) in-process property values that are stored in the Request object. For example, you can use the Name, Class, GUID, and ADsPath property methods to get such properties as "name," "objectClass," "objectGUID," and to retrieve the directory object's ADsPath string for binding to the system ADSI provider.

You can also write (set) property values and specify (put) additional property values into the Request objectwhen executing a pre-event handler; as a result, the property values are set in the underlying Active Directory store after the request is committed.

Since the Request object is ADSI-compliant, its IADs interface conforms to the specification of properties and methods outlined in the MSDN Library and in IADs ( Active Roles ADSI Provider). For more information, see the code snippet in Setting In-Process Property Values.

Note:

You cannot get attributes values unless they are put into the request by the client, your script, or another policy script. Hence, the methods GetInfo and SetInfo have no effect.
If you need to get additional attribute values, not defined in the object Request, you can use the system ADSI provider. You can find detailed help on ADSI in the MSDN Library. To bind to the requested directory object, use the ADsPath property method, as shown in Binding to the Processed Directory Object.
