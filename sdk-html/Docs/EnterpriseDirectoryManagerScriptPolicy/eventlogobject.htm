EventLog Object

The object EventLog allows you to record events, such as information about creation, deletion or modification of objects, to the EDM Server Log from a policy script.

Methods

The object EventLog supports the following methods:

  • ReportEvent Method

Note:

The ReportEvent method is identical to the same method provided by the IEDSRequestParameters interface.

ReportEvent Method

The Sub procedure that allows you to record events to the EDM Server Log from a policy script. The method writes an entry at the end of the EDM Server Log. 

Object.ReportEvent(ByVal eventType, ByVal Msg1[,Msg2,Msg3])
Arguments
Object
Required. Always  the EventLog object.
eventType
[in] Required. Integer. Identifies the type of the event to be logged.
Msg1
[in] Required. String. Contains the user-defined message to be recorded into the EDM Server Log.
Msg2
[in] Optional. String. Contains the user-defined message to be recorded into the EDM Server Log.
Msg3
[in] Optional. String. Contains the user-defined message to be recorded into the EDM Server Log.
Settings

The eventType argument can take one of the following values.

Value

Event Type

0

Example Code

The following code snippets illustrate the use of the ReportEvent method. In this example, the onPostModify event handler allows you to write in the EDM Server Log the report on a modification of the "department" property of the processed user object.

[VBScript]
Sub onPostModify(Request)
  Dim Dept
  If Request.Class <> "user" Then Exit Sub
  Dept=Request.Get("department")
  If (VarType(Dept) <> vbEmpty) Then
     EventLog.ReportEvent EDS_EVENTLOG_WARNING_TYPE, "The department property of the user " + Request.Name,_
    "has been changed:", vbNewLine + "New value: " + Dept
  End If
End Sub  



[PowerShell]
function onPostModify($Request)
{
  if ($Request.Class -ne "user") {return}
  trap {continue}
  $Dept = (Get-QADObject $Request.AdsPath.SubString(7) -IncludedProperties "department").department
  if ($Dept)
    {
     $EventLog.ReportEvent($Constants.EDS_EVENTLOG_WARNING_TYPE, "The department property of the user " + `
                         [string]$Request.Name + " has been changed: `n" + "New value: " + [string]$Dept )
    }
}

After the script terminates, an entry similar to the following example will be added to the EDM Server Log: 

Script policy reported an event.
Request ID: {da61967c-d0d8-4bb7-a603-520ae068d01e}
Name: John J. Smith
Source container: MyDomain.local/Sales
Object class: user
Object GUID: {5bc05a3e-8101-4260-93ed-d96e564ae3c0}
Script Module: 
The department property of the user CN=John J. Smith,OU=Sales,DC=MyDomain,DC=local
has been changed:
New value: Sales
