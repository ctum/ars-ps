DirObj Object

The object DirObj allows you to access properties of the object you are processing (referred to as in-process object).

Since the object Request acts as a property bag for the in-process object data, it can only be used to access the properties that are involved in the request. The object DirObj contains values of the in-process object attributes that are actually stored in Active Directory. When processing a request, your script can use the object DirObj to retrieve or modify properties of the in-process object that are stored in Active Directory.

To access the object properties stored in Active Directory you can also bind to that object with the GetObject function (see Binding to the Object Being Processed). Whenever possible, we recommend that you use DirObj instead.

Limited Use of DirObj Object in Event Handlers

You cannot use the object DirObj within the following event handlers:

  • onPreCreate, onPreGet

  • onPostDelete

The use of the object DirObj is limited within the onPreModify and onPostModify event handlers:

The use of the method DirObj.SetInfo can lead to an infinite recursive call, also known as an infinite loop. To work around this issue, you can use the following sample scripts.

Within the onPreModify event handler, you may also avoid using the DirObj.SetInfo method by setting the appropriate attributes of the object Request, instead. For example:

[VBScript]
Sub onPreModify(Request)
If DirObj.class = "user" Then Request.Put "mail", CStr(DirObj.sn) + "@mycorp.com"
End Sub

[PowerShell]
function onPreModify($Request)
{
 if ($DirObj.class -eq "user")
{
 $email = [string]$DirObj.DirectoryEntry.sn + "@mycorp.com"
 $Request.Put("mail", $email)
}
}

In complicated cases, use the following sample scripts:

[VBScript]
Sub onPreModify(Request)
If DirObj.class = "user" And Request.Get("mail") = "" Then
' Remove empty attribute from request
Dim emptyObject
Request.Put "mail", emptyObject
'Modify property
DirObj.Put "mail", CStr(DirObj.sn) + "@mycorp.com"
DirObj.SetInfo
End If 
End Sub

Sub onPostModify(Request)
If DirObj.class = "user" And DirObj.mail = "" Then
DirObj.Put "mail", CStr(DirObj.sn) + "@mycorp.com"
DirObj.SetInfo
End If 
End Sub

if ($Request.class -eq "user" -and $Request.Get("mail") -eq "")
{
 $DirObj.getinfoex(@("mail", "sn"), 0)
 # Remove empty attribute from request
 $Request.ResetPropertyItem("mail")
 # Modify property
 $DirObj.Put("mail", $DirObj.Get("sn") + "@mycorp.com")
 $DirObj.SetInfo()
}
}

You cannot use the DirObj object within the onPostGet event handler. However, you can access properties of the in-process object by adding those properties to a list of properties requested to be retrieved. To clarify this, consider the following example:

Sub onPreGet(Request)
Request.AddRequestedAttribute "edsvaDomainDN" 
End Sub

Sub onPostGet(Request) 
EventLog.ReportEvent 0, Request.Get("edsvaDomainDN")
End Sub

In this example, in the onPreGet event handler, the AddRequestedAttribute method provided by the IEDSRequestedAttributes interface adds the attribute edsvaDomainDN to a list of properties requested to be retrieved. The event handler onPostGet can retrieve the value of this attribute using the Request.Get method.

You cannot modify the Name property of the DirObj object in the onPreRename and onPostRename event handlers.

Using DirObj Object in PowerShell Scripts

The DirObj object implements the IADs and IEDM interface. You can directly access these interfaces from PowerShell script, while other interfaces must be accessed only through the DirectoryEntry property of the DirObj object.

The DirectoryEntry property is used to access a System.DirectoryServices.DirectoryEntry object and all its members. For example, to access the LastName property of the IADsUser interface, you can use the following code:

[PowerShell]
$value =$DirObj.directoryentry.psbase.invokeget("LastName")

When using the DirectoryEntry property to access the System.DirectoryServices.DirectoryEntry object, you need to use methods InvokeGet (for read) and InvokeSet (for write). To invoke these methods, you must use the Invoke method. For example, 

[PowerShell]
$value =$DirObj.directoryentry.psbase.invokeget("LastName") # Correct.




$value =$DirObj.directoryentry.invokeget("LastName") #Incorrect.

Managing Multi-valued Attributes

Prior to setting values for multivalued attributes, it is recommended to create a variable with an array of the values for the multivalued attribute, and then use the variable to set the multivalued attribute. We recommend to use the System.Collections.Generic.List type for the variable. Here is an example:

[PowerShell]
$value=New-Object System.Collections.Generic.List[string]
$value.Add($member1) 
$value.Add($member2) 
$Request.Put("member",$value.ToArray())

Example Code

The following policy script illustrates how you can use DirObj to get values of the in-process User object.

[VBScript] 
Sub OnPreModify(Request)
 If Request.Class <> "user" Then Exit Sub
 If Request.Get("employeeID") <> "" Then 
    If DirObj.Get("department") <> "Sales" Then
        Err.Raise 1, "Administrative policy", _
        "You can modify employeeID only for users from the department Sales"
    End If
 End If   
End Sub




[PowerShell]
function onPreModify($Request)
{
 if ($Request.Class -ne "user") { return }
 if (($Request.Get("employeeID") + "") -ne "")
 {
    if ($DirObj.Get("department") -ne "Sales")
    {    
        throw "You can modify employeeID only for users from the department Sales"
    }
 }

This script enforces the following policy: an administrator can change the user's property employeeID only if that user belongs to the department Sales.
