Understanding Approval Workflows

Workflow activities of the Approval category are referred to as approval rules. The approval rules added to the workflow determine who is authorized to manually accept or deny operation requests, the required sequence of approvals, and who needs to be notified of approval tasks or decisions. Examples of such operations include the creation, deletion, modification, and deprovisioning of user accounts.

A workflow containing approval rules is referred to as approval workflow. The start conditions of such a workflow determine whether to apply the workflow's approval rules to a given operation. When Active Roles receives a request to perform an operation that meets the start conditions of an approval workflow, Active Roles applies the workflow's approval rules to submit the operation for approval.

Normally, Active Roles does not apply approval rules to operations requested by privileged entities such as:

  • Members of the Active Roles Admin role (by default, the Administrators local group on the computer running the Active Roles Administration Service).

  • The Administration Service itself (so-called internal operation requests, which are considered to be initiated by the service account of the Active Roles Administration Service).

  • The identity that the approval rule designates as an approver.

The operations requested by these privileged entities are performed without waiting for approval. However, operations requested by workflow activities are exceptions to this rule. Active Roles provides a configuration option allowing you to enforce the application of approval rules on the operation requests initiated by the workflow activities regardless of the entity on whose behalf the workflow is being executed.

You can use the Enforce approval option on a workflow definition object to configure the workflow so that Active Roles applies the approval rules to the operations requested by activities in that workflow even though the workflow runs under a privileged user or service account. By default, the workflow-level option setting is inherited by all activities in the workflow. You can configure individual activities to override that inheritance by enabling or disabling the Enforce approval option in the properties of the activity.

Creating and Configuring Approval Rules

You can create and configure approval rules with Active Roles console when configuring a workflow. For more information and related procedures, see Configuring an Approval Activity in the Active Roles Administrator Guide.

Custom Functions for Approval Rules

To specify filtering on operation request and advanced settings for approvers, you should develop your custom functions saved in the library scripts. For more information, refer to Developing Custom Functions for Workflows.

Custom Scripts for Operations Requiring Approval

When your custom script attempts to perform an operation in Active Directory, Active Roles uses approval rules to check whether or not the changes this operation causes must be approved by approvers. If approval is not required, the system starts applying the changes at once. If the potential changes are subject to approval, in some cases, to perform the operation, your script must send the built-in control AllowApproval to the Administration service with a request to perform the operation. The script also can prompt you either to confirm the operation or to cancel it. If you respond positively, an operation is created in the approval system, and the approval workflow thus starts.

For information about how to develop scripts to make changes that are subject to approval, see "Developing Scripts for Operations Requiring Approval" in Related Topics.

For information about how to monitor the completion status of operation requiring approval, see "Monitoring Operations Requiring Approval" in Related Topics.
