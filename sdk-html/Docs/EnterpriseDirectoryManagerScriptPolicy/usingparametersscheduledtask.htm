Using Parameters in Scheduled Task Scripts

Active Roles enables your scheduled task scripts to read or write values of  parameters stored in the scheduled task object that executes the script. To access a parameter, bind to the appropriate scheduled task object using the DirObj property of the Active Roles intrinsic object Task, and then read or write the parameter using the Parameters property of the IEDM interface as shown in the following sample scheduled task script.

This script reads the scheduled task's parameter paramDN that contains the distinguished name of a container. The script enumerates all users that reside within that container and deletes only users that have the "description" property set to "To be deleted". 

[VBScript]
' Read the scheduled task parameter
strDN = Task.DirObj.Parameters("paramDN")
 ' Bind to a container
Set objCon= GetObject("EDMS://" & strDN)
 ' Apply the filter
objCon.Filter=Array("user")
 ' Enumerate the container's child user objects
For Each obj In objCon
    If obj.Description = "To be deleted" Then   
        objCon.Delete "user", obj.Name
    End If    
Next
objCon.SetInfo




[PowerShell]
# Read the scheduled task parameter
$DN = $Task.DirObj.Parameters.Value("paramDN").Value
# Search the container's child user objects
$objs = Get-QADObject -SearchRoot $DN -SearchScope OneLevel -Type "user"
foreach($obj in $objs)
{
    if($obj.Description -eq "To be deleted")
    {
        Remove-QADObject $obj -Force
    }
}

To use this scheduled task script

  1. Register this script as a scheduled task script module on the Administration Service.

  2. Create a scheduled task that will run this script.

  3. Add to the scheduled task the parameter named paramDN, and set it to the distinguished name of the container within which you want the scheduled task to delete the user objects.  To add the parameter, use the information from Specifying Parameters for Scheduled Tasks.
