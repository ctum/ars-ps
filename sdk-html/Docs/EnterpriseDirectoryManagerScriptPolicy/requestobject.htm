Request Object

To provide policy scripts with access to Administration Service in-process data, Active Roles offers an ADSI-compliant scriptable COM object called Request. The Request object  contains all the data Administration Service needs to perform the requested operation. The data in the Request object  falls into two categories:

  • In-process object data - the attribute values of the directory objects involved in the operation, such as object name; the in-process object data only includes the attribute values specified by the client or policy script

  • Request object data - the request parameters, such as request type (creating, updating, deleting, or moving the object), target container for the "move" operation, new object name for the "rename" operation, and user-defined (custom) parameters

Properties and Methods

The Request object  supports properties and methods provided by the following interfaces:

  • IADs- provides policy scripts with access to in-process object data

  • IADsPropertyList - provides policy scripts with access to in-process object data

  • IEDSRequestParameters - provides policy scripts with access to request object data

  • IEDSEffectivePolicyRequest - enables policy scripts to retrieve and modify information about Active Roles Policy constraints and definitions that are effectively applied to the object whose processing has been requested

  • IEDSPolicyComplianceRequest - enables policy scripts to generate the information and error messages to be sent to the Active Roles client. For example, the script can inform the Active Roles client on whether information it supplies is in compliance with the Active Roles policy requirements.

  • IEDSRequestedAttributes  -  enables policy scripts to control a list of the processed object's properties requested to be retrieved.

Managing Multi-valued Attributes

To set the value of a multi-valued attribute for the Request object, you need to construct an array that includes all the values.

To create the array, we recommend to use th System.Collections.Generic.List class, for example:

[PowerShell]
$value=New-Object System.Collections.Generic.List[string]
$value.Add($member1)
$value.Add($member2)
$Request.Put("member",$value.ToArray())

Arguments for Put and PutEx Methods

For the "value" attribute (the second attribute) of the Request object you can use the following variable types:

  • System.string

  • System.int

  • System.bool

  • System.long

  • System.byte[]

  • System.Guid

  • System.Security.Principal.SecurityIdentifier

  • System.Management.Automation.PSObject (only for single-valued attributes)
