Session Object

The object Session stores information about the Active Roles client's session, such as security information, activity counters, locality settings, etc.

Before using the Session object in your scripts, you should obtain the reference to this object using the properties _NewEnum or Current of the Sessions object (see Example Code).

Properties and Methods

The object Session supports the following properties and methods:

  • ClientHost Property

  • ClientLocale Property

  • ClientSite Property

  • ClientVersion Property

  • IsAdmin Property

  • IsCurrent Property

  • OperationalController Property

  • Properties Property

  • SessionPath Property

  • UserName Property

  • UserPath Property

  • Disconnect Method

OperationalController Property

Read/Write. String. Sets and returns the client's operational domain controller (DC).

Object.OperationalController(byVal strDomainName)[=strDCName]

Arguments

Object

Required. Always the Session object.

strDomainName

[in] Required. String. Specifies the DNS or NetBIOS of a managed domain name.

strDCName

[in] Optional. String. Specifies the domain controller name you want to make an operational DC.

Note:

This property returns an empty string if the operational DC is not specified for the managed domain identified with the argument strDomainName. In this case, for client operations, the DirSync DC will be used.

Arguments

Object

PropVal

[in] Optional.  Specifies the custom property value.

Remarks

The Properties property allows you to define your custom properties and store their values in the object Session. This property is declared as the default property for the object Session, that is why you can omit its name as shown in the following code snippet:

[VBScript]
Sub onPreModify(Request)
    Set objSession = Security.Sessions.Current
     ' Set the custom property ProcObj to the reference to the in-processed object
    objSession("ProcObj") = GetObject(Request.AdsPath)
    '
     ' Retrieve properties of the processed object
    strClass = objSession("ProcObj").Class
    strName = objSession("ProcObj").Name
     ' Your code goes here
     '
End Sub




[PowerShell]
function onPreModify($Request)
{
    $objSession = $Security.Sessions.Current
    
     # Set the custom property ProcObj to the reference to the in-processed object
    $objSession.Properties("ProcObj", (Get-QADObject $Request.Name))
    
     # Retrieve properties of the processed object
    $strClass = $objSession.Properties("ProcObj").ClassName
    $strName = $objSession.Properties("ProcObj").Name
     # Your code goes here
     #
}

Example Code

This policy script allows you to enforce the following administrative policy: the Active Roles client can modify properties of a user only if the client's session is running in the security context of the user's manager.

[VBScript]
Sub onPreModify(Request)
    If Request.Class <> "user" Then Exit Sub
    Set Session = Security.Sessions.Current
     ' Retrieve AdsPath of the user in which security context
     ' this session is running
    strUserAdsPath = Session.UserPath
    If strUserAdsPath = "" Then Exit Sub
    On Error Resume Next
    DirObj.GetInfoEx "manager",0
     ' Get AdsPath of the processed user's manager
    strManagerAdsPath = DirObj.Get("manager")
    On Error GoTo 0
    If StrUserPath <> strManagerAdsPath Then
        Err.Raise 1, "administrative policy",_
        "Only the manager can modify the user's properties"
    End If
End Sub




 




[PowerShell]




function onPreModify($Request)
{
    if ($Request.Class -ne "user") { return }
    $Session = $Security.Sessions.Current
     # Retrieve AdsPath of the user in which security context
     # this session is running
    $strUserAdsPath = $Session.UserPath
    
    if ($strUserAdsPath -eq "") { return }
    $StrUserPath = $strUserAdsPath.Replace('EDMS://', '')
    $DirObj.GetInfoEx( ("manager"),0)
     # Get AdsPath of the processed user's manager
    $strManagerAdsPath = $DirObj.Get("manager")
    if ($StrUserPath -ne $strManagerAdsPath)
    {
        throw "Only the manager can modify the user's properties"
    }
}
