IEDsRequestParameters::WhoAmI

The WhoAmI method allows you to get the distinguished name and pre-Windows 2000 name of the security principal in whose security context the Active Roles client that sends the request is running.

Sub  WhoAmI(ByRef SamAccountName, ByRef DN)

Arguments

SamAccountName
[out] Required. The pre-Windows 2000 name of the security principal that has sent the request.
DN
[out] Required. The distinguished name of the security principal that has sent the request.

Remarks

The WhoamI method returns the distinguished name of the security principal only if it resides within one of the managed domains. Otherwise, the second argument (DN) of the method returns the empty string.

For information about how to use this method in PowerShell scripts, see the sample PowerShell script below in this topic.

Example Code (VBScript)

The following VBScript and PowerShell code snippets illustrate the use of the WhoAmI method. In this example, the onPreDelete event handler enforces the following administrative policy: Only users from the 'Support' domain can delete directory objects.

[VBScript]
Sub onPreDelete(Request)
Dim strSan,strDn,strDomainname
' Set the domain name
strDomainname="Support"
' Call the WhoamI method
Request.WhoamI strSan,strDn
If InStr(LCase(strSan),LCase(strDomainName)) <> 1 Then
  Err.Raise 1, "Administrative Policy", "Only users from the " + strDomainName + " domain can delete directory objects"
End If
End Sub




 
[PowerShell]
function onPreDelete($Request)
{
  # ---- Set the domain name ----
  $strDomainname="Support"
  $strSan=""
  $strDN=""
  # --- Call the WhoamI method ---
  $request.WhoAmi([ref]$strSan,[ref]$strDN)
  if ($strSan.ToLower().Contains($strDomainName.ToLower()) -ne 1)
     {
       throw "Only users from the " + $strDomainname + " domain can delete directory objects "
     }
}
