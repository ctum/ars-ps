Script Policies, Scheduled Tasks and Workflows

In Active Roles, the term script policy refers to administrative policy based on running specific scripts referred to as policy scripts (see Understanding Terminology). Policy scripts are used to implement procedures that Administration Service activates upon client requests to perform administrative actions such as the creation, modification, or deletion of objects.

The term scheduled task refers to a script the Administration Service executes at scheduled time (see Understanding Terminology).

The term workflow refers to a set of administrative activities based on Microsoft's Windows Workflows Foundation technology (see Understanding Workflows).

This documentation covers the basics of developing policy scripts, scheduled task scripts and custom script functions for workflows, and provides some examples of using such scripts.
