Overriding Active Roles Built-In Policies

Active Roles built-in policies are used in the built-in Policy Objects created during the installation of the Administration Service. The built-in policies are listed in this table.

A direct modification of built-in policies is not advised. When it is necessary, we recommend that you override those policies with the custom policy scripts instead.

To override a built-in policy

  1. Create a policy script with which you want to override the policy in question.

  2. Add the Function procedure onGetPolicyMarker to that script. Note that onGetPolicyMarker must return the policy ID of the policy you want to override. Policy IDs for the built-in policies are listed in the column 1 of this table.

  3. Apply the policy script to the container on which you want to override the policy. For related procedures, see Applying Policy Scripts.

For more information about onGetPolicyMarker, refer to Using Policy Identifier.

Example Code

The following sample policy script allows you to override the EDS_APE_TYPE_MOVE_TARGETS built-in policy for User objects. 

[VBScript]
Sub onPostGet(Request)
    If Request.Class = "user" Then
        If Request.IsAttributeRequested ("edsvaMoveTargets") Then
            Request.Put "edsvaMoveTargets","ou=London_Branch,dc=mycompany,dc=com"
        End If 
    End If
End Sub
Function onGetPolicyMarker()
   ' Identify your policy with the same policy ID that is used for a built-in policy 
   onGetPolicyMarker=EDS_APE_TYPE_MOVE_TARGETS
End Function




[PowerShell]
function onPostGet($Request)
{
    if($Request.Class -eq "user")
    {
        if($Request.IsAttributeRequested("edsvaMoveTargets"))
        {
            $Request.Put("edsvaMoveTargets", "ou=London_Branch,dc=mycompany,dc=com")
        }
    }
}
function onGetPolicyMarker()
{
    # Identify your policy with the same policy ID that is used for a built-in policy 
    return $Constants.EDS_APE_TYPE_MOVE_TARGETS
}

The policy changes the value of the edsvaMoveTargets property in the object Request. The edsvaMoveTargets property specifies a list of possible target containers for Move operation. In the given example, the Move dialog box will allow you to move users only to folders residing within the Organizational Unit London_Branch in the domain mycompany.com.
