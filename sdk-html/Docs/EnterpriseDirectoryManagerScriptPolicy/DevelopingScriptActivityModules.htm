Developing Script Activity Modules

Script activities are typically used to perform automated steps in a workflow process. A Script activity is defined by a policy script module (script activity module) created in Active Roles. The script module contains a PowerShell script code implementing certain event handlers.

For detailed information about how to develop a policy script and save it into a policy script modules, refer to the Script Policy Reference section and to "Getting Started with Policy Scripts" in Related Topics.

Important

Script activity supports only Windows PowerShell scripts. To configure a script activity, you cannot use policy scripts developed using other script languages, such as VBScript or JScript.

Sample Script Activity Module

The following sample script activity module sets the 'manager' property of a newly created user to the 'managedBy' property of the user's parent Organizational Unit.

[PowerShell]
function onPostCreate($Request)
{
if ($Request.class -eq "user")
{   
$ou = [ADSI] $Request.Parent
[string]$ouManager = $ou.managedBy
if ($ouManager -ne $null)
 {
   $DirObj.Put("manager", $ouManager)
   $DirObj.SetInfo()   
 }
}
}

How to Add This Module to a Script Activity

Use the following steps to add this script module to a Script activity defined for an existing workflow.

To add a script module to a Script activity

  1. Create a policy script module, and add the script function to that module. For detailed information about creating policy script modules, see the Script Policy Reference section.

  2. In the Active Roles console tree, expand Configuration| Policies | Workflow, and select the workflow containing the Script activity you want to configure.
    This opens the Workflow Designer window in the details pane, representing the workflow definition as a process diagram.

  3. In the process diagram, right-click the name of the Script activity to modify and click Properties.

  4. In the Properties dialog box, click Browse to choose the script module you created in Step 1, and then, under Function to run, select the name of the script function.

  5. Click OK to close the Properties dialog box.

  6. Click Save Changes in the Workflow Designer.
