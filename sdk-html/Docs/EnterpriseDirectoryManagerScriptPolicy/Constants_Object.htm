Constants Object

The Constants object provides properties that allow you to access the enumeration constants and identifiers of Active Roles controls. You can use this object in both the policy scripts and the scheduled task scripts.

Important

This object is available only in PowerShell scripts.

Properties

The Constants object supports read-only properties whose names correspond to the names of constants in Active Roles enumerations and built-in Active Roles controls. In addition to Active Roles enumerations, this object allows you to access the following ADSI enumerations: ADSTYPEENUM and ADS_PROPERTY_OPERATION_ENUM.

To employ the Constants object in your policy scripts or scheduled task scripts, use the following syntax:

$Constants.<constant name>
where the <constant name> refers to the name of an enumeration constant or built-in control.

Example

This code snippet illustrates how to use the Constants object to get the identifier of the EDS_CONTROL_TREE_DELETE Active Roles control and the value defined in the ADSTYPE_CASE_IGNORE_STRING constant from the ADSTYPEENUM enumeration.

[PowerShell]
function onPreDelete($Request)
{
$Request.PutInControl($Constants.EDS_CONTROL_TREE_DELETE,$Constants.ADSTYPE_CASE_IGNORE_STRING, "Any")
}
