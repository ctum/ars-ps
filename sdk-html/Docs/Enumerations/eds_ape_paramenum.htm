EDS_APE_PARAMENUM

The EDS_APE_PARAMENUM enumeration is used to identify parameters of Policy Object. The IEDMPolicySetting.Name property can contain one of these values.

Parameter ID

Constant

Description

EDS_APE_PARAM_ERROR_MSG

1

An error message to be displayed if Policy fails.

EDS_APE_PARAM_FILTER

2

The search filter string used in the Property Validation Policy. The filter uses an XML format (see the "Add Policy Entries" code snippet in Managing Policy Objects).

EDS_APE_PARAM_MODULE

4

The GUID of a module with script text.

EDS_APE_PARAM_EXCHANGE_SERVER

7

The DN of Exchange Server (to create Mailbox on that server).

EDS_APE_PARAM_EXCHANGE_MAILBOX_STORE

9

The DN of Exchange Mailbox Store (to create Mailbox on it).

EDS_APE_PARAM_HOME_DIR_PATH

10

Default path for Home Folders created by this policy.

EDS_APE_PARAM_RENAME_DIR

11

Boolean. When TRUE, Home Folder is renamed if User Account is renamed.

EDS_APE_PARAM_DELETE_DIR

12

Defines when to delete Home Folder if the user is deleted:
0 - Home Folder is never deleted;

1 - Delete only empty Home Folder;

2 - Home Folder is always deleted

EDS_APE_PARAM_CREATE_HOME_DIR

13

Boolean. When TRUE, Home Folder is created if User Account is created.

EDS_APE_PARAM_INHERIT_RIGHTS

14

Boolean. When TRUE, permissions on Home Folder created by this policy are inherited from the folder's parent folder.

EDS_APE_PARAM_SET_OWNER

15

Boolean. When True, a user is owner of the user's home folder. The owner of an object is able to control how permissions are set on the object, and can grant permissions to others.

EDS_APE_PARAM_DIR_ACCESS

16

Specifies permissions on the user's Home Folder allowed for user:

0 - No permissions allowed;

1 - Permission to modify the Home Folder properties is allowed;

2 - Full access

EDS_APE_PARAM_CREATE_SHARE

17

Boolean. When TRUE, the policy synchronously creates a home share when creating Home Folder.

EDS_APE_PARAM_SHARE_PREFIX

18

Specifies the Home Share name prefix.

EDS_APE_PARAM_SHARE_SUFFIX

19

Specifies the Home Share name suffix.

EDS_APE_PARAM_USERS_LIMIT

20

Specifies the number of users who can connect to Home Share at one time.

EDS_APE_PARAM_SHARE_COMMENT 

21

A comment about Home Share folder, if needed. A comment is optional.

EDS_APE_PARAM_MAP_DRIVE_LETTER 

22

A letter with column (:), to which Policy maps a newly created Home folder, for example, P:.

EDS_APE_PARAM_EXCH_ALIAS_GENERATION

31

Specifies rules to generate a mailbox-enabled user's alias:

0 - Policy generates the alias using user logon name in the pre-Windows 2000 format.

1 - Policy generates the alias by combining the user's first name initial and last name, such as JSmith for John R. Smith.

2 - Policy generates the alias by combining the user's first name and last name initial, such as JohnS for John R. Smith.

3 - Policy generates the alias by combining the user's first, middle, and last initials, such as JRS for John R. Smith.

5 - Policy generates the alias using a custom rule. The custom rule must be saved in the Policy Object parameter EDS_APE_PARAM_ALIAS_GENERATION_CUSTOM_RULE.

6 - Policy generates the alias by combining user's first name followed by last name, such as  JohnSmith for John R. Smith.

EDS_APE_PARAM_ATTRIBUTE

40

In EDS_APE_TYPE_DYNAMIC_GROUPS policy entry, specifies the LDAP display name of the group attribute to store the group membership rules.

In EDS_APE_TYPE_PICTURE_MANAGEMENT_RULES policy entry, specifies the LDAP display name of the attribute controlled by the policy.

EDS_APE_PARAM_SIGN

41

String. Specifies the logical operator (AND or OR) used to combine validation criteria included in a policy rule for the Property Generation and Validation policy:

'&' - specifies the logical operator AND

'|' - specifies the logical operator OR

EDS_APE_PARAM_REQUIRED_PROPERTY

43

Boolean. When TRUE, the Property Generation and Validation policy prevents assigning no value to the appropriate object property. 

EDS_APE_PARAM_APPLY_ONTO

50

String, Multivalued.

Contains object class names (objectClass).

When this parameter is specified, Active Roles will apply the policy only to the object classes the parameter contains.

When this parameter is not set, Active Roles will try to apply the policy to all object classes.

In EDS_APE_TYPE_PICTURE_MANAGEMENT_RULES policy entry, specifies the object class controlled by the policy.

For a sample code that illustrates the use of this parameter, see Adding Policy Entries.

EDS_APE_PARAM_DESCRIPTION

57

String. Specifies the Policy Entry name.

EDS_APE_PARAM_GENERATION_RULE

58

String. Specifies the automatic name generation rule.

EDS_APE_PARAM_GUID

59

The GUID of the Policy Object Entry parameter.

EDS_APE_PARAM_DG_MAX_NUM_MEMBERS

63

Integer. A maximum allowed number of members in dynamic groups.

EDS_APE_PARAM_DG_NESTED_CREATED_LIMIT 

64

Integer. When a number of members in a dynamic group exceeds this value, Active Roles creates nested groups.

EDS_APE_PARAM_DG_ON_DIRSYNC

65

Boolean. When True, the Dynamic Groups built-in policy will process the DirSync modifications. Otherwise, only the Active Roles client requests rebuild members of the dynamic groups.

EDS_APE_PARAM_SKIP_VALIDATION

66

Boolean. When TRUE,the Property Generation and Validation policy permits modifications of properties that contradicts that policy. 

EDS_APE_PARAM_DISPLAY_NOTE

67

String. A text describing the policy. The text will be displayed by Active Roles client in the Policy Information dialog box.

EDS_APE_PARAM_POLICY_LISTEN_DIRSYNC

68

Boolean. When TRUE, the script can receive control upon DirSync notifications about data changes in Active Directory.

EDS_APE_PARAM_DG_NOTES

69

String. A text to be displayed in the Notes box on the General tab of the Properties dialog box for dynamic groups.

EDS_APE_PARAM_SYNC_POST_PROCESSING

72

Integer. When this parameter is set to 1, the policy will always run synchronously, even if it includes post-event handlers.

EDS_APE_PARAM_CASE_IGNORE

75

Boolean. When FALSE, a policy rule defined in the Property Generation and Validation Policy is case-sensitive.

EDS_APE_PARAM_CONDITION

85

The search filter string used in the 'Member Of' Policy. The filter uses an XML format (see the "Add Policy Entries" code snippet in Managing Policy Objects).

EDS_APE_PARAM_ALIAS_GENERATION_CUSTOM_RULE

181

String. Contains the custom rule used for generating e-mail aliases.

EDS_APE_PARAM_PICTURE_WIDTH_MIN

225

Integer. In EDS_APE_TYPE_PICTURE_MANAGEMENT_RULES policy entry, specifies the minimum allowed width of the picture.

EDS_APE_PARAM_PICTURE_HEIGHT_MIN

226

Integer. In EDS_APE_TYPE_PICTURE_MANAGEMENT_RULES policy entry, specifies the minimum allowed height of the picture.

EDS_APE_PARAM_PICTURE_WIDTH_MAX

227

Integer. In EDS_APE_TYPE_PICTURE_MANAGEMENT_RULES policy entry, specifies the maximum allowed width of the picture.

EDS_APE_PARAM_PICTURE_HEIGHT_MAX

228

Integer. In EDS_APE_TYPE_PICTURE_MANAGEMENT_RULES policy entry, specifies the maximum allowed height of the picture.

EDS_APE_PARAM_PICTURE_RESIZING_ENABLED

229

Boolean. In EDS_APE_TYPE_PICTURE_MANAGEMENT_RULES policy entry, specifies whether the policy causes Active Roles to resize pictures whose dimensions exceed the maximum allowed size.

Remarks

Because VBScript cannot read data from a type library, VBScript applications do not recognize the symbolic IDs as defined above. Use the numerical constants, instead, to set the appropriate flags in your VBScript applications. To use the symbolic IDs, create explicit declarations of the ID constants.
