EDS_EFFECTIVE_POLICY_ENUM

The EDS_EFFECTIVE_POLICY_ENUM enumeration is used to specify the policy constraints defined in the Request object for the specified attribute of the processed objects.

Constant

Value

Description

EDS_EPI_UI_VALUE_REQURIED

1

Defines that the AttrName attribute is a required attribute. An administrator must specify the attribute value when creating the object.

EDS_EFFECTIVE_POLICY_DEFAULT_VALUES

2

Sets the default value for the AttrName attribute. This value is specified in Arr.

EDS_EPI_UI_POSSIBLE_VALUES

3

Provides an administrator with a list of possible values for the AttrName attribute. The list is stored in the Arr argument. When creating a new object with the Active Roles Web Interface or Console, an administrator can select the desired value from the provided list.

EDS_EPI_UI_POLICY_RULE

4

Automatically generates the AttrName attribute value using the name generation rules specified in Arr. Those rules allow the script to build the AttrName attribute value using values of other attributes of the object.

EDS_EPI_UI_GENERATED_VALUE

7

Automatically sets the AttrName attribute to the value specified in Arr.

EDS_EPI_UI_RESTRICTED

8

When the Arr argument is set to TRUE, an administrator must select the value for the AttrName attribute only from the list described above in this table, for EDS_EPI_UI_POSSIBLE_VALUES.

EDS_EPI_UI_AUTO_GENERATED

9

When the Arr argument is set to TRUE, the Active Roles Administration Service generates the AttrName attribute value. You cannot set manually that attribute.

EDS_EPI_UI_DISPLAY_NOTE

10

The Arrargument contains a note that Active Roles displays when you click the attribute title on the Properties dialog box.

EDS_EPI_UI_SERVER_SIDE_GENERATED

12

Specifies that Administration Service generates a set of possible values for the attribute AttrName.

EDS_EPI_UI_MAX_LENGTH

13

Defines the maximum allowed length of the object attribute an Active Roles client can send to Administration Service. This length is specified in Arr.

EDS_EPI_UI_ADJUST_CASE

14

Defines how to change the case of the attribute value. The argument Arr can take any value from the enumeration EDS_CHAR_TRANSFORMATION_TYPEENUM.

EDS_EPI_UI_PROHIBITED_SYMBOLS

15

Defines a set of forbidden characters for the AttrName attribute value. The forbidden characters are specified in Arr as a string value.

EDS_EPI_UI_GENERATION_RULES

16

The Arr argument is an array of policy rules that specify possible values of the AttrName property. Based on those policy rules, an Active Roles client application can create a list from which to select a value for the AttrName property. A policy rule can be a literal value or it can be an instruction to compose a value by combining values of other properties. Such an instruction may use one or more tokens of the following syntax to represent the value of a particular property: %xy<name> where name is the LDAP display name of the property (see "Arr Syntax" in the topic that covers EDS_EPI_UI_POLICY_RULE). In contrast to EDS_EPI_UI_POLICY_RULE, with EDS_EPI_UI_GENERATION_RULES the Arr argument may contain a list of rules and values rather than a single rule or value.

EDS_EPI_UI_ENABLE_ENTRY_BY_RULE

17

Directs the Active Roles client application to make the attribute entry available for editing if the policy rule evaluates to TRUE. If the rule evaluates to FALSE, the client is expected to make the entry unavailable. The attribute name and the policy rule are specified by the AttrName and Arr arguments of the method SetEffectivePolicyInfo, respectively. Thus, the following policy script causes the Web Interface to make the edsvaMinor attribute entry available for editing if the edsvaMajor attribute has a value of TRUE:

function onGetEffectivePolicy($Request)

{

$Request.SetEffectivePolicyInfo('edsvaMinor', `
$Constants.EDS_EPI_UI_ENABLE_ENTRY_BY_RULE, `
'%<edsvaMajor>')

}

EDS_EPI_UI_RELOAD_EPI_BY_RULE

19

Directs the Active Roles client application to reload policy information for the target attribute every time the rule attribute changes on the client side. This type of policy enables the use of different policy rules depending on the value of the rule attribute. For example, with a policy script that defines a separate list of the target attribute's possible values for each possible value of the rule attribute, this policy type makes it possible for the client application to apply the appropriate list of possible values to the target attribute entry when a particular value is selected for the rule attribute.

The target attribute and rule attribute are specified by the AttrName and Arr arguments of the method SetEffectivePolicyInfo, respectively. Thus, the following policy script causes the Web Interface to retrieve and reapply the policy information for the edsvaTarget attribute when you change the value of the edsvaRule attribute on the Web Interface page:

function onGetEffectivePolicy($Request)

{

$Request.SetEffectivePolicyInfo('edsvaTarget', `
$Constants.EDS_EPI_UI_RELOAD_EPI_BY_RULE, `
'edsvaRule')

}

Note: The EDS_EPI_UI_RELOAD_EPI_BY_RULE policy causes the Web Interface to retrieve and reapply the EDS_EPI_UI_POSSIBLE_VALUES and EDS_EPI_UI_GENERATED_VALUE policies only.
