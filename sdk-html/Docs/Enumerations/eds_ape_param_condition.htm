EDS_APE_PARAM_CONDITION

This topic provides information about the format of the EDS_APE_PARAM_CONDITION parameter. This parameter is a part of the 'Member Of' Policy Object. The EDS_APE_PARAM_CONDITION parameter contains description of configured Policy rule, which governs how the configuring policy chooses objects of the controlled type to add to or remove from groups.

XML Elements

The EDS_APE_PARAM_CONDITION parameter uses the Extensible Markup Language (XML). XML elements you can use in EDS_APE_PARAM_FILTER are listed below.

PolicyCondition

This is a root element of the EDS_APE_PARAM_CONDITION parameter.

Parent Elements: None

Child Elements: Item

Attributes

Name

Description

Version

Required. Identifies the current version number of the PolicyCondition element format.

Item

Describes object's attribute, conditions and relationships between them.

Parent Elements: PolicyCondition

Child Elements: None

Attributes

Name

Description

attrName

Required. lDAP name of the object's property, for which you want to configure a condition.

Operator

Required. Determines conditions for specified by the attrName attribute.

The operator attribute possesses the following values:

  • "=*" - denotes the "starts with" condition.

  • "*=" - denotes the "ends with" condition.

  • "==" - denotes the "equals" condition.

  • "!=" - denotes the "does not equal" condition.

  • "$=" - denotes the "matches regular expression" condition.

  • "$^" - denotes the "does not stat with" condition.

  • "^$" - denotes the "does not end with" condition.

  • "!*" - denotes the "is empty" condition.

  • "**" - denotes the "does not empty" condition.

  • "~~" - denotes the "contains" condition.

  • "!~" - denotes the "does not contain" condition.

LinkID

Required. Determines relation between the configured attrName attribute and the next one. The LinkID attribute possesses the following values:

  • "AND" - means that the configured attrName attribute and the next one must be applied.

  • "OR - means that the configured attrName attribute OR next one must be applied.

Content

Type

Description

Syntax

Text

Value of the attrName attribute is a simple text string.

plain text

<Object type> Property

Value of the attrName attribute is obtained from other property of the selected object type.

%XY<Z>

  • "Z" - Stands for lDAP name of obtained property of the configuring object type. Controlled property searches for  the value of the obtained property.

  • "Y" - Integer. Stands for number of characters of obtained property are used in the controlled property.

  • "X" - Stands for filling character that are added at the end of controlled property value, if Z is shorter than Y.

Note:

If X and Y elements are not specified, the controlled property possesses the value of the obtained property entirely.

Example: %*5<displayName>

Parent OU Property

Value of the attrName attribute is obtained from property of the configuring object's parent OU.

%XY<ouN.Z>

  • "Z" - Stands for lDAP name of obtained property of the controlled object' parent OU. Controlled property searches for  the value of the obtained property.

  • "N" - Integer. Specifies how much distant parent OU you want to use. If you do not specify this parameter, then immediate parent OU of the object to be selected.
    Note that lower number means greater distance from the controlled object. For instance, 1 indicates parent OU that is immediate child of the domain.

  • "Y" - Integer. Stands for number of characters of obtained property are used in the controlled property.

  • "X" - Stands for filling character that are added at the end of controlled property value, if Z is shorter than Y.

Note:

If X and Y elements are not specified, the controlled property possesses the value of the parent OU's obtained property entirely.

Example: %+4<ou2.managedBy>

Parent Domain Property

Value of the attrName attribute is obtained from property of the configuring object type's parent domain.

%XY<domain.Z>

  • "Z" - Stands for lDAP name of obtained property of the controlled object' domain. Controlled property searches for  the value of the obtained property.

  • "Y" - Integer. Stands for number of characters of obtained property are used in the controlled property.

  • "X" - Stands for filling character that are added at the end of controlled property value, if Z is shorter than Y.

Note:

If X and Y elements are not specified, the controlled property possesses the value of the parent OU's obtained property entirely.

Example: %d6<domain.edsaDnsName>

Mask

Value of the attrName attribute is configured using specified acceptable characters.

{}

The acceptable values between the braces are the following:

  • "*" - Any series of characters are acceptable in the controlled property's value.

  • "N optional [a-z],[A-Z],[0-9],[!-=]" - At most number N of characters is acceptable. These characters could be the following:

    • "[a-z]" - Lowercase letters

    • "[A-Z]" - Uppercase letters

    • "[0-9]" - Numerals

    • "[!-=]"" - Other printable characters

  • "N required [a-z],[A-Z],[0-9],[!-=]" - Exactly number N of characters is acceptable. These characters could be as it described earlier.

Note:

If you do not want to allow to use some type of characters, numerals, for example, just exclude the [0-9] from the code.

Example: {6 required [a-z],[A-Z],[0-9],[!-=]}

Example

The following example illustrates the use of the XML elements in the EDS_APE_PARAM_CONDITION parameter:

<PolicyCondition Version="1.0">
	<Item attrName="l" operator="==" linkID="and">
		Chicago
	</Item>
	<Item attrName="department" operator="==" linkID="and">
		Sales
	</Item>
	<Item attrName="division" operator="==" linkID="and">
		%<ou.description>
	</Item>
</PolicyCondition>
