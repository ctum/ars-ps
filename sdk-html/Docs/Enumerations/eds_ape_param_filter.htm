EDS_APE_PARAM_FILTER

This topic provides information about the format of the EDS_APE_PARAM_FILTER parameter. This parameter is a part of the Property Generation and Validation Policy Object. The EDS_APE_PARAM_FILTER parameter contains description of configured Policy rule, which determines criteria used to validate input values and generate default value for a controlled property.

XML Elements

The EDS_APE_PARAM_FILTER parameter uses the Extensible Markup Language (XML). XML elements you can use in EDS_APE_PARAM_FILTER are listed below.

PVGRules

This is a root element of the EDS_APE_PARAM_FILTER parameter.

Parent Elements: None

Child Elements: PVRuleItem

Attributes

Name

Description

Version

Required. Identifies the current version number of the PVGRules element format.

PVRuleItem

Describes conditions and relationships between them.

Parent Elements: PVGRules

Child Elements: Value

Attributes

Name

Description

Operator

Required. Determines conditions for configuring policy rule.

The operator attribute possesses the following values:

  • "=*" - denotes the "starts with" condition.

  • "*=" - denotes the "ends with" condition.

  • "==" - denotes the "equals" condition.

  • "!=" - denotes the "does not equal" condition.

  • "$=" - denotes the "matches regular expression" condition.

  • "$^" - denotes the "does not stat with" condition.

  • "^$" - denotes the "does not end with" condition.

  • "!*" - denotes the "is empty" condition.

  • "**" - denotes the "does not empty" condition.

  • "~~" - denotes the "contains" condition.

  • "!~" - denotes the "does not contain" condition.

LinkID

Required. Determines relation between the configured operator attribute and the next one. The LinkID attribute possesses the following values:

  • "AND" - means that the configured operator attribute and the next one must be applied.

  • "OR - means that the configured operator attribute OR next one must be applied.

 Value

Describes values of the configured condition and relationship between the configured condition value and the next one.

Parent Elements: PVGRules

Child Elements: None

Attributes

Name

Description

LinkID

Required. Determines relation between the configured value of condition and the next one. The LinkID attribute possesses the following values:

  • "AND" - means that the configured value of condition and the next value must be applied.

  • "OR - means that the configured value of condition OR next value must be applied.

Content

Type

Description

Syntax

Text

Value of the controlled property is a simple text string.

plain text

<Object type> Property

Value of the controlled property is obtained from other property of the configuring object type.

%XY<Z>

  • "Z" - Stands for lDAP name of obtained property of the controlled object. Controlled property possesses the value of the obtained property.

  • "Y" - Integer. Stands for number of characters of obtained property are used in the controlled property.

  • "X" - Stands for filling character that are added at the end of controlled property value, if Z is shorter than Y.

Note:

If X and Y elements are not specified, the controlled property possesses the value of the obtained property entirely.

Example: %*5<displayName>

Parent OU Property

Value of the controlled property is obtained from property of the configuring object's parent OU.

%XY<ouN.Z>

  • "Z" - Stands for lDAP name of obtained property of the controlled object' parent OU. Controlled property possesses the value of the obtained property.

  • "N" - Integer. Specifies how much distant parent OU you want to use. If you do not specify this parameter, then immediate parent OU of the object to be selected.
    Note that lower number means greater distance from the controlled object. For instance, 1 indicates parent OU that is immediate child of the domain.

  • "Y" - Integer. Stands for number of characters of obtained property are used in the controlled property.

  • "X" - Stands for filling character that are added at the end of controlled property value, if Z is shorter than Y.

Note:

If X and Y elements are not specified, the controlled property possesses the value of the parent OU's obtained property entirely.

Example: %+4<ou2.managedBy>

Parent Domain Property

Value of the controlled property is obtained from property of the configuring object type's parent domain.

%XY<domain.Z>

  • "Z" - Stands for lDAP name of obtained property of the controlled object' domain. Controlled property possesses the value of the obtained property.

  • "Y" - Integer. Stands for number of characters of obtained property are used in the controlled property.

  • "X" - Stands for filling character that are added at the end of controlled property value, if Z is shorter than Y.

Note:

If X and Y elements are not specified, the controlled property possesses the value of the parent OU's obtained property entirely.

Example: %d6<domain.edsaDnsName>

Mask

Value of the controlled property is configured using specified acceptable characters.

{}

The acceptable values between the braces are the following:

  • "*" - Any series of characters are acceptable in the controlled property's value.

  • "N optional [a-z],[A-Z],[0-9],[!-=]" - At most number N of characters is acceptable. These characters could be the following:

    • "[a-z]" - Lowercase letters

    • "[A-Z]" - Uppercase letters

    • "[0-9]" - Numerals

    • "[!-=]"" - Other printable characters

  • "N required [a-z],[A-Z],[0-9],[!-=]" - Exactly number N of characters is acceptable. These characters could be as it described earlier.

Note:

If you do not want to allow to use some type of characters, numerals, for example, just exclude the [0-9] from the code.

Example: {6 required [a-z],[A-Z],[0-9],[!-=]}

Example

The following example illustrates the use of the XML elements in the EDS_APE_PARAM_FILTER parameter:

<PVGRules Version="1.0">
	<PVRuleItem operator="$^" linkID="or">
		<value linkID="and">
			New
		</value>
		<value linkID="and">
			Los
		</value>
	</PVRuleItem>
	<PVRuleItem operator="!=" linkID="and">
		<value linkID="and">
			London
		</value>
		<value linkID="and">
			Chicago
		</value>
	</PVRuleItem>
	<PVRuleItem operator="==" linkID="and">
		<value linkID="or">
			%&lt;ou.ou&gt;
		</value>
		<value linkID="or">
			%&lt;domain.edsaDnsName&gt;
		</value>
	</PVRuleItem>
</PVGRules>
