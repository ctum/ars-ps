Searching for ADSI Objects

An important function of Active Roles is to resolve queries for information about directory objects. To write efficient queries, it is helpful to be familiar with the following:

  • Determining the scope of the query: The scope of a query is determined by the object to which you bind. To query across all objects within an enterprise, bind to the global catalog service. Alternatively, you can bind to a specific domain to scope the search to a more narrow range.

  • Determining the depth of the query: The search query be restricted to the following limits: a base query, a one-level query, or a subtree search.

  • Handling large result sets: The search results set is processed with the ADO Recordset object.

  • Understanding the query syntax: The Active Roles ADSI provider supports both the LDAP syntax as documented in RFC 2254, as well as a subset of SQL.

For more information about how to search for directory objects using the Active Roles ADSI provider, refer to the section Searching for Directory Objects.
