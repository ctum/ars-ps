Binding to an Object Using a GUID

An object distinguished name changes if the object is renamed or moved, while the object's objectGUID property never changes. The ADSI provider allows you to bind to an object using the object's GUID.

To bind to a directory object using the object's GUID, use the following binding string: 

EDMS://<DomainDCName>/<GUID=XXXXX>
<DomainDCName>
Optional parameter. Specifies the name of the domain controller or domain where you want the provider to search for available Administration Service. For information about how the provider searches for Administration Service when this parameter is omitted, see Connecting to Service.
<GUID=XXXXX>
Required parameter. XXXXX is the string representation of the hexadecimal value of the GUID. 
The GUID string can take the form: 
		<GUID=f5913bf3-ee10-4277-b1fc-d3709f09dcb8>
or 
		<GUID=B8DCE0AF4CA57E42BCAABE3206F2E73F>

Example

To retrieve the GUID assigned to an object, use the GUID property of the IADs interface as shown in the following VBScript code snippet: 

' Bind to the object using its DN
set obj = GetObject("EDMS://CN=James Johnson,OU=Sales,DC=mycompany,DC=com")
 ' Retrieve the object's GUID
strGUID = obj.GUID
Set obj = Nothing
 ' Rebind to the object using its GUID
set obj = GetObject("EDMS://<GUID=" & strGUID & ">")

Note:

Unlike the ADSI LDAP provider, the Active Roles ADSI provider supports all methods and properties of the IADs and IADsContainer interfaces even for objects obtained by binding using the object's GUID.
