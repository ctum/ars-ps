top Class

The top class attributes are used in all AD objects as well as the objects specific to Active Roles, such as Managed Units, Policy Objects, and Access Templates. In addition to the standard ADSI top class attributes, Active Roles defines the following attributes, which can be accessed using the Active Roles ADSI Provider.

Note:

For more information about the Top class, please refer to the MSDN Library.

edsaObjectRightsEffective

Syntax: Integer.

Access:  Read-only.

Searchable: No.

Specifies effective permissions assigned to the connected Active Roles user for administrative operations on the object. The attribute value is a mask of bits defined in the EDS_RIGHTS_ENUM enumeration.

 

edsaActiveDirectoryACEs

Syntax: Multi-value octet string.

Access: Read/Write.

Searchable: No.

Specifies the object's security descriptor. Each value is an octet string that represents a single ACE.

 

edsvaKeywords

Syntax:Multi-value DN string.

Access: Read/Write.

Searchable: Yes.

Stored virtual attribute. Stores keywords that can be used to search for objects of interest, such as groups.

 

edsaMemberOf

Syntax: Multi-value DN string.

Access:  Read-only.

Searchable: No.

Specifies a list of distinguished names of groups to which the object belongs directly (as a member), including the object's primary group. Supports ASQ.

 

edsaMemberOfIndirect

Syntax: Multi-value DN string.

Access:  Read-only.

Searchable: No.

Specifies a list of distinguished names of the groups to which the object belongs due to group nesting. Supports ASQ.

 

edsvaMoveTargets

Syntax: Multi-value DN string.

Access:  Read-only.

Searchable: No.

Specifies a list of possible target containers for Move operation. The value of this attribute can be overwritten by a policy to limit or extend that list. For more information, please refer to IEDSRequestParameters.

 

edsaShowInRawViewOnly

Syntax: Boolean.

Access: Read/Write.

Searchable: Yes.

When the attribute is set to TRUE, the Active Roles snap-in displays the object only when the Raw View mode is used.

 

edsaDataSource

Syntax: Integer.

Access:  Read-only.

Searchable: Yes.

Specifies the data storage for the object, such as Active Directory, Network resources, etc. The data storage is identified by a combination of bits defined in the EDS_DATA_SOURCE enumeration (see the table below).

Note:

For schema attributes and classes, the edsaDataSource value indicates if that attribute or class is loaded from the AD schema or defined in the Active Roles Schema.

Constant

Value

Description

EDS_DATA_SOURCE_UNDEF

0

Data storage is undefined

EDS_DATA_SOURCE_AD

1

Object is owned by an AD service

EDS_DATA_SOURCE_EDS

4

Object is owned by the Active Roles Administration Service (is stored in the Active Roles  database)

EDS_DATA_SOURCE_RES

8

Object is a Network resource
 

 

edsvaProtectFromDeletion

Syntax: Boolean

Access: Read/Write

Searchable: No

When set to TRUE on a given object, prevents deletion of the object by appropriately configuring the object's security settings in both Active Roles and Active Directory. As a result the object cannot be deleted, whether using Active Roles or other tools for Active Directory administration.

 

edsvaRestoreObject

Syntax: Boolean

Access: Write only

Searchable: No.

Setting this attribute to TRUE restores (undelete) the deleted object (only for Windows 2008 R2 with Recycle Bin enabled).
