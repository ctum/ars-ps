Enumerating Directory Objects

Enumeration allows you to obtain a list of directory objects in a container, a list of members of a group or a list of objects a Managed Unit includes. To enumerate children of a container object (such as domain, Organizational Unit, folder), use the IADsContainer interface. To enumerate  members of a group, use the Members method of IADsGroup. To enumerate objects included in a Managed Unit, use the ASQ search method to search the edsaMember property of the Managed Unit .

Note:

The enumeration involves binding to directory objects residing in a container object you search. For a large container, this can significantly impact the performance of your script. To search large directory containers, use techniques described in Searching for Directory Objects.

Enumerating Children of Container Object

To list the objects of a container, simply put the object in the For Each ... In ... loop, for example: 

'Enumerate all objects in myContainer
For Each obj In myContainer
    ' Retrieve the RDN of obj
   objName= Cstr(obj.Name)
    ' Do something with obj
Next

The preceding example can result in producing a lot of output in a large container. It is possible to limit the information returned from the enumeration of a container by using the Filter property method of IADsContainer. The Filter property is an array of the schema class names that will be returned in a given enumeration. The following code snippet (VBScript) shows how to only list computers and users in the Organizational Unit: 

Dim con, schemacon,ss
 ' Bind to a container
set con= GetObject("EDMS://Ou=Sales,DC=mydocompany,DC=com")
 ' Retrieve the object schema
set schemacon=GetObject(con.Schema)
 ' Verify if that object is a container
If schemacon.Container = True Then
    ss=""
     ' Apply the filter
    con.Filter=Array("user","computer")
     ' Enumerate the container's child objects
    For Each obj In con
        ss= ss + obj.Name + " (" + obj.Class + ")" + VbCrLf
    Next
     ' Output the list of the child objects
    MsgBox "Objects in the " + con.Name + " container:" + VbCrLf + ss
Else
    MsgBox "The" + con.Name + " object is not a container!"
End If

Enumerating Group Members

To enumerate the members of a group, use the Members method of IADsGroup. You can also use the group object properties edsaMember and edsaMemberIndirect (see Note below).  The Members method  gets the IADsMembers interface on the members object that is the collection of ADSI objects that represent the members of the group.
The following code snippet (VBScript) shows how to enumerate members of the group AdvancedGroup that resides in the OU SoftDept in the domain mycompany.com. 

Dim grp,memberList,member,ss
 ' Bind to the group object
Set grp = GetObject("EDMS://CN=AdvancedGroup,OU=SoftDept,DC=mycompany,DC=com")
 ' Get a collection of the group members
Set memberList = grp.Members
ss=""
 ' Enumerate all members of the group AdvancedGroup
For Each member In memberList
    ss=ss + member.Name & " (" & member.Class & ")" + VbCrLf
Next
MsgBox ss

The IADsMembers interface also supports the Filter property method that allows you to specify an enumeration filter. For example, if you want to enumerate only users belonging to the group AdvancedGroup , you should add to the precedent code snippet the following line (to be inserted directly after ss=""): 

memberList.Filter=Array("user")

Note:

The IADsGroup interface allows you to obtain only a list of the explicit members of a group. To list members of a group due to group nesting, use the edsaMember and edsaMemberIndirect attributes of the Group class. For a sample code, please refer to Managing Groups.

Enumerating Members of Managed Unit

The following code snippet (VBScript) illustrates how to use the ASQ search to list all user accounts the MyMU Managed Unit includes. The script outputs a list of the users distinguished names into the specified text file.

' Set the DN of Managed Unit
 const strMU = "CN=MyMU,CN=Managed Units,cn=Configuration"
 ' Set the output file name
 const strOutputFile = "C:\temp\userlist.txt"
 Set fso = CreateObject("Scripting.FileSystemObject")
 Set NewFile = fso.CreateTextFile(strOutputFile, True)
 Set oConnect = CreateObject("ADODB.Connection")
 oConnect.Provider = "ADsDSOObject"
 oConnect.Open "DS Query", "", ""
 Set Command = CreateObject("ADODB.Command")
 Set Command.ActiveConnection = oConnect
  ' Specify the search criteria
 Command.CommandText = _
 "<EDMS://" + strMU + "//EDS_SEARCHPREF_ATTRIBUTE_QUERY=edsaMember>;(objectClass=user);distinguishedName;base"
 NewFile.WriteLine "Managed Unit: " + strMU + " includes the following users: "
 NewFile.WriteLine ""
 Set rs = Command.Execute
 Do Until rs.EOF
     NewFile.WriteLine "DN: " + rs.Fields("distinguishedName").Value
     rs.MoveNext
 Loop
 NewFile.Close
 Set fso=nothing
 Set NewFile=nothing
 MsgBox "The file " + strOutputFile + " was created."

Remarks

Whenever possible, enumerate objects whose properties are resident in the property cache. Enumeration of  uncached properties takes longer to get and may significantly impact the performance of your script, because the Active Roles Administration Service must retrieve the property from Active Directory.

The Filter property of IADsContainer and IADsMembers expects an array, and even if only one value should be applied in the filter, it must still be made into an array. Passing the schema class name by itself, without using the Array function, does not raise an error, but no filtering will be applied.

In order to change the filter settings and reuse the filter, clear its contents by setting it to an empty string, for example:

objContainer.Filter=""

For more information about using the IADsContainer, IADsGroup and IADsMembers interfaces for enumerating directory objects, refer to the MSDN Library.
