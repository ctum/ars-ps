IADsClass

The IADsClass interface is designed for managing schema class objects that provide class definitions for any ADSI object. Other schema management interfaces include IADsProperty for property definitions and IADsSyntax for property syntax.

IADsClass Methods

The IADsClass interface defines the following methods that can be used by Automation (Visual Basic, VBScript, JScript) clients.

IADsClass Methods

Description

Qualifiers

Returns a collection of ADSI objects that describe provider-specific qualifiers for the schema.

Remarks

Schema objects are organized in the schema container of a given directory. To access an object's schema class, use the object's Schema property defined in the IADs interface to obtain the ADsPath string and use that string to bind to its schema class object.

IADsClass Properties

The property methods of the IADsClass interface get or set the following properties.

IADsClass Properties

Description

Abstract  

Access: Read/Write

Data Type: Boolean

Boolean value indicating whether this class is Abstract or non-abstract. When TRUE, this class is an Abstract class and cannot be directly instantiated in the directory service. Abstract classes can only be used as super classes.

AuxDerivedFrom

Access: Read/Write

Data Type: Variant

Array of ADsPath strings that indicate the super Auxiliary classes this class derives from.

Auxiliary

Access: Read/Write

Data Type: Boolean 

Boolean value that indicates whether or not this class is Auxiliary. When TRUE, this class is an Auxiliary class and cannot be directly instantiated in the directory service. Auxiliary classes can only be used as super classes of other Auxiliary classes or as a source of additional properties on structural classes.

Container

Access: Read/Write

Data Type: Boolean

Boolean value that indicates whether this class can be a container of other object classes. If this value is TRUE, you can call the get_Containment method to get an array of the object classes that this class can contain.

Containment

Access: Read/Write

Data Type: Variant

A BSTR array in which each element is the name of an object class that this class can contain.

DerivedFrom

Access: Read/Write

Data Type: Variant

Array of ADsPath strings that indicate which classes this class was derived from.

MandatoryProperties

Access: Read/Write

Data Type: Variant

SAFEARRAY of BSTRs that lists the properties that must be set for this class to be written to storage.

OptionalProperties

Access: Read/Write

Data Type: Variant

SAFEARRAY of BSTRs that lists the properties that are optional for this schema class.

NamingProperties

Access: Read/Write

Data Type: Variant 

SAFEARRAY of BSTRs that lists the properties that are used for naming attributes of this schema class. For example, in NDS, "CN" and "OU" are naming properties.

OID

Access: Read/Write

Data Type: BSTR

Provider-specific Object Identifier that defines this class. This is provided to allow schema extension, using Active Directory™, in directory services that require provider-specific OIDs for classes.

PossibleSuperiors

Access: Read/Write

Data Type: Variant

Array of ADsPath strings that indicate the schema classes that can contain instances of this class.

Sample Code (Visual Basic)

The following code snippet shows how to work with the IADsClass interface.

Dim obj As IADs
Dim cls As IADsClass
Set obj = GetObject("EDMS://MyServer/CN=MyComputer,CN=Computers,DC=mycompany,DC=com")
Set cls = GetObject(obj.Schema)
' Inspecting mandatory and optional properties.
For Each p In cls.MandatoryProperties
    MsgBox "Must-have: " & p
Next
For Each p In cls.OptionalProperties
    MsgBox "May-have: " & p
Next

Here, p is the name of an attribute. For example, the Computer object has an optional attribute known as Owner.
