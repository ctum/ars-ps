IEDM::RefreshSchema

The RefreshSchema method refreshes the Active Roles ADSI provider's schema cache on the local computer.

Sub  RefreshSchema()

Arguments

This method does not require arguments.

Remarks

Use this method after you change the Active Roles Administration Service schema - for example, after you add a new virtual attribute.

Note

This method refreshes the schema cache only on the local computer where the Active Roles ADSI provider is installed. To refresh the Active Roles ADSI provider schema cache on the computer running the Active Roles Administration service, set the edsaRefreshProviderSchemaNow attribute of the Configuration container object to True. For more information, refer to Sample Scripts below in this topic.

Sample Scripts

This code snippet  illustrates how to refresh the schema cache on the local computer where the Active Roles ADSI Provider is installed.

' Refresh the schema cache on the local computer
'
' Bind to any directory or ARS-specific object
Set obj = GetObject("EDMS://CN=John Smith,OU=Sales,DC=mycompany,DC=com")
' Call the Refresh method
obj.RefreshSchema

This code snippet illustrates how to refresh the schema cache on the computer running the Active Roles Administration service.

' Refresh the schema on the computer running  Active Roles Administration service
'
' Bind to the Configuration container
Set objConfig = GetObject("EDMS://CN=Configuration")
' Set the edsaRefreshProviderSchemaNow to True 
objConfig.Put "edsaRefreshProviderSchemaNow", 1
objConfig.SetInfo
