IEDMPolicyEntry Properties

The following table describes properties provided by the IEDMPolicyEntry interface.

Property

Description

GUID

Access: Read-only

Returns the GUID of a Policy.

Name

Access: Read/Write

Data type: String

Sets and returns the Policy name. The name is displayed on the Policiestab of the Properties dialog box for the appropriate Policy Object.

ErrorMessage

Access: Read/Write

Data type: String

Sets and returns the message displayed to Active Roles clients when the administrative policy fails.

Type

Access: Read/Write

Data type: Long integer

Sets and returns the Policy type. For more information, refer to Types of Administrative Policies.

PolicySettingsCount

Access: Read-only

Data type: Long integer

Returns a number of parameters defined for Policy.

Item

Access: Read-only

Data type: IEDMPolicySetting

Retrieves an IEDMPolicySetting instance by index. The Item property is a read-only indexer that allows for accessing the policy parameter by its ID.

ObjAPE.Item(ID)

Arguments

ObjAPE
Required. Always a name of the processed Policy Entry.
ID
[in] Required. Variant. The identifier of the parameter of policy. For the custom parameters, such as parameters for your policy scripts, this argument must be a string. For the built-in parameters, this argument takes integer values from the second column of the table in EDS_APE_PARAMENUM.

For a sample code that illustrates how to use this property, refer to Specifying Parameters for Policy Scripts.

Types of Administrative Policies

The Type property of the IEDMPolicyEntry interface can take any of values listed in the Value column of the following tables. The Events column lists events (i.e. requests the Active Roles client sends to Administration Service) upon which the Policy is activated.

Policies configured with the Active Roles snap-in

These policies can be configured using the Active Roles snap-in (see "Managing Policy Objects" in Active Roles Help). You can also create a Policy of this type using the Active Roles ADSI Provider (see  Managing Policy Objects). After creating the Policy, use methods of the IEDMPolicySetting interface to set the Policy parameters listed in the Parameters column of the table below.

Remarks

Depending on a Policy Object in which the Active Roles built-in policy is included, it can affect the directory objects or those specific to Active Roles. We don't recommend that you directly modify built-in policies. When it is necessary, Active Roles built-in policies can be overridden using the custom policy scripts. For more information, refer to Overriding Active Roles Built-In Policies.
