Storing Application-Specific Parameters

Active Roles implements the objects of the edsAppSetting type that allow you to store parameters to be used by your custom applications. These objects are referred to as storage objects. The storage objects have the following attributes designed to store values that can be used by your applications:

  • 4 single-value attributes (LDAP display names: edsaExtensionAttribute1 through edsaExtensionAttribute4)

  • 4 multi-value attributes (LDAP display names: edsaMVExtensionAttribute1 through edsaMVExtensionAttribute4)

All the attributes above conform to the DirectoryString syntax.

The storage objects are in the container Services: 

CN=Services,CN=Application Configuration,cn=Configuration

Creation of Storage Objects

In the Services container, you can create an edsAppSetting type object in one of the following ways:

  • With your custom ADSI script using the CreateObject method provided by the IADsContainer interface (see the sample code below in this topic).

  • Using the Active Roles snap-in.

To create the storage object with the Active Roles snap-in

  1. Chose the Raw mode for Active Roles snap-in display:

    • On the View menu, click Mode, and then in the View Mode dialog box, select the Raw Mode option and click OK.

  2. In the console tree, expand the Configuration node, expand the Application Configuration node, right-click Services, point to All Tasks, and then click Advanced Created.

  3. In the first step of the New Object wizard, do the following:

    • In the Name text box, type the name of the object you want to create. In the Object class list, click EDS-Application-Setting. When finished, click Next.

  4. Click Finish to close the wizard.

Note:

You can also create a container object (the objectClass property is edsAppSettingsContainer) that can hold the above described storage objects. To create the container, perform the procedure above, replacing EDS-Application-Setting with EDS-Application-Settings-Container in Step 3.

Accessing and Modifying Storage Objects

Once you have created a storage object, its attributes can be accessed or modified in one of the following ways:

  • With your custom ADSI script using the Put, PutEx, Get, GetEx methods provided by the IADs interface (see the sample code below in this topic).

  • Using the Advanced Properties dialog box provided by the Active Roles snap-in.

To access the storage object attributes with the Active Roles snap-in

  1. Perform Step 1 of the procedure above.

  2. In the console tree, expand the Configuration node, expand the Application Configuration node, and then click Services. In the details pane, right-click the storage object in question, point to All Tasks, and then click Advanced Properties.

  3. In the Advanced Properties dialog box, do the following:

    • Select the Show all possible attributes, Include attributes with empty values check boxes. Select the Use LDAP display names option. In the list of attributes, select the attribute in question, and then click Edit to modify its value.

Example Code (VBScript)

The following sample code illustrates how you can create the object of the edsAppSetting type.

const strSettingsObjName="CN=MyApplicationSettings"
  ' Bind to the Services container object
Set objServicesContainer=GetObject("EDMS://CN=Services,CN=Application Configuration,CN=Configuration")
  ' Create the edsAppSetting type object and set some of its attributes
Set objSettings=objServicesContainer.Create("edsAppSetting",strSettingsObjName)
objSettings.Put "edsaExtensionAttribute1", "Value1"
objSettings.Put "edsaExtensionAttribute2", "Value2"
objSettings.SetInfo

The script creates the object MyApplicationSettings and sets its attributes edsaExtensionAttribute1 and edsaExtensionAttribute2 to "Value1" and "Value2", respectively.
