Connecting to a Service

When binding to a directory object, ADSI Provider establishes a connection to the Active Roles Administration Service. The Administration service to which the provider will connect, depends on the way you bind to the directory object (see Binding to Active Directory Objects).

You can have the provider connect to the specified Administration service (see Connecting to the Specified Service). You can also let the provider select the service using the binding information you have supplied (see Automatic Selecting a Service to Connect To). In the latter case, you can get the name of the connected service (see Determining the Connected Service Name).

Connecting to the Specified Service

To ensure that the provider will connect to the Administration service running on the specified computer, you should specify the computer name in the binding string and bind to the object with the OpenDSObject method of the IADsOpenDSObject interface as shown in the following example:

Const ADS_EDMSERVER_BIND = 32768
Set openDS = GetObject("EDMS:")
Set objUser = openDS.OpenDSObject("EDMS://MyComputer.mycompany.com/CN=John Smith,OU=Research,DC=Gamp,DC=com",_
"Administrator", "password", _
ADS_EDMSERVER_BIND)

MyComputer.mycompany.com is the DNS name of the Administration service computer to which you want the provider to connect. If no Administration service is running on the computer MyComputer.mycompany.com, the OpenDSObject method raises an error.

Important:

To ensure that the provider will connect to the specified service, the last argument of the OpenDSObject method must be set to 32768.

Note:

If the binding string passed to the OpenDSObject method does not define the service computer name, the provider will connect to the service running on the local computer.

Automatic Selecting a Service to Connect to

The provider will automatically select a service to which to connect when you bind to a directory object with the GetObject function. The provider will also automatically select a service when you bind to a directory object using the OpenDSObject method with the binding flag set to 0 (see the table in IADsOpenDSObject).

To clarify the provider's behavior, consider the following examples.

Examples

The following VBScript code snippet illustrates how to bind to the user John Smith using the binding string that does not specify the service name:

Set objUser = GetObject("EDMS://CN=John Smith,OU=Research,DC=Gamp,DC=com")

The provider first tries to connect to the Administration service running on the local computer. If no Active Roles Administration Service is running on that computer, the provider will search the forest to which the domain Gamp.com belongs for any Administration service managing that domain. If no Administration service is found, the provider will search the forest to which belongs the current user's home domain.

The following VBScript code snippet illustrates how to bind to the user John Smith using the binding string that specifies a server name:

Set objUser = GetObject("EDMS://myServer.mycompany.com/CN=John Smith,OU=Research,DC=Gamp,DC=com")

In this scenario, the provider first tries to connect to the Administration service running on the computer myServer.mycompany.com, if any. If this operation fails, the provider searches the forest to which the domain Gamp.com belongs for any Administration service managing that domain.  If no Administration service is found, the provider will search the forest to which the domain mycompany.com belongs. If no Administration service is found, the provider will search the forest to which the current user's home domain belongs. 

Notes:

When you bind to a directory object using the OpenDSObject method with the last argument different than 32768, the provider will select a service in the same way as described above for the GetObject function.

Determining the Connected Service Name

When binding to a directory object using methods described in Automatic Selecting a Service to Connect to, the provider automatically connects to the appropriate service. To get the name of that service, use the GetOption method provided by the IADsObjectOptions interface as shown in the following code snippet.

Const ADS_OPTION_SERVERNAME = 0
Set objUser = GetObject("EDMS://CN=John Smith,OU=Research,DC=Gamp,DC=com")
strServiceName = objUser.GetOption(ADS_OPTION_SERVERNAME)
MsgBox "The connected service name: " + strServiceName

When you pass the zero value (the ADS_OPTION_SERVERNAME value from the enumeration ADS_OPTION_ENUM) to the GetOption method, it returns the DNS name of the Administration service used for the current binding to the directory object. For more information about the IADsObjectOptions interface, refer to the MSDN library.
