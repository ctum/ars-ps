Binding to an Object Using a DN

To bind to a directory object using the object's distinguished name, use the following binding string:

EDMS://<DomainDCName>/<DistinguishedName>
<DomainDCName>
Optional entry. Specifies the name of the domain controller or domain where you want the provider to search for available Administration Service. For details, see Connecting to Service.
<DistinguishedName>
Required entry. Specifies the LDAP distinguished name of the object to which you want to bind.

The following table contains examples of binding strings you can use to bind to Active Directory objects.

Object

Binding String Example

User account (name: John, container: Users, domain: gamp.com)

EDMS://cn=John,cn=Users,dc=gamp,dc=com

Group (name: Users, container: Builtin, domain: gamp.com)

EDMS://cn=Users,cn=Builtin,dc=gamp,dc=com

Organizational Unit (name: Sales, domain: gamp.com)

EDMS://ou=Sales,dc=gamp,dc=com

Contact (name: John Doe, container: OU Sales, domain: gamp.com)

EDMS://cn=John Doe,ou=Sales,dc=gamp,dc=com

Computer (name: North, container: Computers, domain: gamp.com)

EDMS://cn=North,cn=Computers,dc=gamp,dc=com
 

The following table contains examples of binding strings you can use to bind to objects specific to Active Roles.

Object

Binding String Example

Access Template (name: AT1, container: Access Template)

EDMS://cn=AT1,cn=Access Templates,cn=Configuration

Access Template Link (name: Default Security Settings, container: AT Links)

EDMS://cn=Default Security Settings,cn=AT Links,cn=Configuration

Managed Unit (name: MU1, container: Managed Units)

EDMS://cn=MU1,cn=Managed Units,cn=Configuration

Policy Object (name: PO1, container: Policy Objects)

EDMS://cn=PO1,CN=Administration,CN=Policies,CN=Configuration

Note:

The distinguished name of an object specific to Active Roles does not contain the domain name. It contains only the object's relative distinguished name (RDN) and the name of the container within which the object resides.

Directly Binding to an Object Using the GetObject Function

To bind directly to an object, in languages that support Automation (for example Visual Basic, VBScript), use the GetObject function with a fully specified Active Roles provider binding string. This binds to the object using the current user's logon and security credentials. If you need to bind to an object using a different user or security credential, for Automation clients, use IADsOpenDSObject instead (see Binding to an Object Establishing Credentials).

The following code snippet (VBScript) shows how to bind to a directory object using the GetObject function. 

Dim objGroup
Set objGroup = GetObject("EDMS://cn=SalesGroup,OU=Sales,DC=gamp,DC=com")
