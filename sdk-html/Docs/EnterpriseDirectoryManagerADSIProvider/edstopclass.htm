edsTop Class

The edsTop class is the top level class from which all Active Roles classes are derived. In addition to standard AD attributes, Active Roles defines the following attributes, which can be accessed using the Active Roles ADSI Provider.

edsaAllowedExtRightsEffective

Syntax: OID, MultiValued.

Access: Read-only.

Searchable: No.

Contains a list of extended rights that are allowed on the object.

 

edsaAllowedMoveClassesEffective

Syntax: OID, MultiValued.

Access: Read-only.

Searchable: No.

Contains a list of classes that can be moved into the object.

 

edsaDataSource

Syntax: Integer, SingleValued

Access: Read/Write.

Searchable: Yes.

Contains a combination of bits that specify the data storage for the object (e.g. AD, Network resources, etc.)

 

edsaHasAPOLinks

Syntax: Integer.

Access: Read-only.

Searchable: No.

Specifies the number of Policy Objects linked to the object.

 

edsaHasATLinks

Syntax: Integer.

Access: Read-only.

Searchable: No.

Specifies the number of Access Templates linked to the object.

 

edsaIsPredefined

Syntax: Boolean.

Access: Read-only

Searchable: Yes.

Specifies whether the object is a predefined (built-in) object.

 

edsaObjectRightsEffective

Syntax: Integer.

Access: Read-only.

Searchable: No.

Specifies effective permissions assigned to the connected Active Roles user for administrative operations on the object. The attribute value is a mask of bits defined in the EDS_RIGHTS_ENUM enumeration.

 

edsaParameters

Syntax: Case Ignore String, Multivalued.

Access: Read/Write.

Searchable: Yes.

Contains a list of parameters used by scheduled tasks (the object class edsScheduledTask). In the list, each entry has the following XML format: <Parameter's name>Parameter's value</Parameter's name>. For example: <P1>127</P1>.

 

edsaShowInRawViewOnly

Syntax: Boolean.

Access: Read-only.

Searchable: Yes.

Specifies, whether the object should be displayed only in the Raw mode of Active Roles snap-in display.

 

edsaSystemObject

Syntax: Boolean.

Access: Read/Write.

Searchable: Yes.

The attribute is set to TRUE for internal Administration Service objects.

 

edsvaControllingApplication

Syntax: DN string.

Access: Read-only.

Searchable: No.

Contains the GUID of a controlling application. This virtual attribute is the back link for the attribute edsaControlledObjects of edsApplication class.

 

edsvaParentCanonicalName

Syntax: DirectoryString

Access: Read-only.

Searchable: No.

Contains the canonical name of the object's parent object.

 

edsvaProtectFromDeletion

Syntax: Boolean

Access: Read/Write

Searchable: No

When set to TRUE on a given object, prevents deletion of the object by appropriately configuring the object's security settings in both Active Roles and Active Directory. As a result the object cannot be deleted, whether using Active Roles or other tools for Active Directory administration.
