Managing Computer Resources

The Active Roles Administration Service implements a special namespace of scriptable objects used to manage computer resources. The objects in this namespace hold information about resources available on the computer in question. To manage objects in this namespace, you can use Active Roles ADSI Provider interfaces such as IADs and IADsContainer. The root of the namespace is "cn=Network". In this namespace, the objects reside within containers whose names correspond to those of the resource type, such as Services, Printers, etc. Computers are identified by samAccountName and the domain NETBIOS name, while each resource is identified with its name in the appropriate container.

Computer resources are Services, Printers, Shared Folders, Local Users, and Local Groups. The latter two are unavailable on domain controllers.

The following table contains examples of the distinguished names (DN) of containers holding information about computer resources available on the computer mycomp (samAccountName=mycomp$) in the domain mycompany.com (the domain NETBIOS name is mycompany).

Computer Resources

Distinguished Name of Container

Services

EDMS://cn=Services,cn=mycomp$,cn=mycompany, cn=Network

Shared Folders

EDMS://cn=Shares,cn=mycomp$,cn=mycompany,cn=Network

Printers

EDMS://cn=Printers,cn=mycomp$,cn=mycompany,cn=Network

Local Groups

EDMS://cn=Local Groups,cn=mycomp$,cn=mycompany, cn=Network

Local Users

EDMS://cn=Local Users,cn=mycomp$,cn=mycompany, cn=Network
 

Thus the DN of an object representing computer resources are composed of two sub-strings. The first sub-string identifies the computer resource in question and it does not depend on which computer you select (see How to Determine Names of Resources? below).

The second sub-string identifies the container where the appropriate computer resource is located (see the column 2 of the table above). It begins with the resource container name and includes samAccountName of the computer and the NETBIOS name of the domain where the computer resides.

It is recommended that you determine the DN of the container where the computer resources are located using the edsaResourcesContainerDN attribute of the object Computer (see Computer class). The following is a sample code that allows you to get the DN of that container for the computer mycomp in the domain mycompany.com:

' Set DN of the computer in question
const strCompDN="EDMS://CN=mycomp,OU=myDepartment,DC=mycompany,DC=com"
 ' Bind to the computer
Set objComp=GetObject(strCompDN)
 ' Load the value of the attribute edsaResourcesContainerDN
objComp.GetInfoEx Array("edsaResourcesContainerDN"),0
 ' Get the value of the attribute edsaResourcesContainerDN
CompResDN=objComp.get("edsaResourcesContainerDN")
MsgBox "Computer resources are located in: " + CompResDN

How to Determine Names of Resources?

The following procedures allow you to determine names of the resources installed on the computer in question.

Services

You can determine names of the services installed on the computer in the following way:

  1. Run the Services tool (Start » Programs » Administrative Tools » Services).

  2. In the list of services, right-click the service, and then click Properties.

  3. On the General tab, the Service name property identifies the service name.

Shared Folders

You can determine names of the shared folders available on the computer in the following way:

  1. Run the Computer Management tools (Start » Programs » Administrative Tools » Computer Management).

  2. In the console tree, expand the node System Tools, and then expand the node Shared Folders.

  3. Under Shared Folders, click Shares, in the details pane, right-click the shared folder, and then click Properties.

  4. On the General tab, the Share Name text box identifies the share name. You can select it and copy to Clipboard.

Printers

You can determine names of the printers installed on the computer in the following way:

  1. Open Control Panel (Start » Settings » Control Panel), and then click Printers.

  2. In the Printers window, right-click the printer, and then click Properties.

  3. On the General tab, the text box next to the printer icon identifies the printer name. You can select it and copy to Clipboard.

Local Users and Groups

You can determine names of the local users and groups created on the computer in the following way:

  1. Run the Computer Management tools (Start » Programs » Administrative Tools » Computer Management).

  2. In the console tree, expand the node System Tools, and then expand the node Local Users and Groups.

  3. Under the node Local Users and Groups, do one of the following:

    • To determine names of the local users, click Users.

    • To determine names of the local groups, click Groups.

  4. The details pane displays the list of the local users or groups, respectively. Names of users or groups are displayed in the first column of that list. 

Examples

The DN of the object representing the shared folder C:\Temp (the share's name is Temp) on the computer mycomp is as follows:

"CN=Temp,CN=Shares,CN=mycomp$,CN=mydomain,CN=Network"

The following is the DN of the object representing the printer HPLaserJet2100 installed on the computer mycomp: 

"cn=HPLaserJet2100,CN=Printers,cn=mycomp$,cn=mycompany,CN=Network"

For examples of the sample codes that illustrate how to manage computer resources with the Active Roles ADSI Provider, refer to Managing Computer Resources.
