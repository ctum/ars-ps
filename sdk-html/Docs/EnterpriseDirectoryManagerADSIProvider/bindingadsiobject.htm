Binding to Active Directory Objects

You should first bind to a directory object in order to navigate its namespace, search for specific information, read or modify the object properties.

You can bind directly to an object by specifying the object's distinguished name, GUID or SID in the binding string. After binding to an object, you can use the appropriate ADSI interfaces for that type of object. All ADSI objects support the IADs interface. Other interfaces are supported depending on the type of the object. For example, user objects support the IADsUser interface.

You can bind to a container object in order to browse through the objects that reside in the container. Container objects, such as Organizational Units, expose the IADsContainer interface allowing you to enumerate and manage the contained objects.

When specifying binding information, you may optionally supply additional binding characteristics, such as user name, password, or the domain controller name.

Active Roles ADSI Provider Binding String

The Active Roles ADSI Provider binding string has the following format:

EDMS://<DomainDCName>/<Object name>

<DomainDCName>

Optional entry. Specifies the name of the domain controller or domain where you want the provider to search for available Administration Service. For details, see Connecting to Service.

<Object name>

Required entry. Represents a specific directory object. The object name can be a distinguished name (DN), an object's GUID or SID.

Notes:

In the binding string above, EDMS: specifies the Active Roles ADSI provider. The ":"(colon) is part of the provider name. The provider name is case sensitive: EDMS is different from edms.

For examples of binding strings that use a DN, GUID or SID, see the appropriate topics in Related Topics.

Directly Binding to an Object Using the GetObject Function

Binding directly to a directory object through the ADSI Provider is simple. In languages that support Automation (for example Visual Basic, VBScript), use the GetObject function with a fully specified Active Roles ADSI Provider binding string. This binds to the object using the current user's logon and security credentials. If you need to bind to an object using a different user or security credential, for Automation clients, use IADsOpenDSObject instead.

The following code snippet (VBScript) shows how to bind to a directory object using the GetObject function. 

Dim objGroup
Set objGroup = GetObject("EDMS://cn=SalesGroup,OU=Sales,DC=gamp,DC=com")

Binding to an Object using the IADsOpenDSObject Interface

To establish credentials when passing a bind request to Administration Service, use the OpenDSObject method of the IADsOpenDSObject interface instead of GetObject as shown in the following Visual Basic code snippet:

Const ADS_EDMSERVER_BIND = 32768
Dim openDS As IADsOpenDSObject
Dim objUser As IADsUser
Set openDS = GetObject("EDMS:")
Set objUser=openDS.OpenDSObject("EDMS://ServiceCompName/CN=Bob,OU=Sales,DC=gamp,DC=com", _
                                "Administrator", "password", _
                                 ADS_EDMSERVER_BIND)

Note:

Explicit credentials in the OpenDSObject method of IADsOpenDSObject can only be used when you connect to Administration Service running on a remote computer.
Setting the flag in the OpenDSObject method to ADS_EDMSERVER_BIND ensures that the provider will connect to the Administration Service running on the ServiceCompName computer. For more information, please refer to IADsOpenDSObject.
