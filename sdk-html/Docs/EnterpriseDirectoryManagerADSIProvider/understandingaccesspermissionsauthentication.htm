Understanding Permission Validation

When ADSI clients use the Active Roles ADSI Provider to access Active Roles, the provider sends a request to the Active Roles Administration Service to perform operations. Upon receiving of an operation request the service checks whether the ADSI client has sufficient permissions and, if so, issues Windows function calls to perform the requested operation. Otherwise, the service rejects the operation request.

During the permission validation process, Administration Service first determines the permissions of the user account making the request. Then, Administration Service determines whether the user account is permitted to perform the requested actions on the directory objects involved in the operation. Finally, Administration Service performs or rejects the requested actions based on the conducted permission analysis.

Note:

The permission validation process implicitly uses the credentials of the user account in whose security context the ADSI client is running unless different user credentials are explicitly supplied. To supply user credentials that are different from those being used by default, bind to the ADSI object you want to process using the IADsOpenDSObject interface.
