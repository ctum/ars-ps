IEDMPolicyInfo Properties

The following table describes properties provided by the IEDMPolicyInfo interface.

Property

Description

AutoGenerated

Access: Read-only

Data type: Boolean

Returns True if the attribute value has been automatically generated by the policy.

AdminNote 

Access: Read-only

Data type: String

Returns a note to be displayed on the Administrative Note tab of the Policy Information dialog box.

CaseAjusting

Access: Read-only

Data type: Integer

Returns one of values from the EDS_CHAR_TRANSFORMATION_TYPEENUM enumeration.

GeneratedValue

Access: Read-only

Data type: String

Returns a value of the attribute generated by the policy.

MaxLength

Access: Read-only

Data type: Integer

Returns the maximum allowed length of the object attribute the client can send to Administration Service.

Note

This property returns -1 if Administrative policy does not restrict the object attribute length.

Name 

Access: Read-only

Data type: String

Returns the Ldap display name of the directory object attribute about which the interface gets the information.

PolicyRule 

Access: Read-only

Data type: String

Returns a rule used by the policy to generate the attribute values.

PossibleValues

Access: Read-only

An array of strings.

Returns acceptable values for the attribute.

ServerSideGenerated

Access: Read-only

Data type: Boolean

Returns True if Administration Service prompts the client to select the attribute value from a list of values generated by Administration Service.

ValueRequired

Access: Read-only

Data type: Boolean

When True, the attribute must be set.

ValueRestricted

Access: Read-only

Data type: Boolean

When True, the attribute must take one of the values specified using the PossibleValues property. Otherwise, those values can be used as a prompt for the Active Roles client.
