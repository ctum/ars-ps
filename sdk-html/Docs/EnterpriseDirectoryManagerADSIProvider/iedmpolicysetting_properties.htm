IEDMPolicySetting Properties

The following table describes properties provided by the IEDMPolicyEntry interface.

Property

Description

IsUserDefined

Access: Read-only

Data type: Boolean

Returns the parameter type. When TRUE, the parameter is a custom parameter. Otherwise, the parameter is a built-in parameter.

Name

Access: Read/Write

Data type: Variant

Sets and returns the parameter ID.

Note:

For the custom parameters, such as parameters for your policy scripts, this property must be a string. For the built-in parameters, this property takes integer values from the second column of the table in EDS_APE_PARAMENUM.

SettingID (Deprecated)

Access: Read/Write

Data type: Long integer

Deprecated. Sets and returns the parameter ID.

Note:

The SettingID property is considered obsolete. While this property is still supported in the current version of Active Roles, it may be removed in the future. When writing new scripts, you should avoid using this property. When modifying existing scripts, you are strongly encouraged to remove any dependency on this property. Use the Name property instead.

Value 

Access: Read/Write

Data type: String

Sets and returns the value of the Policy Object Entry parameter. Values of all parameters are described in the last column of the table in EDS_APE_PARAMENUM.

The Value property must always be a string.
For example, you should use the strings "0" and "1" instead of FALSE and TRUE for the Boolean-type parameters.

Values

Access: Read/Write

Data type: Array of strings

Sets and returns values for the multi-valued parameters of Policy Object Entry.

The Values property must always be an array of strings.
