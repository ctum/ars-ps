Accessing Properties of ADSI Objects

All ADSI objects have methods to retrieve the properties of the directory object that the ADSI object represents. There are a number of ways you can retrieve properties of an object:

  • Get a specific property by name. All ADSI objects implement the IADs interface, whose properties and methods allow you to access and modify properties of the directory object to which you have bound (see Binding to Directory Objects). In Automation applications, you can use the dot operator to access the so called basic IADs properties of directory objects listed in the table under IADs Properties in the IADs topic. 

    The following script will display the basic IADs properties for a specific user: 

    		Dim usr, ss
		ss="The User object properties: "+ VbCrLf
		' Bind to the user object
		Set usr=GetObject("EDMS://MyServer/CN=John Doe,OU=Sales,DC=mycompany,DC=COM")
		' Retrieve the user object basic IADs properties
		ss=ss+"Name: "+CStr(usr.Name) + VbCrLf
		ss=ss+"ADsPath: "+CStr(usr.ADsPath) + VbCrLf
		ss=ss+"Parent: "+CStr(usr.Parent)
		MsgBox ss

    The basic IADs properties allow you to identify a directory object and locate it in the directory hierarchy, but most of the useful information about the object is stored in other properties that are specific to that object's class. These properties can be accessed by their LDAP display names using both the Get/GetEx method of the IADs interface and the dot operator. The following script shows how to retrieve properties of the user object: 

    		Dim usr, location, otherNumbers,ss, lname
		on error resume next
		' Bind to the user object
		Set usr = GetObject("EDMS://MyServer/CN=John Doe,OU=Sales,DC=mycompany,DC=COM")
		' Load all the user object properties from Administration Service to the property cache
		usr.GetInfo
		' Retrieve the single value property ("l") using the Get method
		location = usr.Get("l")
		MsgBox "Locality: " + VbCrLf + Cstr(location)
		' Retrieve the single value property ("sn") using the dot operator
		lname=usr.sn
		MsgBox "Last Name: " + VbCrLf + Cstr(lname)
		' Retrieve the multiple value property ("otherHomePhone") using the GetEx method
		otherNumbers = usr.GetEx("otherHomePhone")
		ss=""
		For Each homeNum In otherNumbers
		ss=ss+ Cstr(homeNum) + VbCrLf
		Next
		MsgBox "Other home numbers: " + VbCrLf + ss

    Note:
    To avoid slow network calls, the Active Roles ADSI Provider caches properties locally. When you first bind to an object using the GetObject function, the object's properties are empty in the property cache. When you access property values using the dot operator or  the Get method, the Active Roles ADSI Provider loads all the property values for that object from the Active Roles Administration Service and places them in the property cache. All further retrieval of properties, using either syntax, will retrieve the values from the property cache.
    You can explicitly fill an object's property cache by using the GetInfo or GetInfoEx method of IADs.

  • Get a specified list of properties. The non-Automation clients (for example, C++ applications) can use methods of the IDirectoryObject to directly access properties of directory objects. For more information about using IDirectoryObject, refer to the MSDN Library.

  • Enumerate all properties on the object. The IADsPropertyList interface allows you to enumerate all the initialized properties on an object. For more information about using IADsPropertyList, refer to the MSDN Library.
