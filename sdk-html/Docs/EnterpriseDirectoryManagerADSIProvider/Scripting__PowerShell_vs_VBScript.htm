Scripting: PowerShell vs VBScript

Script development practices in PowerShell and VBScript are basically the same, although when developing PowerShell scripts, you should keep in mind some consideration which are specific for PowerShell as compared to VBScript. This topic highlights the key differences between PowerShell and VBScript when used to develop scripts for Active Roles.

Using Functions

In PowerShell, functions pipeline all the output instead of returning a single return value. Here is an example:

Function foo
{
"Start processing"
Get-qadobject "CN=Active Directory"
Return 0
}

This function will return three values instead of a single value.

To have function return only one value, you need to suppress unwanted output. The following function returns only one value:

Function foo
{
"Start processing" | out-null
Get-qadobject "CN=Active Directory" | out-null
Return 0
}

Properties in PowerShell

In PowerShell, the IEDM:Control property may be used only as a method. Here is an example:

 

[VBScript]
DirObj.Control("AllowApproval")="Confirm"

 

[PowerShell]
$DirObj.Control("AllowApproval", "Confirm")
