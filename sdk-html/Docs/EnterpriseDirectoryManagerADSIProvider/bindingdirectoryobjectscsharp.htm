Binding to Directory Objects

This section contains information about how to bind to directory objects.

You should first bind to a directory object in order to navigate its namespace, search for specific information, read or modify the object properties.

You can bind directly to an object by specifying the object's distinguished name or GUID in the binding string. After binding to an object, you can use the appropriate ADSI interfaces for that type of object. All ADSI objects support the IADs interface. Other interfaces are supported depending on the type of the object. For example, user objects support the IADsUser interface.

When specifying binding information, you may optionally supply additional binding characteristics, such as user name, password, domain or the domain controller name.

Active Roles ADSI Provider Binding String

The Active Roles ADSI Provider binding string has the following format:

EDMS://<DomainDCName>/<DistinguishedName>
<DomainDCName>
Optional parameter. Specifies the name of the domain controller or domain where you want the provider to search for an available Administration Service. For information about how the provider searches for Administration Service when this parameter is omitted, see Connecting to Service.
<DistinguishedName>
Required parameter. Specifies the LDAP distinguished name of the object to which you want to bind.

Notes:

The ":"(colon) is part of the provider name. The provider name is case sensitive: EDMS is different from edms.

The following table contains examples of Active Roles ADSI Provider binding strings.

Binding to an Object Using the DirectoryEntry Object

To bind to directory objects, we recommend that you create an instance of the DirectoryEntry class (hereafter referred to as DirectoryEntry object), and then set its property Path to the appropriate binding string.

The following code examples show how to bind to an object using the serverless binding string or the binding string with the specified domain/domain controller name:

[Visual Basic .NET] 

' The serverless binding string
Dim ent As New DirectoryEntry("EDMS://CN=John Doe,OU=Sales,DC=mycompany,DC=com")
' The binding string that specifies the domain or domain controller name
Dim ent As New DirectoryEntry("EDMS://DomainDCName/CN=John Doe,OU=Sales,DC=mycompany,DC=com",<userName>,<password>)

[C#] 

// The serverless binding string
DirectoryEntry ent = new DirectoryEntry("EDMS://CN=John Doe,OU=Sales,DC=mycompany,DC=com");
// The binding string that specifies the domain or domain controller name
DirectoryEntry ent = new DirectoryEntry("EDMS://DomainDCName/CN=John Doe,OU=Sales,DC=mycompany,DC=com",<userName>,<password>);

For more information about the DirectoryEntry class, refer to the MSDN library.

Remarks

To specify the credentials with which to connect to the Administration Service, set the UserName and Password properties of the DirectoryEntry object to the appropriate user login and password. When those properties are not set, the provider will connect to the Administration Service using the current user's credentials.

To force ADSI to connect to the Administration Service running on the server specified by the binding string, set the AuthenticationType property of the DirectoryEntry object to ADS_EDMSERVER_BIND=32768. Without the ADS_EDMSERVER_BIND flag, ADSI connects to any Administration Service available in the domain specified by the binding string.

For more information about how the provider selects the Administration Service to connect to, see Connecting to Service.

Binding to an Object Using GUID

An object distinguished name changes if the object is renamed or moved, while the object's objectGUID property never changes. The Active Roles ADSI provider allows you to bind to an object using the object GUID. The binding string format is as follows: 

EDMS://<DomainDCName>/<GUID=XXXXX>
<DomainDCName>
Optional parameter. Specifies the name of the domain controller or domain where you want the provider to search for an available Active Roles Administration Service. For information about how the provider searches for Administration Service when this parameter is omitted, see Connecting to Service.
<GUID=XXXXX>
Required parameter. XXXXX is the string representation of the hexadecimal value of the GUID.

To retrieve the GUID assigned to an object, use the NativeGuid property provided by the DirectoryEntry class. The following code example shows how to use the NativeGuid property.

[Visual Basic .NET] 

Dim ent As New DirectoryEntry("EDMS://CN=John Doe,OU=Sales,DC=mycompany,DC=com")
Dim ADGuid As [String] = ent.NativeGuid
' Rebind using ADGuid.
Dim x As New DirectoryEntry("EDMS://<GUID=" + ADGuid + ">")
Console.WriteLine("These should be the same: {0} == {1}", ent.Name, x.Name)

[C#] 

DirectoryEntry ent = new DirectoryEntry("EDMS://CN=John Doe,OU=Sales,DC=mycompany,DC=com");
String ADGuid = ent.NativeGuid;
// Rebind using ADGuid.
DirectoryEntry x = new DirectoryEntry("EDMS://<GUID="+ADGuid+">");
Console.WriteLine("These should be the same: {0} == {1}", ent.Name,x.Name);

Note:

Unlike the ADSI LDAP provider, the Active Roles provider supports all methods and properties of the IADs and IADsContainer interfaces even for objects obtained by binding using the object GUID.
