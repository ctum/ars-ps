Binding to an Object Establishing Credentials

The GetObject function allows you to bind to directory objects using only the current user's logon and security credentials. If you want to bind to directory objects using alternate credentials, use the OpenDSObject method provided by the IADsOpenDSObject interface. The OpenDSObject method allows you to employ binding strings that use the object's DN, GUID or SID (see Related Topics).

Note:

The OpenDSObject method can be used only in Automation languages, such as VBScript. For information about how to establish alternate credentials for binding in your .NET applications, refer to Binding to Directory Objects (ADSI .NET Tutorial).

Example Code (VBScript)

The following code snippet illustrates how to use the OpenDSObject method. 

Const ADS_EDMSERVER_BIND = 32768
Set openDS = GetObject("EDMS:")
Set objUser=openDS.OpenDSObject("EDMS://ServiceCompName/CN=John Doe,OU=Sales,DC=mycompany,DC=com", _
                                "Administrator", "password", _
                                 ADS_EDMSERVER_BIND)

Note:

Setting the flag in the OpenDSObject method to ADS_EDMSERVER_BIND ensures that the provider will connect to the Administration Service running on the ServiceCompName computer. For more information, please refer to IADsOpenDSObject and to Connecting to Service.
