Developing Scripts for Operation Requiring Approval

This topic explains how you can develop your custom scripts for performing operations requiring approval.

When a script attempts to perform an operation in Active Directory, Active Roles uses approval rules to check whether or not the changes this operation causes must be approved by approvers. If approval is not required, the system starts applying the changes at once. If the potential changes are subject to approval, in some cases, to perform the operation, your script must send the built-in control AllowApproval to the Administration Service with a request to perform the operation. The script also can prompt you either to confirm the operation or to cancel it. If you respond positively, an operation is created in the approval system, and the approval workflow thus starts.

Important

A script for operations requiring approval may fail to perform the required operations if you have not set the built-in control AllowApproval to appropriate values. For more information, refer to Using Active Roles Controls.

Setting the AllowApproval Control

If your script performs an operation in the security context of an Active Roles Admin, the AllowApproval control has no effect. In all other cases, if the operation the script will perform requires approval, you should set the AllowApproval control to one of the following values:

  • "Check" - the information about the approval will be returned, however, the operation will not be created in the approval system, and the approval workflow does not start.

  • "Confirm" - the information about the approval will be returned, the operation will be created in the approval system, and the approval workflow thus starts. The operation generates an approval task and assigns it to an Approver designated in the approval rule. The Approver completes the task by applying to it one of the two possible actions: Approve or Reject.

Sample Script (VBScript)

To clarify the use of the AllowApproval and OperationStatus controls, consider the following sample script.

This script performs the following steps:

  1. Binds to the specified user object and sets the control AllowApproval to "Check."

  2. Sends a request to modify the user property "description" to the Administration service.

  3. Retrieves the value of the control OperationStatus the Administration service returns.

  4. If this control returns the value "Pending" (i.e. the operation is subject to approval), prompts you confirm the operation or cancel it. Otherwise, the script completes the operation, returning the "Operation has been completed" message, and terminates.

  5. If you respond positively, the script sets the control AllowApproval to "Confirm", resends a request to modify the user property "description," returns the message "Operation has been assigned to Approver" and terminates. This creates an approval task and assigns it to an Approver designated in the approval rule.
    If you respond negatively, the script returns the message "Operation has been cancelled" and terminates.

Set usr = GetObject("EDMS://CN=Joghn Doe,OU=Software,DC=mycompany,DC=com")
 ' Set the control AllowApproval to Check
usr.Control ("AllowApproval", true) = "Check"
usr.Put "description", "database manager"
usr.SetInfo
  ' Verify if this operation requires approval
If "Pending" = usr.Control ("OperationStatus") Then
       ' If the operation requires approval, prompt you to proceed with the operation  
     If vbYes = MsgBox ("This operation requires approval. To create an approval task, click Yes.",vbYesNo+vbQuestion) Then
          usr.Control ("AllowApproval", true) = "Confirm"
          usr.SetInfo
          wscript.echo "Operation has been assigned to Approver."
         
        Else
          wscript.echo "Operation has been canceled."  
     End If
Else
     wscript.echo "Operation has been completed." 
End If
