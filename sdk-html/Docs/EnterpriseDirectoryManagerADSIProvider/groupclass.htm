group Class

The group class provides a formal description of a group object. In addition to the standard ADSI group class attributes, Active Roles defines the following group object attributes, which can be accessed using the Active Roles ADSI Provider.

Note:

For more information about the Group class, please refer to the MSDN Library.

 
edsaMember

Syntax: Multi-valued DN string.

Access: Read-only.

Searchable: No.

Returns a list of the DN of explicit members of the group including those for which this group is primary group.

 
edsaMemberIndirect

Syntax: Multi-valued DN string.

Access: Read-only.

Searchable: No.

Returns a list of the DN of objects that are members of the group due to the group nesting.

 
edsaManagerCanUpdateMembershipList

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

When this attribute is set to TRUE, the manager of this group can modify the 'member' attribute of the group.

 
edsaDelegatedPermissions

Syntax: Multi-valued DN string.

Access: Read-only.

Searchable: No.

Stores the information about permissions assigned to members of the group . Specifies a list of distinguished names of the Access Template Links (ATL). An ATL is included in that list only if the SID of the ATL's Trustee is among SIDs of the group's access token. Supports ASQ.

 
edsvaResourceURL

Syntax: DirectoryString, SingleValued.

Access: Read/Write.

Searchable: Yes.

Contains the address (URL) of the resource for a group that is used to control access to a Web-based resource.

Dynamic Groups

Active Roles provides rules-based groups called dynamic groups. Membership rules determine whether an object is a member of a dynamic group. The following attributes concerning the membership rules are available.

edsaIsDynamicGroup

Syntax: Boolean.

Access: Read/write.

Searchable: No.

Indicates whether this group is a dynamic group.

When this attribute is not set or returns FALSE, the group is a basic group.

When this attribute returns TRUE, the group is a dynamic group. In this case, the edsaDGConditionList attribute stores a list of membership rules specified for the group.

For a dynamic group, setting this attribute to FALSE converts the dynamic group to a basic group.
Setting this attribute to TRUE takes no effect. To convert a basic group to a dynamic group, define the desired membership rules in the edsaDGConditionList attribute.

 

edsaDGConditionsList

Syntax: DirectoryString.

Access: Read/write.

Searchable: No.

When this attribute is not set, no membership rules have been specified for the group. Otherwise, this attribute stores a list of membership rules specified for that group.

 

edsaDGOriginatingService

Syntax: CaseIgnoreString.

Access: Read/write.

Searchable: No.

This attribute specifies the Administration service that is used to re-calculate the membership list of the Dynamic Group upon any modification of membership rules or upon an administrator request to rebuild the Dynamic Group.

 

edsaCanBeMadeDynamicGroup

Syntax: Boolean.

Access: Read-only.

Searchable: No.

When set to TRUE,  the group can have membership rules. The membership rules are specified using the edsaDGConditionslist attribute.
