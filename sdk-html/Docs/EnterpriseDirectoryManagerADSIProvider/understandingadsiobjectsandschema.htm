Understanding ADSI Objects and Schema

The ADSI object model consists of COM objects. ADSI clients manipulate objects with the use of interfaces. An application called an ADSI provider implements the objects and interfaces defined by ADSI. The data exposed by the provider is organized in a namespace. In addition to objects and interfaces, the provider also implements the ADSI schema. The schema is used to provide metadata about the namespace structure and objects that are supported by the ADSI provider.

ADSI objects are COM objects used to represent the entities known to a directory service, such as user accounts, groups and computers (directory objects). ADSI defines different kinds (classes) of objects to represent different kinds of entities. Every ADSI object supports a number of COM interfaces that provide access to information about what the object represents. Each COM interface defines a set of object properties and methods.

The Active Roles ADSI Provider implements the IADs and IDirectoryObject interfaces for ADSI clients to access and modify directory objects through Active Roles. The IADs interface is intended for use by late-bound clients such as those written in Microsoft Visual Basic or VBScript. You can use dot notation or the Get and Put methods of an object to retrieve and modify property values in the property cache. The IDirectoryObject interface provides direct access to objects by early-bound clients such as those written in C and C++. The IDirectoryObject interface bypasses the property cache, thereby providing direct access to object properties through a single request, rather than through individual get and put calls. For a detailed list of interfaces exposed by Active Roles-specific objects, see ADSI Provider Reference.

The Active Roles Schema contains the definition of every type of object known to Active Roles. The Active Roles client applications can browse through the schema to discover the features of any given ADSI object. The Active Roles ADSI provider supports three kinds of schema objects: classes, properties, and syntaxes, each respectively exposing the schema management interfaces IADsClass, IADsProperty, and IADsSyntax.

The Active Roles ADSI Provider enables ADSI clients to find out what object classes are available by enumerating the schema object of the EDMS namespace and getting a list of class objects, property objects, and syntax objects. To find out more about any unknown object, ADSI clients can look up its associated schema objects. For example, to access an object's schema class, obtain the ADsPath string using the object's Schema property and use that string to obtain the IADsClass interface on the object's schema class. To access individual attribute definitions, inspect the schema attribute objects for individual properties with the IADsProperty interface. And the Active Roles ADSI Provider lets you use the IADsSyntax interface to identify the syntax needed to represent an object's data.
