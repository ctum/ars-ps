IEDMPermissionEntry Properties

The IEDMPermissionEntry interface has the following properties.

Property

Description

AccessMask

Access: Read/Write

Data type: Long integer

Sets and returns the access rights to directory objects (a category of Permission). This property can take any value that is a result of the bitwise OR operation on any of values from the EDS_RIGHTS_ENUM enumeration. 

To specify generic access permissions, such as generic read or write access to directory objects, you should set this property to one of the values listed in Generic Permissions. 

AteType

Access: Read/Write

Sets and returns a type of Permission (allow or deny). For permissions that allow administrative operations, this property must be set to 0. For permissions that deny administrative operations, this property must be set to 1.

ObjectTypeADsPath

Access: Read/Write

Data type: String

Sets and returns an object property to which Permission applies. The object class can be set using the InheritedObjectTypeADsPath property.

This property must be set using the following format:
EDMS://<The property LDAP display name>,Schema

For example, to apply Permission to the locality property (LDAP display name "l"), use the following string:
"EDMS://l,Schema"

Note:

When the ObjectTypeADsPath property is not specified, Permission applies to all properties.

InheritedObjectTypeADsPath

Access: Read/Write

Data type: String

Sets and returns the object class to which Permission applies. The object property can be set using the ObjectTypeADsPath property.

This property must be set using the following format:
EDMS://<Class name>,Schema

For example, to apply Permission to the Group class object, use the following string:
"EDMS://group,Schema"

Note:

When the InheritedObjectTypeADsPath property is not specified, Permission applies to all object classes.

ObjectType

Access: Read/Write

Data type: String

Sets and returns a value of the Schema-Id-Guid attribute for the object property identified using ObjectTypeADsPath. The Schema-Id-Guid attribute uniquely identifies the object property in the schema.

Note:

When you have specified the ObjectTypeADsPath property, the Active Roles ADSI Provider automatically calculates the ObjectType property. The use of the ObjectType property is recommended only for advanced programmers.

InheritedObjectType

Access: Read/Write

Data type: String

Sets and returns the object class to which Permission applies. The object property can be set using the ObjectTypeADsPath property.

Note:

When you have specified the InheritedObjectTypeADsPath property, the Active Roles ADSI Provider automatically calculates the InheritedObjectType property. The use of the InheritedObjectType property is recommended only for advanced programmers.

Sample Code (VBScript)

The following sample code illustrates how to add Permission Entries to Access Template.

option explicit
 ' DN of Access Template
const strAccessTemplateDN = "EDMS://CN=MyAT,CN=Access Templates,cn=Configuration"
const EDS_RIGHT_DS_READ_PROP = 16
const EDS_RIGHT_DS_WRITE_PROP = 32
const ACCESS_ALLOWED_ACE_TYPE = 0
Dim atObject
 ' Bind to Access Template
Set atObject = GetObject(strAccessTemplateDN)
Dim NewATE
 ' Create a new Permission Entry
set NewATE = atObject.CreatePermissionEntry
 ' Set properties of the newly created Permission Entry: read all properties for all classes
NewATE.AteType = ACCESS_ALLOWED_ACE_TYPE
NewATE.AccessMask = EDS_RIGHT_DS_READ_PROP OR EDS_RIGHT_DS_WRITE_PROP
 ' Add Permission Entry to Access Template
atObject.AddPermissionEntry(NewATE)
Set NewATE = Nothing
 ' Create a new Permission Entry
set NewATE = atObject.CreatePermissionEntry
 ' Set properties of the newly created Permission Entry: write description for user
NewATE.AteType = ACCESS_ALLOWED_ACE_TYPE
NewATE.AccessMask = EDS_RIGHT_DS_WRITE_PROP
NewATE.InheritedObjectTypeADsPath = "EDMS://user,Schema"     ' Apply to the User class
NewATE.ObjectTypeADsPath = "EDMS://description,Schema"       ' Apply to the "description" property
 ' Add Permission Entry to Access Template
atObject.AddPermissionEntry(NewATE)
Set NewATE = Nothing
 ' Commit changes
atObject.SetInfo

This script adds two permission entries to the MyAT Access Template. The first permission allows a Trustee to read/write all properties of any object. The second permission allows a Trustee to modify the description property of the user objects. For information about how to designate a Trustee, and for more examples of the VBScript codes, refer to Managing Access Templates.
