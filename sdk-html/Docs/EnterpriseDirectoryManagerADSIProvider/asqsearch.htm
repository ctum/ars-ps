Search Using an Attribute Scope Query

The Active Roles ADSI Provider supports the ADO search for directory objects using an attribute scope query (referred to as ASQ).

The ASQ is a new search preference that enables a search of an object's distinguished name valued attributes to be performed. The attribute to search must be of the DN type (syntax ID: 2.5.5.1). Examples of the DN type attributes are: member, memberOf, edsaMember, etc. When the search is performed, the provider will enumerate the distinguished name values of the attribute and perform the search on the objects represented by the distinguished names. For example, if an attribute scoped search is performed on the edsaMember attribute of a Managed Unit, the provider will enumerate the distinguished names in the edsaMember attribute and search each of the members of the Managed Unit for the specified search criteria.

To specify the ASQ, the search scope must include the distinguished name of the object whose the DN type attribute you want to search followed by the string:
 "//EDS_SEARCHPREF_ATTRIBUTE_QUERY=DN_valued_attr"
where DN_valued_attr is referred to the LDAP display name of the DN type attribute to search.

Important:

If an ASQ is performed on an attribute that is not of the DN type, the search will fail. ASQ also requires that the search depth be set to Base. Otherwise, the search will fail.

Examples of Search Strings

Let us consider the following search string that allows you to search the member attribute of the Administrators group of the mycompany.com domain.

"<EDMS://CN=Administrators,CN=Builtin,DC=mycompany,DC=com//EDS_SEARCHPREF_ATTRIBUTE_QUERY=member>;(objectClass=*);mail;Base"
Optionally, the search string may include Active Roles controls (InControls) to send to the Administration Service. For example, the following search string allows you to search indirect members of the Domain Guests group of the mycompany.com domain.

"<EDMS://CN=Domain Guests,CN=Builtin,DC=mycompany,DC=com//EDS_SEARCHPREF_ATTRIBUTE_QUERY=member//IndirectMembership-GetData=1//PrimaryGroup-GetData=1>;(objectClass=*);mail;Base"

When performing an ASQ on the member and memberOf attributes of groups, the search string can include the following Active Roles controls:

  • PrimaryGroup-GetData

  • IndirectMembership-GetData

  • ScheduledLink-GetPending

  • ScheduledLink-GetStartEndTime

Important:

The ASQ on the edsaMember, edsaMemberIndirect, edsaMemberOf, and edsaMemberOfIndirect attributes of groups and users is considered obsolete. Instead, perform ASQ search on the member and memberOf attributes using an appropriate combination of Active Roles controls.

Sample Script (VBScript)

The following sample scripts illustrate how to perform an ASQ on the Managed Unit and group attributes.

Listing the Mail-Enabled Members of Managed Unit

The following script shows how to find all the mail-enabled members of the specified Managed Unit. The search result set contains  the distinguished name, class, and email address of all the mail-enabled objects contained in the edsaMember attribute of the "MyMU" Managed Unit.

' Set the DN of Managed Unit
strMU = "CN=MyMU,CN=Managed Units,cn=Configuration"
Set oConnect = CreateObject("ADODB.Connection")
oConnect.Provider = "ADsDSOObject"
oConnect.Open "DS Query", "", ""
Set Command = CreateObject("ADODB.Command")
Set Command.ActiveConnection = oConnect
 ' Specify the search criteria
Command.CommandText = _
"<EDMS://" + strMU + "//EDS_SEARCHPREF_ATTRIBUTE_QUERY=edsaMember>;(objectClass=*);distinguishedName,mail;base"
ss = "Managed Unit: " + " contains the following mail-enabled recipients: " + vbNewLine
Set rs = Command.Execute
Do Until rs.EOF
    If rs.Fields("mail").Value <> "" Then
        ss = ss + "Object: " + rs.Fields("distinguishedName").Value  + vbNewLine
        ss = ss + "Object class: " + OClass(rs.Fields("distinguishedName").Value)  + vbNewLine
        ss = ss + "Email: " + rs.Fields("mail").Value + vbNewLine
    End If
    rs.MoveNext
Loop
MsgBox ss
Function OClass(byVal strDN)
 ' This function gets the object class
  Set obj = GetObject("EDMS://" + strDN)
  OClass = obj.Class
  Set obj = Nothing
End Function

Listing Indirect Members of Group

The following script shows how to find indirect members of the specified group.

The script performs the ASQ on the member attribute of the "MyGroup" group with the IndirectMembership-GetData and PrimaryGroup-GetData controls specified. The search result set contains values of the distinguishedname and edsva-IsIndirectMembership attributes of the found objects. The script outputs distinguished names of all indirect members of the "MyGroup" group.

' --- Set DN of the group to search
const strDN = "CN=MyGroup,OU=Custom Groups,DC=mycompany,DC=com"
Set oConnect = CreateObject("ADODB.Connection")
oConnect.Provider = "ADsDSOObject"
oConnect.Open "DS Query", "", ""
Set Command = CreateObject("ADODB.Command")
Set Command.ActiveConnection = oConnect
' --- Specify the search criteria
Command.CommandText = "<EDMS://" & strDN & _
"//EDS_SEARCHPREF_ATTRIBUTE_QUERY=member//IndirectMembership-GetData=1//PrimaryGroup-GetData=1>;(objectClass=*);distinguishedName,edsva-IsIndirectMembership;base"
Dim output
output = "Indirect members of the " & strDN & " group: " & vbNewLine & vbNewLine
' --- Process the search result set
Set rs = Command.Execute
Do Until rs.EOF
if rs.Fields("edsva-IsIndirectMembership").Value = true  Then
output = output & rs.Fields("distinguishedName").Value & vbNewLine
End If
rs.MoveNext
Loop
MsgBox output
