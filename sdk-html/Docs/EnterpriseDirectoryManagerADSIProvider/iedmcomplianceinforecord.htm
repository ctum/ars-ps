IEDMComplianceInfoRecord

Using this interface, you can do the following:

  • Get information on Policy Object that affects an attribute of the processed object. The following information is available: the GUID of Policy Object, the name of a Policy Entry used by the administrative policy, and the administrative note to be displayed to the Active Roles clients.

  • Get a status of the administrative policy checking.

Properties

The IEDMComplianceInfoRecord interface has the following properties:

Property

Description

Message

Returns the message that informs the Active Roles clients about results of checking the administrative policy.

PolicyEntryName

Returns the name of the policy entry identifying a policy in the Policy Object. That name is displayed on the Policies tab in the Properties dialog box of the appropriate Policy Object.

PolicyObjectGuid

Returns GUID of the Policy Object.

PropertyName

Returns the Ldap display name of the attribute whose value is checking.

Status

Returns a status of the administrative policy checking. This property returns one of values from the EDS_POLICY_COMPLIANCE_ENUMenumeration.

Example Code (VBScript)

The following  code snippet illustrates how to use the IEDM, IEDMComplianceInfoRecord interfaces to get information about violations of the administrative policy affected a user object. 

Dim cont
Dim usr,strUser,Policy, PP, PP1,PP2
str="Violations of the administrative policy:" + vbCrLf
strUser="EDMS://CN=Test,OU=Sales,DC=mydomain,DC=com"
Set usr=GetObject(strUser)
usr.Put "l", "Berlin"
usr.Put "department", "Berlin Software Department"
usr.Put "co", "Germany"
set PP = usr.CheckPropertyValues
' An implicit call of _NewEnum of IEDMComplianceInfoList
For Each PP1 in PP
   ' An implicit call of _NewEnum of IEDMComplianceInfo
   For Each PP2 in PP1
      str = str + PP2.PropertyName + " " + Cstr(PP2.Status) + " " + PP2.Message + " "_
      + CStr(PP2.PolicyObjectGuid) + " " + vbCrLf
   Next
Next
MsgBox str
