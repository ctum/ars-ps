IEDMDynamicGroup Properties

The following table describes properties provided by the IEDMDynamicGroup interface.

Property

Description

IsDynamicGroup

Access: Read/Write

Data type: Boolean

Sets and returns the value of the edsaIsDynamicGroup property of the group object.

To convert a dynamic group to a basic group, set this property to FALSE, and then call the SetInfo method of the IADs interface. This clears the MembershipRules property.
We don't recommend that you set the IsDynamicGroup property to TRUE. This property is set automatically to TRUE after you add any rule to the MembershipRules property and call the SetInfo method of the IADs interface.

MembershipRules (Deprecated)

Access: Read/Write

Sets and returns an array of membership rules for dynamic group. The elements of the array can be processed using the IEDMMembershipRuleinterface.

Active Roles supports six predefined types of membership rules for dynamic groups. The membership rules are identical to those used for Managed Units. For information about how to create a membership rule of the specified type, see IEDMMembershipRule.

Note:

The MemberShipRules property is considered obsolete. While this property is still supported in the current version of Active Roles, it may be removed in the future. When writing new scripts, you should avoid using this property. When modifying existing scripts, you are strongly encouraged to remove any dependency on this property. Use the MemberShipRuleCollection property instead.

MemberShipRuleCollection

Access: Read-only.

Returns an object used to represent the IEDMMembershipRuleCollection interface. The returned object comprises information about membership rules for Managed Unit. For more information, see IEDMMembershipRuleCollection.

Example Code (VBScript)

The following code snippet illustrates how you can convert a basic group to a dynamic group.

const strGroupDN = "EDMS://CN=TestGroup,OU=Sales,DC=mycompany,DC=com"
Set obj = GetObject(strGroupDN)
 ' Return a reference to membership rule collection for this group
Set RuleCollection = Obj.MembershipRuleCollection
ss = "Current type of group: " & Cstr(obj.IsDynamicGroup) & vbNewLine
 ' Create a new Include by Query rule
Set rule1 = CreateObject("EDSIManagedUnitCondition")
rule1.Base = "EDMS://OU=Sales,DC=mycompany,DC=com"
rule1.Filter = "(|(objectClass=user)(objectClass=computer))"
rule1.Type=1 
 ' Add a newly created rule to the rule collection 
RuleCollection.Add rule1
Obj.SetInfo
 ' Create a new Include Explicitly rule
Set rule2 = CreateObject("EDSIManagedUnitCondition")
rule2.Base = "EDMS://CN=Domain Guests,CN=Users,DC=mycompany,DC=com"
rule2.Filter = vbNullString
rule2.Type=3
 ' Add a newly created rule to the rule collection
RuleCollection.Add rule2
Obj.SetInfo
ss = ss + "Group type after adding the membership rules: " & Cstr(obj.IsDynamicGroup)
MsgBox ss

This script converts the group TestGroup to a dynamic group that includes two membership rules:

  • The Include by Query membership rule: Populates the group membership list with all users and computers that reside into the Organizational Unit Sales

  • The Include Explicitly membership rule: Adds statistically the group Domain Guests to the membership list of the group TestGroup
