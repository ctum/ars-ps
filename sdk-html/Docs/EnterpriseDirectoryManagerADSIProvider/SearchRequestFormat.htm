OperationSearchRequest Format

This topic provides information about the format of a request to perform a search operation you can send to the Administration Service using the ExecuteRequest method of the IEDM interface. The request is in an Extensible Markup Language (XML) format. XML elements you can use are listed below.

XML Elements

OperationSearchRequest

Describes the search request settings.

Parent Elements: None

Child Elements: Filter

Attributes:

Name

Description

includeTasks

Optional. boolean.
The default value is FALSE.

pageSize

Required. int.

Sets the maximum number of operations to be returned in OperationSearchResponse.
Set this attribute to "1"
 
Note
 

The OperationSearchRequest element uses the namespace declarator xmlns. The declarator must be set to "urn:schemas-quest-com:ActiveRolesServer"(see Example).

Filter

Defines the search criteria to be used in the XML request.

Parent Elements: OperationSearchRequest

Child Elements: None

Attributes:

Name

Description

Operator

Required. string.
Defines the search filter operator. This attribute can take one of the following values:

  • And

  • Or

  • Not

  • EqualityMatch

  • GreaterOrEqual

  • LessOrEqual

  • Present

  • StartsWith

  • EndsWith

Name

Required. string. Operation property name.

For example, this attribute can take one of the following values:

  • "OperationID" - specifies a search by the operation ID

  • "OperationStatus" - specifies a search by the operation status

  • "InitiatorSid" -specifies a search by the operation initiator SID

  • "TargetGuid" - specifies a search by the target object GUID

StringValue

Required. string.

Specifies the value to search for.

 

Important:

All names of attributes are case sensitive.

Example

The following code snippet illustrates a request to search for the operation with a specific operation ID. In this example, the operation ID is set to "1-212."

   <OperationSearchRequest xmlns="urn:schemas-quest-com:ActiveRolesServer" pageSize="1">
      <Filter operator="EqualityMatch" name="OperationID" stringValue="1-212" />



  </OperationSearchRequest>
