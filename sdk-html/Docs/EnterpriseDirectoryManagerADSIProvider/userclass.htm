user Class

The user class provides a formal description of a user object that is an Active Directory domain user account. In addition to the standard ADSI user class attributes, Active Roles defines the following user object attributes, which can be accessed using the Active Roles ADSI Provider.

Note:

For more information about the User class, please refer to the MSDN Library.

edsaAccountIsDisabled

Syntax: Boolean.

Access: Read/Write.  

Searchable: No.

Specifies the UF_ACCOUNTDISABLE flag for the userAccountControl property. This flag prevents the user from logging on with the user account. This flag is associated with the check box Account is disabled on the Account tab in the User Properties dialog box.

 

edsaAccountIsSensitiveAndCanNotBeDelegated 

Syntax: Boolean.

Access: Read/Write.  

Searchable: No.

Specifies the user account control flag UF_NOT_DELEGATED. When set to TRUE, this flag marks the account as sensitive, which means that other users cannot act as delegates of this account. This flag is associated with the check box Account is sensitive and cannot be delegated on the Account tab in the User Properties dialog box.

 

edsaAccountIsTrustedForDelegation 

Syntax: Boolean.

Access: Read/Write.  

Searchable: No.

Specifies the user account control flag UF_TRUSTED_FOR_DELEGATION. When this flag is set to TRUE, the account is enabled for delegation. This allows a service running under the account to assume a client's identity and authenticate as that user to other remote servers on the network. This flag is associated with the check box Account is trusted for delegation on the Account tab in the User Properties dialog box.

 

edsaAccountLockedOut 

Syntax: Boolean.

Access: Read/Write.  

Searchable: No.

Indicates whether the user account is locked out. Setting this attribute to FALSE results in unlocking the user account. Note that setting this attribute to TRUE takes no effect.

 

edsaDelegatedPermissions 

Syntax: Multi-value DN string.

Access: Read-only.  

Searchable: No.

Comprises information about permissions assigned to the user. Specifies a list of distinguished names of the Access Template Links (ATL). An ATL is included in that list only if the SID of the ATL's Trustee is among SIDs of the user's access token. Supports ASQ.

 

edsvaDeprovisionCommands 

Syntax: Integer

Access: Read-only.  

Searchable: No.

This virtual attribute can take one of the following values:

1 - Indicates that the edsvaDeprovisionStatus attribute is not set, and the Active Roles user has the 'Deprovision' extended right.

2 - Indicates that the edsvaDeprovisionStatus attribute is set to some value.

 

edsvaDeprovisionDeletionDate 

Syntax: GeneralizedTime

Access: Read/Write 

Searchable: Yes.

This virtual attribute specifies the date when Active Roles will delete the deprovisioned user account after a retention period. The date is specified by the built-in policy "User account permanent deletion".

 

edsvaDeprovisionReportXML

Syntax: CaseIgnoreString

Access: Read-only.  

Searchable: No.

This virtual attribute contains an XML report on the user deprovisioning results.

 

edsvaDeprovisionStatus 

Syntax: Integer

Access: Read/Write  

Searchable: Yes.

Possible values:
0 or not set - the user is not deprovisioned;
1 - if the deprovisioning operation succeeded;
2 - if the deprovisioning operation failed.

 

edsvaDeprovisionType 

Syntax: Integer

Access: Write-only.  

Searchable: No.

Setting this virtual attribute to 1 sends a request to deprovision the user to the Administration Service.

 

edsaDoNotRequireKerberosPreauthentication 

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies the user account control flag UF_DONT_REQUIRE_PREAUTH. When this flag is set to TRUE, the account does not require Kerberos pre-authentication for logon. This flag is associated with the check box Do not require Kerberos preauthentication on the Account tab in the User Properties dialog box.

 

edsvaOneViewReportXM

Syntax: : CaseIgnoreString, SingleValued.

Access: Consturcted, ReadOnly. 

Searchable: No.

Contains XML data for Entitlement Profile report.

 

edsaPassword

Syntax: DirectoryString.

Access: Write-Only.

Searchable: No.

Specifies the user's password. The password is stored in UTF-8 format using an encryption method specified with the user account properties.

 

edsaPasswordNeverExpires 

Syntax: Boolean.

Access: Read/Write.  

Searchable: No.

Specifies the user account control flag UF_DONT_EXPIRE_PASSWD. When set to TRUE, this flag prevents the password from expiring. This flag is associated with the check box Password never expires on the Account tab in the User Properties dialog box.

 

edsaSmartCardIsRequired 

Syntax: Boolean.

Access: Read/Write.  

Searchable: No.

Specifies the user account control flag UF_SMARTCARD_REQUIRED. When set to TRUE, this flag requires that the user log on to the user account with a smart card. This flag is associated with the check box Smart card is required for interactive logon on the Account tab in the User Properties dialog box.

 

edsaStorePasswordUsingReversibleEncryption 

Syntax: Boolean.

Access: Read/Write.  

Searchable: No.

Specifies the user account control flag UF_ENCRYPTED_TEXT_PASSWORD_ALLOWED. When set to TRUE, the user's password is stored in Active Directory using reversible encryption. This flag is associated with the check box Store password using reversible encryption on the Account tab in the User Properties dialog box.

 

edsaUPNPrefix 

Syntax: DirectoryString.

Access: Read/Write.  

Searchable: No.

Specifies the prefix part of the user logon name stored in the userPrincipalName property of the user object. The prefix part is the user account name. For example, if the user logon name is JSmith@company.com, the edsaUPNPrefix property stores "JSmith".

 

edsaUPNSuffix 

Syntax: DirectoryString.

Access: Read/Write.  

Searchable: No.

Specifies the suffix part of the user logon name stored in the userPrincipalName property of the user object. The suffix part is the user domain name. For example, if the user logon name is JSmith@company.com, the edsaUPNSuffix property stores "company.com".

 

edsaUseDESEncryptionTypesForThisAccount 

Syntax: Boolean.

Access: Read/Write.  

Searchable: No.

Specifies the user account control flag UF_USE_DES_KEY_ONLY. When set to TRUE, this flag restricts the user to use only Data Encryption Standard (DES) encryption types for keys. This flag is associated with the check box Use DES encryption types for this account on the Account tab in the User Properties dialog box.

 

edsaUserCannotChangePassword 

Syntax: Boolean.

Access: Read/Write.  

Searchable: No.

Specifies the user account control flag UF_PASSWD_CANT_CHANGE. When set to TRUE, this flag prevents the user from changing the password. This flag is associated with the check box User cannot change password on the Account tab in the User Properties dialog box.
