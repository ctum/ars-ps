IEDM

Every directory object supports the IEDM interface, which provides a set of properties and methods allowing you to do the following:

  • Check whether values of the object's attributes are in compliance with the administrative policy

  • Get information on administrative policies that apply to directory objects

  • Set and get values of the Active Roles controls

  • Set and get values of the edsaParameters property of the Active Roles-specific objects. The edsaParameters property is used by scheduled tasks and policy scripts.

Methods

The IEDM interface defines the following methods:

Method

Description

CheckPropertyValues

Returns an object used to represent the IEDMComplianceInfoList interface. This method allows you to check whether values of the object's attributes are in compliance with the administrative policy.

ExecuteRequest

Returns a response the Administration Service sends after performing the specified request. The request and response are saved in XML-format.

GetPolicyInfoList

Returns an object used to represent the IEDMPolicyInfoList interface. This method gets information on administrative policies that apply to the appropriate directory object.

RefreshSchema

Refreshes the Active Roles ADSI provider schema.

Properties

The IEDM interface has the following properties:

Property

Description

Control

Read/Write. Sets and returns values of the Active Roles controls.

Parameters

Read/Write. Sets and returns an object used to represent the IEDMParameters interface. This interface provides properties and methods that allow you to process values of parameters stored into the the edsaParameters property of the Active Roles-specific objects. 

DN

Read. Returns distinguished name of a directory object.
