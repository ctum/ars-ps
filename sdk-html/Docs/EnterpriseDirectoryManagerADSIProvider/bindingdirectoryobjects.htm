Binding to Directory Objects

In Active Roles, the act of associating a programmatic object with a directory object is known as binding. You should first bind to a directory object in order to navigate its namespace, search for specific information, read or modify the object properties. In this documentation, the term directory object refers to both Active Directory objects and those specific to Active Roles, such as Policy Objects, Access Templates, etc.

You can bind directly to an object by specifying the object's distinguished name, GUID or SID. After binding to an object, you can use the appropriate ADSI interfaces for that type of object. All ADSI objects support the IADs interface. Other interfaces are supported depending on the type of the object. For example, user objects support the IADsUser interface.

You can bind to a container object in order to browse through the objects that reside in the container. Container objects, such as Organizational Units, expose the IADsContainer interface allowing you to enumerate and manage the contained objects.

When specifying binding information, you may optionally supply additional binding characteristics, such as user name, password, or the domain controller name.

How to Bind to a Directory Object

The method for programmatically binding to a directory object depends on the programming technology that is used. For example, in VBScript, you can bind to directory objects using the GetObject function or the OpenDsObject method provided by the IADSOpenDSObject interface. If you use the .NET technologies (C#, Visual Basic .NET), use the DirectoryEntry object to bind to directory objects.

For more information about binding to directory objects with specific programming technologies, see the topics in the sections Tutorial: ADSI Scripting (VBScript) and Tutorial: ADSI .NET.

Binding String

All bind methods require a binding string.

The Active Roles ADSI Provider binding string has the following format:

EDMS://<DomainDCName>/<Object name>
<DomainDCName>
Optional entry. Specifies the name of the domain controller or domain where you want the provider to search for available Administration Service. For details, see Connecting to Service.
<Object name>
Required entry. Represents a specific directory object. The object name can be a distinguished name (DN), an object's GUID or SID.

Notes:

  • In the binding string above, EDMS: specifies the Active Roles ADSI provider. The ":"(colon) is part of the provider name. The provider name is case sensitive: EDMS is different from edms.

  • Examples of binding strings that use a DN, GUID or SID are available in the appropriate topics within this section.
