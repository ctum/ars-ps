Binding to an Object Using a SID

In addition to a GUID, it is also possible to bind to an object using the object Security Identifier (SID). The object SID is stored in the objectSID attribute.

To bind to a directory object using the object's SID, use the following binding string: 

EDMS://<DomainDCName>/<SID=XXXXX>
<DomainDCName>
Optional parameter. Specifies the name of the domain controller or domain where you want the provider to search for available Administration Service. For information about how the provider searches for Administration Service when this parameter is omitted, see Connecting to Service.
<SID=XXXXX>
Required parameter. XXXXX is a string specified in a form where each character in the string is the hexadecimal representation of each byte of the SID. To convert the value of the objectSID attribute to the SID binding string, use the GetSidString method provided by the IEDMOctetString interface. 

The SID string can take the form:

   <SID=S-1-5-21-3437999760-1408407091-3255257847-1307>
or
   <SID=01050000000000051500000090B6EBCC3396F253F74A07C21B050000>

Example

The following VBScript code snippet illustrates how to bind to the user John Doe using the user's SID: 

Set OctetString = CreateObject("AelitaEDM.EDMOctetString")
Set Obj = GetObject("EDMS://CN=John Doe,OU=Sales,DC=mycompany,DC=com")
Obj.GetInfoEx "objectSID",0
OctetString.Set(Obj.Get("objectSID"))
Set obj = Nothing
 ' Convert the user's SID 
strSID = OctetString.GetSidString()
 ' Rebind to the user object using the user's SID
Set obj=GetObject( "EDMS://<SID=" & strSID & ">")
MsgBox "The object's parent container: " & obj.Parent

This code allows you to get the DN of the container where the user John Doe resides.

Note:

Unlike the ADSI LDAP provider, the Active Roles ADSI provider supports all methods and properties of the IADs and IADsContainer interfaces even for objects obtained by binding using the object's SID.
