Understanding Active Roles ADSI Provider

The Active Roles ADSI Provider maintains the implementation of objects for the namespace supported by Active Roles. The namespace is identical to that of the system LDAP ADSI provider. With the Active Roles ADSI Provider, client applications and scripts are concerned only with getting and using standard interfaces on an object, and not with the details of how Active Roles processes the requested operations.

The Active Roles ADSI Provider supports the following features.

  • ADsPath strings whose format is similar to that of Microsoft's LDAP provider; the only difference is that ADsPath strings for the Active Roles ADSI Provider start with EDMS: rather than LDAP:

  • A top-level namespace object that contains the root nodes for the EDMS: namespace and supports the interfaces IADsOpenDSObject and IADs.

  • The IADs interface - includes a property cache implementation through the methods GetInfo, GetInfoEx, and SetInfo. Operations that get or set object properties are performed using a property cache. The cached values are sent to Active Roles during the SetInfo method call.

  • The IADsPropertyList, IADsPropertyEntry and IADsPropertyValue interfaces for directly accessing and enumerating the properties in the property cache.

  • The IADsContainer interface on ADSI container objects - provides the ability to create, delete and manage contained ADSI objects.

  • A schema class container object and the syntax, property, and schema class objects that support the IADsSyntax, IADsProperty, and IADsClass interfaces, respectively.

  • The IADsMembers interface for collections of ADSI objects, such as security groups.

  • The IDispatch interface - the Automation interface for languages like Visual Basic that do not use compile-time binding; related to this the Active DS Type Library information is to be used.

  • The IDirectoryObject interface for non-Automation clients - bypasses the property cache and uses on-the-wire protocol for maximum performance.

  • The IDirectorySearch interface to gain access to a subset of OLE DB read-only functionality, which makes it possible to query directory services through Active Roles with no extra efforts.

  • The IEDM interface - defines the basic object features, that is, properties and methods, of any ADSI object or object specific to Active Roles. Every ADSI object or Active Roles- specific object must support this interface.

  • The IEDMPolicyInfoList, IEDMPolicyInfo interfaces for processing information on administrative policies set on directory objects.

  • The IEDMComplianceInfoList, IEDMComplianceInfo and IEDMComplianceInfoRecord interfaces for checking whether the object's attributes are in compliance with the administrative policy.

  • The IEDMManagedUnit and IEDMMembership interfaces for administrating Managed Units.

  • The IEDMPolicyObject, IEDMPolicyEntry and IEDMPolicySetting interfaces for managing Policy Objects.

  • The IEDMAccessTemplate, IEDMPermissionEntry, IEDMATLinks, and IEDMATLink interfaces for managing Access Templates and Access Template Links.

  • The IEDMClass interface for managing schema class objects.
