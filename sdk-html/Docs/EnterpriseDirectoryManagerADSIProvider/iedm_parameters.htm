Parameters Property

Read/Write. Sets and returns an object used to represent the IEDMParameters interface. This interface provides properties and methods that allow you to process values of parameters stored into the edsaParameters property of the Active Roles-specific objects.

For example, using the Parameters property, you can pass parameter information to scheduled tasks at run time. For more information, see the sample code below.

Obj.Parameters([PName])[=Value]

Arguments

Obj
Required. Always a name of the Active Roles-specific object whose parameters you want to access. As a rule, you will use the Parameters property only for scheduled tasks (the object class edsScheduledTask).
PName
[in] Optional. String. Contains the parameter name. When provided, allows you to implicitly call the default property Value of IEDMInterface. The property Value sets and returns values of the parameter identified with PName (see the sample policy script below).
Value
[in] Optional. When provided, assigns the specified value to the parameter identified by the PName argument.

Remarks

The Parameters property can be used only for the Active Roles-specific objects, such as Access Templates, scheduled tasks, etc.

Example Code (VBScript)

This scheduled task script reads the scheduled task's parameter paramDN that contains the distinguished name of a container. The script enumerates all users that reside within that container and deletes only users that have the "description" property set to "To be deleted". 

' Read the scheduled task parameter
strDN = Task.DirObj.Parameters("paramDN")
 ' Bind to a container
Set objCon= GetObject("EDMS://" & strDN)
 ' Apply the filter
objCon.Filter=Array("user")
 ' Enumerate the container's child user objects
For Each obj In objCon
    If obj.Description = "To be deleted" Then   
        objCon.Delete "user", obj.Name
    End If    
Next
objCon.SetInfo

For more information about how to use this script, refer to Using parameters in Scheduled Task Scripts.
