ADSI Provider Reference

This section contains some reference materials for programming with the Active Roles ADSI provider. Since all interfaces and methods of the Active Roles ADSI Provider exhibit the same characteristics as those of the system LDAP provider, you can use ADSI Reference in the MSDN Library for additional information.

Interfaces

The Active Roles ADSI Provider supports a set of interfaces that fall into these categories:

  • Core Interfaces provide the basic object management functions of ADSI objects. The core functions include providing an entry point into a directory store, loading properties from the Active Roles Administration Service into the property cache, and sending changes to Administration Service.

  • Schema Interfaces provide methods for managing the directory schema.

  • Persistent Object Interfaces provide methods for managing persistent data in such directory objects as user accounts, groups, computer accounts, and organizational units.

  • Property Cache Interfaces provide methods for manipulating properties in the property cache. 

  • Non-Automation Interfaces allow non-Automation clients (for example, C/C++ applications) low-overhead access to directory objects by providing an access to methods for managing directory objects and searching for directory objects.

  • Active Roles-Specific Interfaces provide methods for managing the directory objects and objects specific to Active Roles, such as Managed Units, Policy Objects and Access Templates.
