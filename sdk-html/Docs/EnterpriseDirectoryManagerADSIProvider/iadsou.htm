IADsOU

The IADsOU interface is a dual interface that inherits from IADs. It is designed for representing and managing the Organizational Unit to which an account belongs. This interface provides support for directory services to organize accounts by country/region, locality (state/city/region), organization (company), and department. 

When a directory service provides hierarchical groupings of directory entries by country/region, locality, organization, and organization unit, you can use this and the related interfaces to expand the directory tree accordingly. In this case, the IADsOU interface is implemented by an Organizational Unit object that implements the IADsContainer interface as well.

IADsOU Properties

The property methods of the IADsOU interface get or set the properties described in the following table. For more information about the IADsOU interface, please refer to the MSDN Library.

Property

Description

BusinessCategory

Access: Read/Write

DataType: BSTR

A string indicating the general business function(s) performed by the organizational unit, for example "Accounting."

Description

Access: Read/Write

DataType: BSTR

The text that describes the Organizational Unit.

FaxNumber

Access: Read/Write

DataType: BSTR

The fax number of the Organizational Unit.

LocalityName

Access: Read/Write

DataType: BSTR

The name of the geographical region of the Organizational Unit.

PostalAddress

Access: Read/Write

DataType: BSTR

The postal address of the Organizational Unit.

TelephoneNumber

Access: Read/Write

DataType: BSTR

The telephone number of the Organizational Unit.

SeeAlso

Access: Read/Write

DataType: VARIANT

An array of ADsPath names of other directory objects which may be relevant to this object.
