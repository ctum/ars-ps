IADsOpenDSObject

Every directory object supports the IADsOpenDSObject interface that is designed to supply a security context for binding to a directory object. The Active Roles ADSI provider maintains the security context in its cache. Thus, throughout the connection within a process, once authenticated, the supplied user credentials are applied to any actions performed on this object and its child objects. This credential caching model applies to binding to different objects as well, provided that the binding takes place within the same connection and process.

In addition, with this interface, you can specify the binding options, such as the way in which the ADSI provider will search for Administration Service to connect to (for more information, see Connecting to a Service).

This topic describes the following methods provided by the IADsOpenDSObject interface:

  • OpenDSObject method

OpenDSObject Method

Returns a reference to an object to which you want to bind. The OpenDSObject method allows you to establish alternate credentials when binding to the specified directory objects. With this method, you can also specify the binding options (see the following Settings section). 

Function  OpenDSObject(ByVal BindStr,ByVal UserName, ByVal Passw, ByVal BindFlag)

Arguments

BindStr
[in] Required. String. Specifies the ADsPath of the directory object. For more information and examples of binding strings, refer to Binding to Directory Objects.
User
[in] Required. String. Specifies the user name in which security context you want to bind to the directory object identified with the argument BindStr. Set this argument to "" when you want to bind in the current user's security context.
Password
[in] Required. String. Specifies the user's password. Set this argument to "" when you want to bind in the current user's security context.
BindFlag
[in] Required. Integer. Specifies a binding option. For details, see the following Settings section.

Settings

The BindFlag argument can take any of the following values:

Constant

Value

Description

-

0

Means that the binding flag is not set. The provider will automatically select the Administration service to connect to. For more information, refer to Connecting to a Service.

ADS_FAST_BIND

32

The provider will not attempt to query the objectClass property of the object to which it tries to bind. Thus it will only expose the base interfaces supported by all directory objects instead of the full object support. A user can use this option to increase the performance in a series of object manipulations that involve only methods of the base interfaces. However, the provider will not verify that any of the requested objects actually exist on the server

ADS_EDMSERVER_BIND

32768

When the first argument (BindStr) includes a specific service computer name, ensures that the provider will connect to the service running on that computer.

ADS_EDMSERVER_REPLGROUP

65536

When this flag is set, use the binding string in the following format:
EDMS://<PublisherDNSName>/<objectDN>

The  <PublisherDNSName> refers to the DNS name of the computer that is the Publisher of a replication group while the <objectDN> refers to the DN of the directory object to which you want to bind.

The provider will try to connect to any Administration service that is a member of the replication group.

Note:

The BindFlag argument can also take values that are results of the following bitwise OR operations:
ADS_FAST_BIND OR ADS_EDMSERVER_BIND
ADS_FAST_BIND OR ADS_EDMSERVER_REPLGROUP

Remarks

The following code snippet (VBScript) illustrates how to use the OpenDSObject method.

Const ADS_EDMSERVER_BIND = 32768
Const ADS_FAST_BIND = 32
Set dso = GetObject("EDMS:")
 ' Supply full credentials to initiate a connection to the specific service
Set obj1 = dso.OpenDSObject("EDMS://AdminServiceComputer1/CN=John Doe,OU=Sales,DC=mycompany,DC=com", _
                "Administrator", _
                "secrete", _
                ADS_EDMSERVER_BIND)
 ' Do something with the bound object, obj1
MsgBox obj1.Class
 ' Bind to another object in the current user's security context.
 ' Set the binding flag to ADS_FAST_BIND
Set obj2 = dso.OpenDSObject("EDMS://CN=Ann Smith,OU=Sales,DC=mycompany,DC=com", _
                "","",ADS_FAST_BIND)
 ' Only basic interfaces can be used for the obj2 object
MsgBox obj2.Name

The first part of this code allows you to bind to the user object John Doe in the Administrator's security context. The use of the flag ADS_EDMSERVER_BIND ensures that the ADSI provider will connect to the Administration Service running on the computer AdminServiceComputer1.

The second part of this code illustrates a serverless fast binding to the user object Ann Smith (the flag ADS_FAST_BIND is set). The current user's security context is used.
