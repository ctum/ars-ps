edsScriptModule Class

This class is used to store information about the script module. In addition to standard AD attributes, Active Roles defines the following script module object attributes, which can be accessed using the Active Roles ADSI Provider.

edsaScriptLanguage

Syntax: DirectoryString

Access: Read/Write

Searchable: Yes.

Identifies the scripting language used to develop the script: PowerShell,  PerlScript,  VBScript, or  JScript. The attribute is case-sensitive.

 

edsvaFunctionsList

Syntax: DirectoryString, MultiValued

Access: Read/Write

Searchable: No.

For a given script module, lists the names of all the functions available in the script.

 

edsvaPowerShellDebugTraceOutput

Syntax: DirectoryString, SingleValued

Access: Read/Write

Searchable: No.

For a Windows PowerShell based script module, contains the debug trace output generated as specified in the debug trace level setting.

 

edsaPowerShellDebugTraceLevel

Syntax: Integer, SingleValued

Access: Read/Write

Searchable: Yes.

For a Windows PowerShell based script module, specifies the debug trace level: 0 - Turn script tracing off; 1 - Trace script lines as they are executed; 2 - Trace script lines, variable assignments, and function calls.

 

edsaScriptText

Syntax: DirectoryString

Access: Read/Write

Searchable: Yes.

Contains the script source text.

 

edsaScriptType

Syntax: Integer

Access: Read/Write

Searchable: Yes.

Identifies the script type: 0 - policy scripts; 1 - scheduled task scripts; 2 - library scripts.

 

edsvaPolicyScriptParameterDefinitions

Syntax: CaseIgnoreString, SingleValued

Access: Read only

Searchable: No.

Used to a request for parameter definitions in order to pass the name of the function that is used to declare parameters in the script module.  If not specified, it is assumed that the parameters are declared using the onInit function.
