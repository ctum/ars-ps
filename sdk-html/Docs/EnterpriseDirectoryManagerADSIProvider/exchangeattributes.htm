Exchange Related Attributes

In addition to the standard Exchange-related attributes, Active Roles defines the following attributes to extend the management of Exchange Server. The Active Roles ADSI Provider can be used to access all Exchange-related attributes, both those defined by Exchange Server and those defined by Active Roles itself.

edsaAdminGroup  

Syntax: Multi-value Case Ignore String.

Access: Read-only. Applies to all recipients.

Searchable: No.

Specifies a list of Administrative Groups that are defined in the recipient's Exchange Organization.

 

edsaAllExchangeTasks  

Syntax: Multi-value Case Ignore String.

Access: Read-only. Applies to all recipients.

Searchable: No.

Specifies a list of all Exchange tasks related to the recipient's object class. The list contains lDAPDisplayNames of Active Roles-specific attributes used to initiate Exchange tasks related to the recipient's object class.

 

edsaCreateMsExchMailbox  

Syntax: Boolean.

Access: Write-only. Applies to user objects.

Searchable: No.

Setting this attribute to TRUE initiates the Exchange task 'Create Mailbox'.This task requires that you set the following standard Exchange attributes: mailNickname and homeMDB

 

edsaDeleteEmail  

Syntax: Boolean.

Access: Write-only. Applies to all mail-enabled recipients.

Searchable: No.

Setting this attribute to TRUE initiates the Exchange task 'Delete E-Mail Addresses'.

 

edsaDeleteMailbox  

Syntax: Boolean.

Access: Write-only. Applies to all mail-enabled recipients.

Searchable: No.

Setting this attribute to TRUE initiates the Exchange task 'Delete Mailbox'.

 

edsaEstablishEmail  

Syntax: Boolean.

Access: Write-only. Applies to all mail-enabled recipients.

Searchable: No.

Setting this attribute to TRUE initiates the Exchange task 'Establish E-Mail Addresses'. This task requires that the following attributes have been set: edsaAdminGroup, mailNickname, and targetAddress.

 

edsaEstablishGroupEmail  

Syntax: Boolean.

Access: Write-only. Applies to groups.

Searchable: No.

Setting this attribute to TRUE initiates the Exchange task 'Establish E-mail Addresses'. This task requires that the following attributes have been set: edsaAdminGroup, mailNickname.

 

edsaExchangeTasksAvailable  

Syntax: Multi-value Case Ignore String

Access: Read-only. Applies to all recipients.

Searchable: No.

Specifies a list of all possibleExchange tasks for the recipient. The list contains lDAPDisplayNames of Active Roles-specific attributes used to initiate Exchange tasks that can actually be performed on the recipient, and depends on the recipient's object class and Exchange-related attributes previously set for the recipient.

 

edsaHideMembership

Syntax: Boolean.

Access: Write-only. Applies to groups.

Searchable: No.

Setting this attribute to TRUE initiates the Exchange task 'Hide Membership'.

 

edsaHomeMDB

Syntax: DN string.

Access: Read/Write. Applies to mailbox-enabled user objects.

Searchable: No.

Auxiliary attribute for the Exchange task 'Move Mailbox'. The client should set this attribute instead of explicitly setting the homeMDB attribute when performing the 'Move Mailbox' task.

 

edsaIsMsExchangePresent

Syntax: Boolean.

Access: Read-only. Applies to all recipients.

Searchable: No.

Specifies whether Exchange is present in current configuration. You can get a list of all possible Exchange Tasks for the recipient by reading the attribute edsaExchangeTasksAvailable.
When this attribute is set to TRUE, Exchange administration support is enabled in the Active Roles snap-in. Otherwise, Exchange administration support is disabled.

 

edsaMailboxSecurityDescriptor

Syntax: Security descriptor.

Access: Read/Write. Applies to mailbox-enabled user objects.

Searchable: No.

Specifies permissions assigned to security principals for administrative operations on the user's mailbox.

 

edsaMoveMailbox  

Syntax: Boolean.

Access: Write-only. Applies to mailbox-enabled user objects.

Searchable: No.

Setting this attribute to TRUE initiates the Exchange task 'Move Mailbox Exchange'. This task requires that you set the attribute edsaHomeMDB to specify the mailbox destination.

 

edsaMsExchMixedMode

Syntax: Boolean.

Access: Read-only. Applies to all recipients.

Searchable: No.

If the recipient's Exchange Organization is in mixed mode, this attribute is set to TRUE. Otherwise, the attribute value is not specified.

 

edsaRemoveAllMsExchAttributes

Syntax: Boolean.

Access: Write-only. Applies to all recipients.

Searchable: No.

Setting this attribute to TRUE initiates the Exchange task 'Remove Exchange Attributes Exchange'. This task removes all Exchange settings from the selected recipient object.

 

edsaUnhideMembership

Syntax: Boolean.

Access: Write-only. Applies to groups.

Searchable: No.

Setting this attribute to TRUE initiates the Exchange task 'Unhide Membership'.

 

Mailbox Attributes

This section lists the attributes specific to mailboxes on Exchange 2013 or later.

 

edsva-MsExch-ConvertToLinkedMailbox 

Syntax: Boolean.

Access: Write-only. Applies to users.

Searchable: No.

On a user account associated with a mailbox of the User Mailbox  type, set this attribute to TRUE in order to change the type of the mailbox to Linked Mailbox.

 

edsva-MsExch-CreateLinkedMailbox 

Syntax: Boolean.

Access: Write-only. Applies to users.

Searchable: No.

Setting this attribute to TRUE creates a linked mailbox.

 

edsva-MsExch-CreateEquipmentMailbox 

Syntax: Boolean

Access: Write-only. Applies to users.

Searchable: No.

Setting this attribute to TRUE creates an equipment mailbox.

 

edsva-MsExch-SharedMailbox 

Syntax: SID. Multivalued.

Access: Read/Write. Applies to users.

Searchable: No.

Lists the security identifiers of the user accounts that are allowed to access a given shared mailbox. When configuring a user object with a shared mailbox, a client can use this attribute to retrieve or change the list of mailbox users.

 

edsva-MsExch-CreateRoomMailbox

Syntax: Boolean

Access: Write-only. Applies to users.

Searchable: No.

Set this attribute to TRUE creates a room mailbox.

 

edsva-MsExch-MailboxDatabasesXML 

Syntax: String(Teletex).

Access: Read-only. Applies to Server Configuration Containers (the edsConfigurationContainer object class).

Searchable: No.

Contains XML information about the mailbox stores or databases that exist in the Exchange organization.

 

edsva-MsExch-MailboxType

Syntax:Integer

Access: Read-only. Applies to users.

Searchable: No.

Contains a numeric value that identifies the type of the mailbox, if any, associated with the user object. A client cannot set this attribute; it can only get the attribute value. This attribute can take the following values:

  • 0: Mailbox does not exist;

  • 1: User mailbox;

  • 2: Legacy mailbox (pre-Exchange 2007);

  • 3: Linked mailbox;

  • 4: Shared mailbox;

  • 5: Room mailbox;

  • 6: Equipment mailbox.

 

edsva-MsExch-MailboxTypeDescription 

Syntax: Case ignore string.

Access: Read-only. Applies to users.

Searchable: No.

Contains the mailbox type name, such as User Mailbox,  Equipment Mailbox, etc. (see the edsva-MsExch-MailboxType attributedescription).

 

edsva-MsExch-ResourceCustomProperties 

Syntax: Case ignore string. Multivalued

Access: Read/Write. Applies to users.

Searchable: No.

Lists the resource custom properties assigned to a given equipment or room mailbox. When configuring a user object with an equipment or room mailbox, a client can use this attribute to get or set the resource custom properties on the mailbox.

 

edsva-MsExch-ResourceCapacity 

Syntax: Integer.

Access: Read/Write. Applies to users.

Searchable: No.

Specifies the resource capacity for a given equipment or room mailbox. Attribute value is a non-negative integer in the range from 0 through 2,147,483,647. When configuring a user object with an equipment or room mailbox, a client can use this attribute to get or set the resource capacity on the mailbox.

 

edsva-MsExch-SharedMailboxUsers 

Syntax: SID. Multivalued.

Access: Read/Write. Applies to users.

Searchable: No.

Lists the security identifiers of the user accounts that are allowed to access a given shared mailbox. When configuring a user object with a shared mailbox, a client can use this attribute to retrieve or change the list of mailbox users.

 

edsva-MsExch-AutoReplyState

Syntax: Case Ignore String.

Access: Write-only.

Searchable: No.

Specifies whether to enable the mailbox for Automatic Replies. Possible values:

  • Enabled  The mailbox sends automatic replies until this attribute is cleared or set to Disabled.

  • Disabled  The mailbox does not send automatic replies; same as <not set>.

 

edsva-MsExch-AutoReplyExternalAudience

Syntax: Case Ignore String.

Access: Write-only.

Searchable: No.

Specifies whether to send Automatic Replies to senders outside of the organization. Possible values:

  • Known  Auto-reply to My Contacts only.

  • All  Auto-reply to all external senders.

  • None  Do not auto-reply to external senders; same as <not set>.

 

edsva-MsExch-AutoReplyExternalMessage

Syntax: Case Ignore String.

Access: Write-only.

Searchable: No.

Specifies the Automatic Replies message to be sent to senders outside of the organization.

 

edsva-MsExch-AutoReplyInternalMessage

Syntax: Case Ignore String.

Access: Write-only.

Searchable: No.

Specifies the Automatic Replies message to be sent to senders within the organization.

 

edsva-MsExch-EnableArchiveMailbox

Syntax: Boolean.

Access: Write-only.

Searchable: No.

Set this attribute to TRUE to create an archive mailbox for the mailbox user.

 

edsva-MsExch-DisableArchiveMailbox

Syntax: Boolean.

Access: Write-only.

Searchable: No.

Set this attribute to TRUE to disconnect the archive mailbox from the mailbox user.

 

edsva-MsExch-ArchiveMailboxName

Syntax: Case Ignore String.

Access: Read/Write.

Searchable: No.

Specifies the name of the mailbox user's archive mailbox.

 

edsva-MsExch-ArchiveMailboxDatabase

Syntax: DN String.

Access: Read/Write.

Searchable: No.

Specifies the Exchange mailbox database that contains the mailbox user's archive mailbox.

 

edsva-MsExch-ArchiveMailboxQuota

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies the mailbox user's archive mailbox size at which messages will no longer be accepted.

 

edsva-MsExch-ArchiveMailboxWarningQuota

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies the mailbox user's archive mailbox size at which a warning message is sent to the user.

 

edsva-MsExch-EnableCalendarAttendant

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether the Calendar Attendant is enabled on the mailbox.

 

edsva-MsExch-RemoveForwardedMeetingNotifications

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether the Calendar Attendant removes the meeting forwarding notifications.

 

edsva-MsExch-RemoveOldMeetingMessages

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether the Calendar Attendant removes old and redundant updates and responses.

 

edsva-MsExch-AddNewRequestsTentatively

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether the Calendar Attendant marks incoming meeting requests as "Tentative" on the calendar.

 

edsva-MsExch-ProcessExternalMeetingMessages

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether the Calendar Attendant processes meeting requests that originate outside the Exchange organization.

 

edsva-MsExch-SharingPolicyDN

Syntax: DN String.

Access: Read/Write.

Searchable: No.

Identifies the sharing policy that applies to the mailbox.

 

edsva-MsExch-RoleAssignmentPolicyDN

Syntax: DN String.

Access: Read/Write.

Searchable: No.

Identifies the management role assignment policy that applies to the mailbox.

 

edsva-MsExch-AddressBookPolicyDN

Syntax: DN String.

Access: Read/Write.

Searchable: No.

Identifies the address book policy that applies to the mailbox.

 

edsva-MsExch-RejectMessagesFrom

Syntax: Multi-value DN String.

Access: Read/Write.

Searchable: No.

Specifies the Exchange recipients that are not allowed to send messages to this mailbox. The mailbox will not accept messages from these senders.

 

edsva-MsExch-AcceptMessagesOnlyFrom

Syntax: Multi-value DN String.

Access: Read/Write.

Searchable: No.

Specifies the Exchange recipients that are allowed to send messages to this mailbox. If this attribute is not set, the mailbox accepts messages from all senders except those specified by the edsva-MsExch-RejectMessagesFrom attribute.

 

edsva-MsExch-RequireSenderAuthentication

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether the mailbox requires all senders to be authenticated.

 

edsva-MsExch-MailboxItemsTotal

Syntax: Integer.

Access: Read-only.

Searchable: No.

Identifies the total number of items held in the mailbox.

 

edsva-MsExch-MailboxLastLoggedOnBy

Syntax: DN String.

Access: Read-only.

Searchable: No.

Identifies the user account most recently used to log on to this mailbox.

 

edsva-MsExch-MailboxSize

Syntax: String.

Access: Read-only.

Searchable: No.

Identifies the current size of the mailbox.

 

edsva-MsExch-MailboxLastModified

Syntax: DateTime.

Access: Read-only.

Searchable: No.

Identifies the last date and time that a configuration change was made to the mailbox.

 

edsva-MsExch-MailboxMoveRequestScope

Syntax: Integer.

Access: Write-only.

Searchable: No.

When initiating a mailbox move request, use this attribute to specify whether you want to move both the primary mailbox and archive mailbox, or only the archive mailbox. If you don't set this attribute, the mailbox move request is configured to move only the primary mailbox. If you want to move the archive mailbox only, set this attribute to 1. If you want to move both primary mailbox and archive mailbox, set this attribute to 2.

 

edsva-MsExch-ArchiveMailboxEnabled

Syntax: Boolean.

Access: Read-only.

Searchable: No.

When set to TRUE, indicates that an archive mailbox is enabled for this mailbox user.

 

Unified Messaging Related Attributes

This section lists the attributes specific to Unified Messaging.

 

edsva-MsExch-UM-IsEnabled

Syntax: Boolean.

Access: Read-only.

Searchable: No.

Retrieve this attribute to check if Unified Messaging (UM) is enabled for the user. The attribute value of TRUE indicates that UM is enabled.

 

EDSVA-MsExch-UM-Enable

Syntax: Boolean.

Access: Write-only.

Searchable: No.

Setting this attribute to TRUE enables Unified Messaging (UM) for the user.

 

edsva-MsExch-UM-Disable

Syntax: Boolean.

Access: Write-only.

Searchable: No.

Setting this attribute to TRUE disables Unified Messaging (UM) for the user.

 

edsva-MsExch-UM-PINReset

Syntax: Boolean.

Access: Write-only.

Searchable: No.

Setting this attribute to TRUE resets the Outlook Voice Access PIN for the Unified Messaging (UM) enabled user.

 

edsva-MsExch-UM-MailboxPolicyDN

Syntax: DN string.

Access: Read/Write.

Searchable: No.

Specifies the distinguished name of the Unified Messaging (UM) mailbox policy that is associated with the UM-enabled user.

 

edsva-MsExch-UM-PIN

Syntax: Case Ignore String.

Access: Read/Write.

Searchable: No.

Use this attribute to specify the Outlook Voice Access PIN for the Unified Messaging (UM) enabled user.

 

edsva-MsExch-UM-PINResetOnFirstLogon

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Setting this attribute to TRUE forces the user to reset the Outlook Voice Access PIN when the user accesses the Unified Messaging system from a telephone.

 

edsva-MsExch-UM-ExtensionNumbers

Syntax: Multi-value Case Ignore String.

Access: Read/Write.

Searchable: No.

Use this attribute to specify the Outlook Voice Access PIN for the Unified Messaging (UM) enabled user.

 

edsva-MsExch-UM-SIPAddress

Syntax: Case Ignore String.

Access: Read/Write.

Searchable: No.

Specifies the Session Initiation Protocol (SIP) address or E.164 address for the UM-enabled user.

 

edsva-MsExch-UM-LockedOut

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Retrieve this attribute to check if the Unified Messaging (UM) lockout status of the user's mailbox. The attribute value of TRUE indicates that the user is locked out of their UM mailbox due to a number of attempts to input an incorrect Outlook Voice Access PIN.

 

edsva-MsExch-UM-AutomaticSpeechRecognitionEnabled

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether Automatic Speech Recognition (ASR) is enabled for the user's Unified Messaging mailbox.

 

edsva-MsExch-UM-CallsFromNonUsersAllowed

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether to allow incoming calls from unauthenticated callers through an auto attendant to be transferred to the Unified Messaging (UM) enabled user.

 

edsva-MsExch-UM-FaxEnabled

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specify whether the Unified Messaging (UM) enabled user is allowed to receive incoming faxes.

 

edsva-MsExch-UM-AnonymousCallersCanLeaveMessages

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether, for diverted calls without a caller ID, the caller is allowed to leave a message in the user's Unified Messaging (UM) mailbox.

 

edsva-MsExch-UM-CallAnsweringRulesEnabled

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether or not the Unified Messaging (UM) enabled user is allowed to create personal auto attendants by setting up Call Answering Rules for their user account.

 

edsva-MsExch-UM-OperatorExtensionNumber

Syntax: Case Ignore String.

Access: Read/Write.

Searchable: No.

Specify the extension number of the personal operator (assistant) for the Unified Messaging (UM) enabled user.

 

edsva-MsExch-UM-DialPlanDN

Syntax: DN string.

Access: Read/Write.

Searchable: No.

When adding a Unified Messaging (UM) address or extension for an UM-enabled user, use this attribute to specify the distinguished name of the UM dial plan to be associated with that address or extension.

 

Distribution Group Attributes

This section lists the attributes specific to distribution groups on Exchange 2013 or later.

 

edsva-MsExch-MemberJoinRestriction

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies whether owner approval is required to join the distribution group. The attribute value can be one of the following:

  • 0 (Open) - Anyone can join the group without being approved by the group owners.

  • 1 (Closed) - Members can be added only by the group owners.

  • 2 (ApprovalRequired) - Approval by group owners is required to join the group.

 

edsva-MsExch-MemberDepartRestriction

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies whether owner approval is required to leave the distribution group. The attribute value can be one of the following:

  • 0 (Open) - Anyone can leave the group without being approved by the group owners.

  • 1 (Closed) - Members can be removed only by the group owners.

  • 2 (ApprovalRequired) - Approval by group owners is required to leave the group.

 

edsva-MsExch-ModerationEnabled

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

When set to TRUE on a distribution group, indicates that messages sent to that group have to be approved by a moderator.

 

edsva-MsExch-ModerationNotificationSending

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies whether notifications are sent to users when they send a message to the moderated distribution group. The attribute value can be one of the following:

  • 0 (Always) - Notifications are sent to all external and internal senders.

  • 1 (Internal) - Notifications are sent only to the senders who are internal to the Exchange organization.

  • 2 (Never) - Notifications are not sent, except that the message is rejected by the moderator.

 

edsva-MsExch-ModeratedBy

Syntax: Multi-value DN String.

Access: Read/Write.

Searchable: No.

Identifies the persons who are responsible for moderating the messages sent to this distribution group.

 

edsva-MsExch-BypassModerationFor

Syntax: Multi-value DN String.

Access: Read/Write.

Searchable: No.

Identifies the senders whose messages do not require approval by the moderators of the distribution group.

 

edsva-MsExch-UseMAPIRichTextFormat

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies how messages in the MAPI rich text format (RTF) are handled for this mail recipient. The attribute value can be one of the following:

  • 0 (Never) - All messages are sent in plain text.

  • 1 (Always) - All messages are sent in RTF.

  • 2 (UseDefaultSettings) - Messages are sent in either RTF or plain text, depending on the settings of the MAPI client from which the message is sent.
