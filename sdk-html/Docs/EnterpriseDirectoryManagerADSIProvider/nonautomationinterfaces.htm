Non-Automation Interfaces

Non-Automation interfaces allow non-Automation clients (for example, C/C++ applications) low-overhead access to directory objects by providing Vtable access to methods for managing and searching directory service objects. This section describes the following non-Automation ADSI interfaces:

  • IDirectoryObject

  • IDirectorySearch

Note:

Automation clients (for example VBa, VBScript) cannot directly use the IDirectoryObject and IDirectorySearch interfaces. To make the Automation clients to search Active Directory for directory objects, use techniques described in Searching for Directory Objects.
