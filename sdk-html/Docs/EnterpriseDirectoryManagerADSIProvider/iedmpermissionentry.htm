IEDMPermissionEntry

The IEDMPermissionEntry interface is designed to manage Permission Entries (Permissions) that Access Template includes.

Properties

The IEDMPermissionEntry interface has the following properties.

Property

Description

AccessMask

Sets and returns the access rights to directory objects (a category of Permission).

AteType

Sets and returns a type of Permission (allow or deny).

ObjectTypeADsPath

Sets and returns an object property to which Permission applies. The object class can be set using the InheritedObjectTypeADsPath property.

InheritedObjectTypeADsPath

Sets and returns the object class to which Permission applies. The object property can be set using the ObjectTypeADsPath property.

ObjectType

Sets and returns a value of the Schema-Id-Guid attribute for the object property identified using ObjectTypeADsPath. The Schema-Id-Guid attribute uniquely identifies the object property in the schema.

InheritedObjectType

Sets and returns the object class to which Permission applies. The object property can be set using the ObjectTypeADsPath property.

Typical Permission Entries

The following code snippets illustrate how to create typical Permissions. For information about how to add those Permissions to an existing Access Template, refer to the sample code below in this topic.

In these examples, the variable NewATE refers to a Permission Entry you can create using the CreatePermissionEntry method provided by the IEDMAccessTemplate interface. The AteType property of that variable is set to 0 (permission that allows administrative operations). To employ the code snippets in your custom scripts, do the following:

  • Replace NewATE with the actual name of the Permission Entry variable used in the script

  • If you want to create a permission that denies administrative operations, replace NewATE.AteType=0 with NewATE.AteType=1

Sample Code (VBScript)

The following sample code illustrates how to add Permission Entries to Access Template.

option explicit
 ' DN of Access Template
const strAccessTemplateDN = "EDMS://CN=MyAT,CN=Access Templates,cn=Configuration"
const EDS_RIGHT_DS_READ_PROP = 16
const EDS_RIGHT_DS_WRITE_PROP = 32
const ACCESS_ALLOWED_ACE_TYPE = 0
Dim atObject
 ' Bind to Access Template
Set atObject = GetObject(strAccessTemplateDN)
Dim NewATE
 ' Create a new Permission Entry
set NewATE = atObject.CreatePermissionEntry
 ' Set properties of the newly created Permission Entry: read all properties for all classes
NewATE.AteType = ACCESS_ALLOWED_ACE_TYPE
NewATE.AccessMask = EDS_RIGHT_DS_READ_PROP OR EDS_RIGHT_DS_WRITE_PROP
 ' Add Permission Entry to Access Template
atObject.AddPermissionEntry(NewATE)
Set NewATE = Nothing
 ' Create a new Permission Entry
set NewATE = atObject.CreatePermissionEntry
 ' Set properties of the newly created Permission Entry: write description for user
NewATE.AteType = ACCESS_ALLOWED_ACE_TYPE
NewATE.AccessMask = EDS_RIGHT_DS_WRITE_PROP
NewATE.InheritedObjectTypeADsPath = "EDMS://user,Schema"     ' Apply to the User class
NewATE.ObjectTypeADsPath = "EDMS://description,Schema"       ' Apply to the "description" property
 ' Add Permission Entry to Access Template
atObject.AddPermissionEntry(NewATE)
Set NewATE = Nothing
 ' Commit changes
atObject.SetInfo

This script adds two permission entries to the MyAT Access Template. The first permission allows a Trustee to read/write all properties of any object. The second permission allows a Trustee to modify the description property of the user objects. For information about how to designate a Trustee, and for more examples of the VBScript codes, refer to Managing Access Templates.
