IEDM::CheckPropertyValues

The CheckPropertyValues method returns an object used to represent the IEDMComplianceInfoList interface. This method allows you to check whether values of the object's attributes are in compliance with the administrative policy. 

Function  CheckPropertyValues([ByVal RequestType],[ByVal RDN])

Arguments

RequestType

[in] Optional. Identifies the type of the request.

RDN

[in] Required, when RequestType is set to 9 (Rename); otherwise, this argument is optional. Identifies the object's relative distinguished name.

Settings

The RequestType argument can take any of the following values of the EDST_REQUEST_TYPE enumeration:

Constant

Value

Request Description

EDST_REQ_CREATE

4

Create a directory object

EDST_REQ_DELETE

5

Delete a directory object

EDST_REQ_GET

6

Retrieve properties of a directory object

EDST_REQ_SET

7

Set properties of a directory object

EDST_REQ_RENAME

9

Rename a directory object

EDST_REQ_COPY

10

Make a copy of a directory object

Remarks

When the RequestType argument is omitted, the Active Roles ADSI Provider sets that argument to -1. In this case, the default request type depends on the type of the previous operation:

  • When the processed directory object is copied or created, the default request type is COPY or CREATE, respectively. Otherwise, the default request type is SET.

For more information, refer to IEDMComplianceInfoRecord.
