Windows Terminal Services Attributes

The Active Roles ADSI Provider defines the following Windows Terminal Services (WTS) properties applied to the user objects to set and retrieve user configuration information in a Terminal Services environment.

Terminal Service

This section describes the user's attributes associated with the user interface elements on the Environment, Remote Control, Sessions, and Terminal Services Profile tabs in the User Properties dialog box.

edsaWTSUserConfigBrokenTimeoutSettings 

Syntax: Boolean.

Access: Read/Write.  

Searchable: No.

Indicates what happens when the connection or idle timers expire, or when a connection is lost due to a connection error. When set to False, the terminal session is disconnected. Otherwise, the session is reset.

This attribute is associated with the radio button group When a session limit is reached or connection is broken on the Sessions tab in the User Properties dialog box.

 

edsaWTSUserConfigDeviceClientDrives  

Syntax: Boolean.

Access: Read/Write.

Available only for Citrix ICA clients.

Searchable: No.

Specifies whether the terminal server automatically reestablishes client drive mappings at logon. When set to False, this attribute prevents the terminal server from an automatic reconnection to previously mapped client drives.

This attribute is associated with the check box Connect client drives at logon on the Environment tab in the User Properties dialog box.

 

edsaWTSUserConfigDeviceClientPrinters  

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether the terminal server automatically reestablishes client printer mappings at logon. When set to False, this attribute prevents the terminal server from an automatic reconnection to previously mapped client printers.

This attribute is associated with the check box Connect client printers at logon on the Environment tab in the User Properties dialog box.

 

edsaWTSUserConfigDeviceDefaultPrinter  

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether to automatically print to the client's default printer. When set to False, the client printer is not the default printer.

This attribute is associated with the check box Default to main client printer on the Environment tab in the User Properties dialog box.

 

edsaWTSUserConfigDisallowLogonTerminalServer

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether the user is permitted to log on to the Terminal server. When set to True, this attribute prevents the user from log on to the Terminal server.

This attribute is associated with the check box Allow logon to terminal server on the Terminal Services Profile tab in the User Properties dialog box.

 

edsaWTSUserConfigInheritInitialProgram  

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether to start the application defined by the edsaWTSUserConfigInitialProgram attribute when the user logs on. When set to FALSE, this attribute allows you to start that application.

This attribute is associated with the check box Start the following program at logon on the Environment tab in the User Properties dialog box.

 

edsaWTSUserConfigInitialProgram

Syntax: DirectoryString.

Access: Read/Write.

Searchable: No.

Specifies the path and file name of the application that starts when the user logs on to the Terminal server.

This attribute is associated with the box Program file name on the Environment tab in the User Properties dialog box.

 

edsaWTSUserConfigReconnectSettings

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies how you can reestablish a disconnected session. When set to False, the session can be reestablished from any computer. Otherwise, reconnection to a disconnected session is restricted to the computer that started the session. The latter option is supported only for Citrix ICA-based clients that provide a serial number when connecting.

This attribute is associated with the radio button group Allow reconnection on the Sessions tab in the User Properties dialog box.

 

edsaWTSUserConfigShadowingSettings

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies whether the user session can be shadowed as well as the parameters of shadowing. Shadowing allows the user to remotely control an active session of another user. You can either view or actively control the session. The value of this attribute defines the following modes of shadowing:

  • 0  - Remote control of the user session is disabled.

  • 1 - Specifies that the user's session can be actively controlled by other user with the keyboard and mouse. The system displays a message on the client that asks permission to view or take part in the session.

  • 2 - Specifies that the user's session can be actively controlled by other user with the keyboard and mouse. The user will be not notified about the control.

  • 3 - Specifies that other user can monitor the user's session. The system displays a message on the client that asks permission to view the session.

  • 4 - Specifies that other user can monitor the user's session. The user will be not notified about the session monitoring.
     

This attribute is associated with the user interface elements on the Remote Control tab in the User Properties dialog box.

 

edsaWTSUserConfigTerminalServerHomeDir

Syntax: DirectoryString.

Access: Read/Write.

Searchable: No.

Specifies the path to the user's home folder for terminal server logon. If edsaWTSUserConfigTerminalServerHomeDirDrive contains an empty string, this attribute specifies a local folder. Otherwise, it specifies a shared network folder. Note that each user on a Terminal Server should have a unique home folder to ensure that the application information is stored separately for each user in the multi-user environment.

This attribute is associated with the Local box and To boxes on the Terminal Services tab in the User Properties dialog box.

 

edsaWTSUserConfigTerminalServerHomeDirDrive

Syntax: DirectoryString.

Access: Read/Write.

Searchable: No.

Specifies a drive letter (followed by a colon) to be assigned to the shared network folder designated as the Terminal Server home folder using the edsaWTSUserConfigTerminalServerHomeDir attribute.

This attribute is associated with the drop-down list next to Connect on the Terminal Services tab in the User Properties dialog box.

 

edsaWTSUserConfigTerminalServerProfilePath

Syntax: DirectoryString.

Access: Read/Write.

Searchable: No.

Specifies the full network path to the user's profile when the user logs on to the Terminal Server. If you specify a profile path for logging on to Windows, the path is also used for logging on to Terminal servers unless you specify a Terminal Services profile path here.

This attribute is associated with the User Profile box on the Terminal Services tab in the User Properties dialog box.

 

edsaWTSUserConfigTimeoutSettingsConnections

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies the maximum amount of time (in milliseconds) that the user's session can remain active on the server. When the time limit is reached, depending on values of the edsaWTSUserConfigBrokenTimeoutSettings and edsaWTSUserConfigReconnectSettings attributes, the session will be disconnected or reset. When set to zero, this attribute allows the sessions to continue indefinitely.

This attribute is associated with the Active session limit list on the Sessions tab in the User Properties dialog box.

 

edsaWTSUserConfigTimeoutSettingsDisconnections

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies the maximum amount of time (in milliseconds) that a disconnected session remains active on the server. When the time limit is reached, the disconnected session ends. When set to zero, this attribute allows disconnected sessions to remain active on the server indefinitely.

This attribute is associated with the End a disconnected list on the Sessions tab in the User Properties dialog box.

 

edsaWTSUserConfigTimeoutSettingsIdle

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies the maximum amount of time (in milliseconds) that an idle session (session without client activity) remains on the server. When the time limit is reached, depending on values of the edsaWTSUserConfigBrokenTimeoutSettings and edsaWTSUserConfigReconnectSettings attributes, the session will be disconnected or reset. When set to zero, this attribute allows idle sessions to remain on the server indefinitely.

This attribute is associated with the Idle session limit list on the Sessions tab in the User Properties dialog box.

 

edsaWTSUserConfigWorkingDirectory

Syntax: DirectoryString.

Access: Read/Write.

Searchable: No.

Specifies the working directory path for the application that starts when the user logs on to the Terminal server. The full name of that application is stored in the edsaWTSUserConfigInitialProgram attribute.

This attribute is associated with the box Start in on the Environment tab in the User Properties dialog box.

Dial-in

This section describes the user's attributes associated with the user interface elements on the Dial-in tabs in the User Properties dialog box.

edsaDialinAccessPermissions

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies the remote access permissions assigned to the user. The permissions are determined by the attribute value in the following way:

  • 0 - Specifies that dial-in or virtual private network (VPN) remote access is allowed for this user.

  • 1 - Specifies that dial-in or VPN remote access is prohibited for this user.

  • 2- Specifies that remote access permission for this user is controlled through a remote access policy. The remote access permission profiles defined by the remote access policy apply the same permissions to more than one user.
     

This attribute is associated with the radio button group Remote Access Permission (Dial-in or VPN) on the Dial-in tab in the User Properties dialog box.

 

edsaDialinApplyStaticRoutes

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether to configure predefined routes for one-way initiated demand-dial routed connections. When set to TRUE, the predefined routes must be specified using the edsaDialinStaticRoutes attribute.

This attribute is associated with the check box Always Callback to on the Dial-in tab in the User Properties dialog box.

 

edsaDialinAssignStaticIP

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether to disregard the group dial-in profile settings and assign a static TCP/IP address to this user. When set to TRUE, the address specified using the edsaDialinStaticIP attribute will be assigned.

This attribute is associated with the check box Assign a Static IP Address on the Dial-in tab in the User Properties dialog box.

 

edsaDialinCallbackNumber

Syntax: DirectoryString.

Access: Read/Write.

Searchable: No.

Specifies the phone number to call back to when the edsaDialinCallbackOptions attribute is set to 2.

This attribute is associated with the box next to the check box Always Callback to on the Dial-in tab in the User Properties dialog box.

 

edsaDialinCallbackOptions

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies the callback options. The options are determined by the attribute value in the following way:

  • 0 - The user cannot use the callback feature.

  • 1- The user can set options for the callback feature.

  • 2 - The user must use the callback feature.
     

This attribute is associated with the radio button group Callback Options on the Dial-in tab in the User Properties dialog box.

 

edsaDialinCallerID

Syntax: DirectoryString.

Access: Read/Write.

Searchable: No.

Specifies the phone number from which the user is required to dial in. This attribute is used only if the edsaDialinVerifyCallerID attribute is set to TRUE. This attribute is associated with the box next to the Verify Called-ID check box on the Dial-in tab in the User Properties dialog box.

 

edsaDialinStaticIP

Syntax: Integer.

Access: Read/Write.

Searchable: No.

Specifies the static TCP/IP address for the user.

This attribute is associated with the box next to the check box Assign a Static IP Address on the Dial-in tab in the User Properties dialog box.

 

edsaDialinStaticRoutes

Syntax: Multi DirectoryString.

Access: Read/Write.

Searchable: No.

Specifies a list of the static routes enabled for dial-in connection. Each value of the list has the following format:

d.d.d.d/x.x.0.0. m, where

d.d.d.d - destination network IP address, in dotted decimal notation.

x.x.0.0 - the network mask used to determine the Network ID and Host ID portions of the destination address. In the network mask, every bit that is set (indicated as "x" here) belongs to the Network ID and every bit that is off (a value of zero) belongs to the Host ID.

m - cost metric for this route. The cost metric typically is the number of hops to the destination network. For example, if there is only one router between the calling router and the destination network, m=1.

For more information, see Microsoft Windows Server Routing and Remote Access Service Help.

 

edsaDialinVerifyCallerID

Syntax: Boolean.

Access: Read/Write.

Searchable: No.

Specifies whether the Caller-ID feature is enabled. This feature can be used to verify that the incoming call is coming from a specified phone number. When set to TRUE, the edsaDialinCallerID attribute must specify the phone number from which the dialing is allowed.

This attribute is associated with the checkbox Verify Called-ID on the Dial-in tab in the User Properties dialog box.
