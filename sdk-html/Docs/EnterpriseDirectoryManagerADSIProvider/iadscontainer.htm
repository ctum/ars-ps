IADsContainer

The IADsContainer interface enables a container object to create, delete and manage contained objects. Examples of container objects are domains and Organizational Units.

You can use the IADsContainer interface to either enumerate contained objects or manage these objects' life-cycle. An example would be to recursively navigate down a directory tree. By querying the IADsContainer interface on an ADSI object, you can determine if the object it represents has child objects. If the interface is not supported, the object is a leaf. Otherwise, it is a container. You can continue this process for the newly found container objects. To create, copy, or delete an object within a container, you can use the appropriate methods of the IADsContainer interface.

IADsContainer Methods

The IADsContainer interface defines the following methods:

    IADsContainer Methods

    Description

    get__NewEnum

    Creates an enumerator object that implements the IEnumVARIANT interface to enumerate through the children of the container object.

    GetObject

    Obtains an IADs interface pointer to a directory object in the container. The object is identified by its schema class and the relative name. If the schema class name is not provided, retrieves an interface pointer on the first object found in the container with the specified name.

    Create

    Sets up a request to create a directory object of the specified schema class and a given name in the container. The object is not made persistent until SetInfo is called on the new object. This allows for setting mandatory properties on the new object.

    Delete

    Given the name of a directory object, deletes that object from the container. The object to be deleted must be a leaf object or an empty sub-container.

    CopyHere

    Creates a new copy of the specified directory object in the container.

    MoveHere

    Moves a specified object within a directory service.
    Note, that when you move an object, you cannot rename that object simultaneously.

Remarks

In Visual Basic, you can use the For Each statement to invoke the get__NewEnum method implicitly (see Sample Code below).
The CopyHere method only allows you to create a copy of a user, a group, a Managed Unit, and an inetOrgPerson. Copying other objects is not supported as well as copying objects across OUs.

IADsContainer Properties

In addition, the IADsContainer interface property methods provide access to the following properties:

Property

Description

Filter

Access: Read /Write

Data Type: Variant

The filter is used to select object classes in a given enumeration. This is a variant array, each element of which is the name of a schema class. If Filter is not set or set to empty, all objects of all classes are retrieved by the enumerator.

 Sample Code (Visual Basic)

The following code snippet shows how to enumerate objects using the Filter property of the IADsContainer interface.

Dim cont As IADsContainer
Dim usr As IADsUser
Set cont = GetObject("EDMS://OU=Sales,DC=mycompany,DC=com")
' Define the filter to only enumerate the user and computer objects
cont.Filter = Array("user","computer")
For Each usr In cont
   Debug.Print usr.Name
