Connecting to ADSI Provider with ADO

A major function of Active Roles is to resolve queries for information about directory objects. To facilitate search operations, the Active Roles ADSI provider supports both OLE DB and IDirectorySearch. Since ActiveX Data Objects (ADO) is a simple scriptable data access object model on top of OLE DB, the OLE DB interfaces are very helpful to Visual Basic programmers and Web page script writers.

In order for ADO to communicate with ADSI, you must use at least two ADO objects: Connection and RecordSet. These ADO objects serve to authenticate users and enumerate results, respectively. For information about using RecordSet, refer to Searching for Directory Objects.

The Connection object loads the OLE DB provider and validates user credentials. In Visual Basic, you call CreateObject("ADODB.Connection") to create an instance of a Connection object, and then set the Provider property of the Connection object to "ADsDSOObject". "ADODB.Connection" is the ProgID of the Connection object and "ADsDSOObject" is the name of the OLE DB provider in ADSI. If no credentials are specified, the credentials of the currently logged on user are assumed.

The following code snippet illustrates how to create an instance of a Connection object using VBScript.

Set con = CreateObject("ADODB.Connection")
con.Provider = "ADsDSOObject"

You can specify user authentication data by setting the properties of the Connection object. The user-authentication properties supported by ADSI are listed in the following table.

Property

Description

"User ID"

A string that identifies the user whose security context is used when performing the search. If not specified, the default is the logged on user (or the user being impersonated by the calling process). Precede the user name with the name of the user's home domain and separate the names with a backslash, such as DOMAIN\USER.

"Password"

A string that specifies the password of the user identified by "User ID".

"Encrypt Password"

A Boolean value that specifies whether the password is encrypted. The default is False.

"ADSI Flag"

A set of flags from the ADS_AUTHENTICATION_ENUM enumeration (see the MSDN library) that specify the binding authentication options. The default is zero.

Note:

To connect to ADSI provider installed on a remote computer, the ADSI Flag property must be set to 32768. The NETBIOS name of that computer is specified in the binding strings used in the script. For examples of the code samples, refer to Searching for Directory Objects.

The following code snippet shows how to set these properties in the Connection object.

Set oConnect = CreateObject("ADODB.Connection")
oConnect.Provider = "ADsDSOObject"
oConnect.Properties("User ID") = stUser
oConnect.Properties("Password") = stPass
oConnect.Properties("ADSI Flag") = 32768
oConnect.Properties("Encrypt Password") = True
oConnect.Open "DS Query", stUser, stPass
