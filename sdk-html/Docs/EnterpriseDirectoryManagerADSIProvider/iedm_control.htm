Control Property

Read/Write. Sets and returns values of the Active Roles controls.

Using this property, you can set the value of the InControl-type control that will be placed in a request to perform an administrative operation.  You can also get values of the OutControl-type controls Administration Service returns to your client application. For more information, refer to Active Roles controls.

[VBScript]
Obj.Control(CName,[bKeep])[=Value]




[PowerShell]
$Obj.Control($CName,[$bKeep],$Value)

Arguments

Obj
Required. Always a name of the processed directory object.
CName
[in] Required. String. Contains the Active Roles control ID.
bKeep
[in] Optional. Boolean. The default value is FALSE.
When FALSE, the script sends an InControl-type control to Administration Service only when you call such methods as SetInfo, GetInfo, GetInfoEx or CheckPropertyValues for the first time. Otherwise, the control will be sent each time you call those methods. This argument takes no effect when you get the control value.
Value
[in] Optional. When provided, assigns the specified value to the Active Roles control identified by the CName argument. 
 
Note

In PowerShell, the Control property is a method of the IEDM interface.

Example Code (VBScript)

The following sample code illustrates how you can use the Control property to set an Active Roles built-in control in your custom script. 


const EDS_CONTROL_LDAP_SERVER = 13
 ' Create User object in the specified OU
Set cont = GetObject("EDMS://OU=Sales,DC=Mycompany,DC=com")
Set usr = cont.Create("user", "CN=John MCDuglas")
 ' Set the User properties
usr.Put "samAccountName", "JMCDuglas"
usr.Put "edsaAccountIsDisabled", FALSE
 ' Set the InControl value
usr.Control(EDS_CONTROL_LDAP_SERVER)="FirstDC.Mycompany.com"
 ' Commit changes
usr.SetInfo

This script creates the new user John MCDuglas and sets the built-in control EDS_CONTROL_LDAP_SERVER to the name of the domain controller FirstDC.Mycompany.com. When Administration Service receives that control, the request  to create the user will be processed by the specified DC (FirstDC.Mycompany.com).
