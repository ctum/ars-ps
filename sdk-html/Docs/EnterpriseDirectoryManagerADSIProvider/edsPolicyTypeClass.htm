edsPolicyType Class

This class stores information about Active Roles policy extension object which can be used either in policy objects to define a custom policy or in workflows to define a custom workflow activity. By instantiating this class, you can extend the list of policies or the list of workflow activities available in Active Roles. In addition to standard AD attributes, Active Roles defines the following Policy Type object attributes, which can be accessed using the Active Roles ADSI Provider.

edsaPolicyTypeIcon

Syntax: OctetString, SingleValued

Access: Read/Write

Searchable: Yes.

Stores an icon used to denote the Policy Type object. You should supply an Icon (ICO) file containing two images, 16 x 16 and 32 x 32 pixels in size, with color depth not more than True Color (24-bit).

 

edsaPolicyTypeID

Syntax: CaseIgnoreString

Access: Read only

Searchable: Yes.

Stores the Policy Type object's ID. The attribute can store any string value, though it must be unique amongst all the instances of the edsPolicyType class.

 

edsaPolicyTypeScriptModuleGUID

Syntax: OctetString

Access: Read/Write

Searchable: Yes.

Identifies the script module associated with the Policy Type object.

 

edsaPolicyTypeCategory

Syntax: Enumeration, SingleValued

Access: Read/Write

Searchable: Yes.

Specifies the category of policy type. Possible values: 1 - provisioning; 2 - deprovisioning; 3 - workflow activity.

 

edsvaPolicyScriptParameterDefinitions

Syntax: CaseIgnoreString, SingleValued

Access: Read only

Searchable: No.

Calculated attribute. Contains information (in XML format) about parameters declared in the script module associated with the Policy Type object.

 

edsaFunctionToDeclareParameters

Syntax: DirectoryString, SingleValued

Access: Read/Write

Searchable: Yes.

Contains the name of the script function that declares the configurable parameters for the administration policy or workflow activity that is based on this policy type. If not set, assumes that the parameters are declared using onInit.

 

edsaFunctionToRun

Syntax: DirectoryString, SingleValued

Access: Read/Write

Searchable: Yes.

Used in workflow activity policy type. Contains the name of the script function to run by the workflow activity that is based on this policy type. If not set, the  policy event handler function found in the script is run.

 

edsvaPolicyTypeXMLDefinition

Syntax: CaseIgnoreString

Access: Read only

Searchable: No.

Calculated attribute. Contains information (in XML format) that describes the Policy Type object.
