Binding to Active Roles-Specific Objects

The Active Roles ADSI provider allows you to bind to objects that are specific to Active Roles, such as Managed Units, Access Templates, Policy Objects, etc.

The following table contains the values of objectClass attribute for some Active Roles-specific objects. 

Object Type

The ObjectClass Attribute Value

Access Template

edsAccessTemplate

Access Template Link

edsACE

Managed Unit

edsManagedUnit

Policy Object

edsPolicyObject

The above objects reside in the Configuration container (EDMS://CN=Configuration), within containers having the names corresponding to the object type, for example Access Templates, Managed Units, etc. Only Policy Objects reside in the Administration/Policies container.

You should first bind to an object in order to navigate its namespace, search for specific information, read or modify the object properties.

You can bind directly to the object in question by specifying the object's distinguished name in the binding string. After binding to the object, you can use the appropriate ADSI interfaces. All objects specific to Active Roles support the IADs and IADsPropertyList interfaces. Other interfaces are supported depending on the type of the object (for more information, see Active Roles-Specific Interfaces).

You can bind to a container within which the objects reside and then use the IADsContainer interface to enumerate and manage the contained objects.

When specifying binding information, you may optionally supply additional binding characteristics, such as user name, password, or the domain controller name.

Active Roles ADSI Provider Binding String

The Active Roles ADSI Provider binding string has the following format:

EDMS://<DomainDCName>/<Object name>
<DomainDCName>
Optional entry. Specifies the name of the domain controller or domain where you want the provider to search for available Administration Service. For details, see Connecting to Service.
<Object name>
Required entry. Represents a specific directory object. For Active Roles-specific objects, <Object name> can refer to the object's distinguished name (DN) or the object's GUID.

Notes:

In the binding string above, EDMS: specifies the Active Roles ADSI provider. The ":"(colon) is part of the provider name. The provider name is case sensitive: EDMS is different from edms.

For examples of binding strings that use a DN or GUID, see the appropriate topics in Related Topics.

The following table contains examples of binding strings you can use to bind to objects specific to Active Roles.

Object

Binding String Example

Access Template (name: AT1, container: Access Template)

EDMS://cn=AT1,cn=Access Templates,cn=Configuration

Access Template Link (name: Default Security Settings, container: AT Links)

EDMS://cn=Default Security Settings,cn=AT Links,cn=Configuration

Managed Unit (name: MU1, container: Managed Units)

EDMS://cn=MU1,cn=Managed Units,cn=Configuration

Policy Object (name: PO1, container: Administration)

EDMS://cn=PO1,CN=Administration,CN=Policies,CN=Configuration

Note:

The distinguished name of an object specific to Active Roles does not contain the domain name. It contains only the object's relative distinguished name (RDN) and the name of the container within which the object resides.

Using the RootDSE Properties

Active Roles supports binding to the rootDSE object. The properties of the rootDSE object allow you to get distinguished names of the Active Roles schema and the configuration namespace containers that hold objects specific to Active Roles. Whenever possible, we recommend you use the rootDSE object to bind to those objects. For more information about rootDSE, see Using the rootDSE object. 

The following code snippet shows how to use the IADsContainer interface for enumeration of all available Policy Objects. This example demonstrates how to get the distinguished name of the container where the Policy Objects reside using the rootDSE object.

Dim cont, obj,ss,EDMConfig,objDN
' Bind to the rootDSE
Set rootDSE = GetObject("EDMS://rootDSE")
rootDSE.GetInfoEx Array("edsvaAdministrationPoliciesDN"),0
' Get DN of the container for Policy Objects.
contDN = rootDSE.Get("edsvaAdministrationPoliciesDN")
' Bind to the Policy Objects container
Set cont = GetObject("EDMS://" & contDN)
'Define the filter to only enumerate Policy Objects
cont.Filter = Array("edsPolicyObject")
ss=""
For Each obj In cont
   ss= ss + CStr(obj.Name) + vbCrLf
Next
' Output names of the found Policy Objects
MsgBox ss

Directly Binding to Object Specific to Active Roles Using the GetObject Function

In languages that support Automation (for example Visual Basic, VBScript), use the GetObject function with a fully specified Active Roles ADSI Provider binding string. This binds to the object using the current user's logon and security credentials. If you need to bind to an object using a different user or security credential, for Automation clients, use IADsOpenDSObject instead.

The following VBScript code snippet shows how to bind to the desired object using the GetObject function.

Dim objEDM,rootDSE,objDN,EDMConfig
' Bind to the rootDSE
Set rootDSE = GetObject("EDMS://RootDSE")
rootDSE.GetInfoEx Array("edsvaAdministrationPoliciesDN"),0
' Get DN of the container for Policy Objects.
contDN=rootDSE.Get("edsvaAdministrationPoliciesDN")
' Set the distinguished name of the desired Policy Object
objDN="EDMS://cn=MyPolicyObject," + contDN
' Bind to the desired object
Set objARS = GetObject(objDN)

Binding to Object Specific to Active Roles Using the IADsOpenDSObject Interface

To establish credentials when passing a bind request to Administration Service, use the OpenDSObject method of the IADsOpenDSObject interface instead of GetObject as shown in the following Visual Basic code snippet:

Const ADS_EDMSERVER_BIND = 32768
Dim openDS As IADsOpenDSObject
Dim objUser As IADsUser
Set openDS = GetObject("EDMS:")
Set objUser=openDS.OpenDSObject("EDMS://ARServiceName/cn=MyPolicyObject,CN=Administration,CN=Policies,CN=Configuration",_
                                "Administrator","password",ADS_EDMSERVER_BIND)

Note:

Explicit credentials in the OpenDSObject method of IADsOpenDSObject can only be used when you connect to Administration Service running on a remote computer. In this script, the ARServiceName refers to the DNS name of the Administration service computer.
Setting the flag in the OpenDSObject method to ADS_EDMSERVER_BIND ensures that the provider will connect to the Administration Service running on the ARServiceName computer. For more information, please refer to IADsOpenDSObject.
