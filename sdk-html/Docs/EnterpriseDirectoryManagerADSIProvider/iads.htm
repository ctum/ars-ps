IADs

The IADs interface defines the basic object features, that is, properties and methods, of any ADSI object. Examples of ADSI objects include users, computers, and groups.

For any ADSI object, the IADs interface delivers methods that enable ADSI clients to:

  • Get object identification by name, class, or ADsPath

  • Identify the object's container that manages the object's creation and deletion

  • Retrieve the object's schema definition

  • Load object's attributes to the property cache and send changes to the Active Roles Administration Service

  • Access and modify the object's attribute values in the property cache

IADs Methods

The IADs interface defines the following methods:

 

IADs Methods

 Description

GetInfo

Loads the property values of the ADSI object from the Active Roles Administration Service into the property cache.

SetInfo

Sends the property values of the ADSI object from the property cache to the Active Roles Administration Service.

Get

Retrieves from the property cache a property of a given name. The property can be single-value or multi-value. The property value is represented as either a variant for a single-value property or a variant array (of VARIANT or bytes) for a property that allows multiple values.

Put

Sets new values to a property of a given name in the property cache. To propagate the changes to the directory store, you must call SetInfo on the object after calling Put. To manipulate multi-value properties, use PutEx, which allows you to append a value to or remove one from an existing array of attribute values.

GetEx

Retrieves from the property cache a property of a given name. The returned property values can be single-value or multi-value. Unlike the Get method, the property values are returned as a variant array of VARIANT or a variant array of bytes for binary data. A single-value property is then represented as an array of a single element.

PutEx

Modifies cached values of a specified property with a greater degree of control. For example, you can append a value to the existing values of a multi-value property, change all the values at once, remove the specified values from the set, or delete all the values at once.

GetInfoEx

Loads the values of specified properties of the object from the directory store into the ADSI object's property cache. Use GetInfoEx to refresh values of the selected properties in the property cache of an ADSI object. Use GetInfo to refresh all the property values.

IADs Properties

In addition, the IADs interface property methods provide access to the following properties:

 

Property

Description

ADsPath  

Access: Read

Data Type: BSTR

The ADsPath string of the object. The string uniquely identifies the directory object to ADSI. The object can always be retrieved using this path.

Class

Access: Read

Data Type: BSTR

The name of the Schema class of the directory object.

GUID

Access: Read

Data Type: BSTR

The globally unique identifier of the directory object. The IADs interface converts the GUID from an octet string, as stored on the directory server, into a string format.

Name

Access: Read

Data Type: BSTR

The relative name of the directory object as named within the directory service (common name).

Parent

Access: Read

Data Type: BSTR

The ADsPath string of the parent container. The ADsPath should always be used to retrieve an object's interface pointer.

Schema

Access: Read

Data Type: BSTR

The ADsPath string of the Schema class object of the directory object.

Automation clients use the dot notation to access object properties, for example:

Dim Obj As IADs
Dim objName As String
' Bind to an object
Obj=GetObject(EDMS://<Object's DN>)
' Retrieve the object name
objName=Obj.Name 

Examples (VBScript)

The following code snippet shows how to retrieve single-value and multi-value properties of the user object.

Dim usr, location, otherNumbers, ss
on error resume next
' Bind to the user object
Set usr = GetObject("EDMS://CN=bosse,OU=Sales,DC=mycompany,DC=Com")
' Load the user object properties from the Active Roles Administration Service
usr.GetInfo
' Retrieve the single value property (locality)
location = usr.Get("l")
MsgBox "Locality: " + VbCrLf + CStr(location)
' Retrieve the multiple value property (otherHomePhone)
otherNumbers = usr.GetEx("otherHomePhone")
' Display the values of the otherHomePhone property
ss = ""
For Each homeNum In otherNumbers
   ss = ss + Cstr(homeNum) + VbCrLf
Next
MsgBox "Other home numbers: " + VbCrLf + ss

The following code snippet shows how to set single-value and multi-value properties of the user object.

Dim usr, location, phone
Dim Ads_Property_Append
' Set the PutEx control code
ADS_PROPERTY_APPEND = 3
' Bind to the user object
Set usr = GetObject("EDMS://CN=bosse,OU=Sales,DC=mycompany,DC=Com")
' Prompt to specify new locality
location = InputBox("Specify the locality for the user " + usr.Name + " User's new locality:")
' Set the new locality
usr.put "l", location
' Prompt to specify the first phone number
phone = InputBox("Specify the first home phone for the user " + usr.Name + " User's home phone:")
usr.PutEx ADS_PROPERTY_APPEND, "OtherhomePhone", Array(Cstr(phone))
' Send the cached property values of the user to the Active Roles Administration Service.
usr.SetInfo' Prompt to specify the second phone number
phone = InputBox("Input the second home phone of the " + usr.Name +  " User's home phone")
usr.PutEx ADS_PROPERTY_APPEND, "OtherhomePhone", Array(Cstr(phone))
' Send the cached property values of the user to the Active Roles Administration Service.
usr.SetInfo

Note:

To save changes in the object's multi-value properties, you should call the SetInfo method each time you call the PutEx method. SetInfo only sets values for the last PutEx call; multiple calls for PutEx prior to SetInfo does not produce an accumulative affect.
